    통합 결제 대시보드 기획서 [4-21]¶

    작성일: 2026-03-10 | 버전: v0.1 참조: research/shopify-japan-payment.md (komoju·Shopify 대시보드 분리 문제)

    1. 문제 정의¶

    1.1 Shopify 구조적 한계¶

    Shopify에서 일본 판매자가 Shopify Payments + komoju를 병행 사용할 경우:

    Shopify Payments 매출
    → Shopify 어드민 대시보드에만 집계

komoju 매출 (콘비니·PayPay·LINE Pay 등)
    → komoju 별도 대시보드에서만 확인 가능
    → Shopify 대시보드와 자동 통합 없음

    판매자가 겪는 실제 불편: - 두 개의 대시보드를 오가며 매출 수동 합산 필요 - 정산 주기 상이 (Shopify vs komoju 별도 정산 사이클) - 환불 시 어느 PG에서 발생했는지 추적 복잡 - 세무·회계 처리 시 이중 데이터 대조 필수 (월 평균 3~5시간 추가 소요 추정) - 판매자가 실수로 재고를 중복 확인하는 운영 오류 발생

    1.2 누리메이트의 해결 방향¶

    누리메이트는 모든 결제 수단의 정산 데이터를 단일 어드민 대시보드에 통합 표시한다. 판매자는 어떤 PG사를 통해 결제가 이루어졌든, 하나의 화면에서 전체 매출·정산·수수료를 확인한다.

    2. 기능 범위¶

    2.1 통합 대시보드 개요 (어드민 홈)¶

    ┌─────────────────────────────────────────────────────────┐
│  누리메이트 어드민 — 정산 현황                              │
├─────────────────────────────────────────────────────────┤
│  [오늘 매출]         [이번 달 누계]       [미정산 잔액]     │
│  ₩ 1,234,500        ₩ 23,450,000        ₩ 8,900,000     │
├─────────────────────────────────────────────────────────┤
│  채널별 매출 현황 (오늘)                                   │
│  ┌────────────┬────────────┬─────────────────────┐      │
│  │ 채널       │ 매출액     │ 수수료              │      │
│  ├────────────┼────────────┼─────────────────────┤      │
│  │ 한국 카드  │ ₩ 780,000  │ 2.5% = ₩ 19,500    │      │
│  │ 카카오페이 │ ₩ 230,000  │ 1.9% = ₩ 4,370     │      │
│  │ 콘비니(JP) │ ¥ 45,000   │ 2.75% = ¥ 1,238    │      │
│  │ PayPay(JP) │ ¥ 12,000   │ 2.0% = ¥ 240       │      │
│  └────────────┴────────────┴─────────────────────┘      │
│                                              합계 → 상세  │
└─────────────────────────────────────────────────────────┘

    2.2 핵심 기능 정의¶

    기능 1: 실시간 정산 현황¶

    표시 항목 설명
    전체 매출 (오늘/주/월/커스텀) 모든 PG 채널 합산
    채널별 매출 분리 한국 PG, 일본 PG, 글로벌(Stripe) 각각 표시
    수수료 차감 후 실수령액 판매자 실제 입금 예정 금액
    정산 예정일 채널별 정산 주기 기반 D+1, D+2 등 표시
    미정산 잔액 아직 입금 전 금액

    기능 2: 채널별 수수료 비교 UI¶

    이번 달 수수료 현황
──────────────────────────────────────────────
 KG이니시스 (한국 카드)    ₩ 45,230   2.3%
 카카오페이               ₩ 12,450   1.9%
 GMO-PG (일본 카드)       ¥ 8,900    2.5%
 콘비니 결제              ¥ 3,200    2.75%
 Stripe (글로벌)          $ 23.40    2.9%
──────────────────────────────────────────────
 총 수수료 비용           ₩ 96,800 (환산)
 누리메이트 플랫폼 수수료  ₩ 234,500  1.0%

    기능 3: 정산 내역 상세 조회¶

    • 주문별 정산 상태 추적 (결제 완료 → 정산 처리 중 → 정산 완료)
    • 환불 처리 시 원 PG에서 자동 차감 처리 및 정산 재계산
    • 채널별 정산 주기 달력 뷰 (언제 얼마가 입금되는지 미리 확인)

    기능 4: 통화 통합 뷰¶

    • 다통화 매출을 선택한 기준 통화(기본: KRW)로 환산 표시
    • 환율 기준일 명시 (정산일 기준 환율 vs 결제일 기준 환율 선택)
    • 원화·엔화·달러 각각 원본 금액도 병행 표시 (세무 대응)

    3. 기술 아키텍처¶

    3.1 PG사별 정산 API 연동 구조¶

    [PG사 정산 API]                    [누리메이트 정산 집계 레이어]

KG이니시스 Settlement API  ──────▶
토스페이먼츠 Settlement API ──────▶  정산 데이터 수집기
GMO-PG Settlement API      ──────▶  (각 PG별 어댑터)   ──▶ 통합 DB ──▶ 대시보드
Stripe Connect API          ──────▶
Paymentwall API             ──────▶

    3.2 데이터 수집 방식¶

    방식 설명 적용 PG
    Webhook 실시간 결제 완료 즉시 이벤트 수신 Stripe, Toss
    API 폴링 주기적으로 정산 내역 조회 KG이니시스, GMO-PG
    CSV Import 자동화 어려운 경우 파일 가져오기 지원 (임시) 초기 단계 일부 PG

    3.3 정산 데이터 표준화¶

    각 PG사마다 정산 데이터 형식이 다르므로 누리메이트 내부 표준 스키마로 변환:

    {
  "settlement_id": "NRM-2026-031001",
  "order_id": "ORD-20260310-0042",
  "pg_channel": "gmo_pg",
  "currency": "JPY",
  "gross_amount": 15000,
  "pg_fee_rate": 0.025,
  "pg_fee_amount": 375,
  "net_amount": 14625,
  "settlement_date": "2026-03-11",
  "status": "settled"
}

    4. 사용자 경험 (판매자 관점)¶

    4.1 온보딩 시나리오: 일본 진출 판매자¶

    기존 (Shopify + komoju): 1. Shopify 대시보드 로그인 → 카드 매출 확인 2. komoju 대시보드 별도 로그인 → 콘비니·PayPay 매출 확인 3. 엑셀로 두 데이터 수동 합산 4. 수수료 별도 계산 후 실수령액 추산 → 소요 시간: 30~60분/일

    누리메이트: 1. 어드민 대시보드 접속 → 모든 채널 매출 즉시 확인 → 소요 시간: 1분

    4.2 주요 알림 자동화¶

    알림 유형 조건 채널
    정산 입금 완료 PG사 정산 완료 시 어드민 + 이메일 + 카카오 알림톡
    정산 지연 감지 예상 정산일 초과 시 어드민 + 이메일
    수수료 이상 감지 평소 대비 수수료율 2배 초과 어드민
    미정산 잔액 기준 초과 미정산 잔액 설정 기준값 초과 이메일

    5. 구현 로드맵¶

    MVP (2026년 6월 런칭 시)¶

    • [ ] 한국 PG (KG이니시스, 토스페이먼츠) 정산 통합 표시
    • [ ] 실시간 매출 합산 대시보드 홈 위젯
    • [ ] 주문별 정산 상태 추적
    • [ ] KRW 기준 다통화 환산 표시

    Phase 2 (2026 Q3)¶

    • [ ] 일본 PG (GMO-PG) 정산 통합
    • [ ] 채널별 수수료 비교 UI
    • [ ] 정산 예정일 달력 뷰
    • [ ] 정산 데이터 CSV 내보내기 (세무용)

    Phase 3 (2026 Q4+)¶

    • [ ] Stripe Connect 정산 통합
    • [ ] Paymentwall 정산 통합
    • [ ] 수수료 트렌드 분석 (월별 채널별 수수료율 변화 추적)
    • [ ] 회계 솔루션 연동 (세금계산서 자동 발행 API)

    6. 차별화 포지셔닝¶

    항목 Shopify + komoju 누리메이트
    정산 현황 조회 2개 대시보드 분리 단일 통합 대시보드
    일본 PG 수수료 확인 komoju 별도 로그인 어드민에서 즉시 확인
    정산 합산 수동 엑셀 작업 자동 실시간 집계
    다통화 뷰 불가 KRW/JPY/USD 통합 환산
    정산 예정일 예측 불가 달력 뷰로 미리 확인
