    누리메이트 기능 기획서 개발 우선순위¶

    작성일: 2026-03-11 | 버전: v1.0 팀 리소스: 웹프론트엔드 2명, 백엔드 2명, Flutter 앱 2명, 디자이너 1명. 총 7명 개발·디자인. 1차 런칭 목표: 2026년 6월

    우선순위 기준¶

    등급 정의
    P0 런칭 필수 — 2026년 6월 1차 런칭에 없으면 서비스 자체가 불가능한 기능
    P1 런칭 직후 — 런칭 후 3개월 이내(2026년 9월) 출시 필요. 판매자 리텐션에 직결
    P2 2026 H2 — 2026년 하반기 목표. 경쟁 우위 강화
    P3 장기 로드맵 — 2027년 이후 또는 조건부 실행

    전체 Spec 우선순위 매트릭스¶

    파일명 우선순위 근거
    seller-store-creation-ux.md P0 셀러 온보딩 플로우 없으면 서비스 진입 불가. 핵심 사용자 경험
    domain-architecture.md P0 서브도메인 자동 생성·커스텀 도메인 연결 없으면 스토어 URL 자체가 없음
    migration.md (카페24·메이크샵) P0 초기 판매자 확보의 핵심 진입 도구. 마이그레이션 없으면 전환 장벽 극복 불가
    jp-payment-native.md (Phase 1) P0 일본 런칭 시 신용카드·콘비니·PayPay 지원 없으면 매출 발생 불가
    unit-localization.md P0 일본 출시 시 엔화·소비세·도량형 현지화 없으면 상품 표기 오류
    operator-admin.md (기본 기능) P0 플랫폼 운영팀의 최소 관리 도구 (셀러 상태 관리, PG 심사 모니터링)
    module-definition.md (M-01~M-05) P0 스토어 기본 설정·상품·주문·결제·배송 기본 모듈 없으면 이커머스 불가
    unified-payment-dashboard.md P1 통합 정산 뷰는 일본 런칭 직후 판매자 신뢰 확보에 필수
    first-sale-engine.md P1 첫 매출 없으면 판매자 조기 이탈. 런칭 직후 가장 중요한 활성화 도구
    analytics-spec.md (기본 통계) P1 매출·주문 기본 통계 없으면 판매자 운영 불가. 고급 통계는 P2
    sns-marketing.md (Meta+TikTok) P1 D2C 판매자 주력 채널 연동. 런칭 직후 마케팅 활성화 필요
    sns-review-collection.md P1 UGC 리뷰 수집은 초기 사회적 증거(Social Proof) 확보에 중요
    affiliate-tracking.md P1 인플루언서 협찬 추적 없으면 첫 매출 경로 측정 불가
    global-launch.md (텍스트 번역) P1 한→일 텍스트 번역 없으면 일본 진출 판매자 상품 등록 어려움
    country-preset.md P1 국가별 설정 프리셋 없으면 판매자가 일본 설정을 수동으로 해야 함
    influencer-matching.md P2 인플루언서 DB 매칭 플랫폼. 초기는 수동 연결로 대체 가능
    benchmark-report.md P2 벤치마크 리포트는 락인 강화 도구. 런칭 직후보다 성숙기에 필요
    ai-assistant.md P2 AI 운영 어시스턴트. 기본 기능 런칭 후 고도화
    ai-chat-setting.md P2 AI 채팅 설정. 판매자 수 증가 후 운영 효율화 목적
    global-dashboard.md P2 글로벌 통합 대시보드. 다국가 진출 이후 필요
    sellex-integration.md P2 셀익스 연동. 셀익스 사업 진행 상황에 연동
    shipping-integration.md P2 3PL 연동·콘비니 픽업은 Phase 2. 기본 배송 추적은 P0~P1
    global-launch.md (이미지 번역·다국어 확장) P2 이미지 번역은 기술 구현 난이도 높음. 텍스트 번역 이후 진행
    jp-payment-native.md (Phase 2~3) P3 은행 자동이체·착불 결제 등 심층 지원. 일본 시장 성숙 후 필요
    migration.md (아임웹·식스샵·Shopify) P3 카페24·메이크샵 이후. 해당 플랫폼 이탈 니즈 확인 후 진행
    analytics-spec.md (MMP·ShopifyQL 수준) P3 고급 분석 기능. 활성 판매자 100명 이후 우선순위

    Phase별 개발 목표 요약¶

    P0 목표 (2026년 6월 런칭)¶

    웹프론트엔드 - 셀러 스토어 생성 UX (seller-store-creation-ux.md) - 서브도메인·커스텀 도메인 연결 (domain-architecture.md) - 한국·일본 기본 스토어 프론트엔드

    백엔드 - 스토어 기본 모듈 (상품·주문·결제·배송 기본) - 도메인 라우팅·멀티테넌시 DB 격리 - PG 연동 (이니시스, 토스페이먼츠, GMO-PG Phase 1) - 마이그레이션 엔진 (카페24·메이크샵)

    디자인 - 셀러 어드민 UI 시스템 - 스토어 기본 테마 2~3개

    P1 목표 (2026년 9월)¶

    • 통합 정산 대시보드
    • 첫 매출 견인 도구 (웰컴 쿠폰, SNS 공유 링크)
    • 기본 통계 대시보드
    • Meta·TikTok 카탈로그 연동
    • 어필리에이트 링크 트래킹

    주의 사항¶

    • P0 스펙 범위 내에서도 MVP 범위를 최소화해야 6월 런칭 가능. 각 스펙의 Phase 1 기능만 우선 구현
    • 운영자 어드민(operator-admin.md)은 P0이지만 기본 기능(셀러 조회·상태 변경·PG 심사 모니터링)만 런칭 포함
    • Flutter 앱은 P0에서 웹뷰 기반 MVP로 시작 가능. 네이티브 앱 기능은 P1~P2
