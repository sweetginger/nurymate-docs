    1. 개요¶

    1.1 배경¶

    셀메이트CM은 셀러의 스토어 개설 이후 매출 성장까지 책임지는 플랫폼을 목표로 한다. 스토어를 만드는 것과 실제로 잘 파는 것은 다른 문제이며, 전환율 향상 기능은 셀러가 플랫폼에 잔류하는 핵심 이유가 된다.

    구매 전환율 2025년 이커머스 글로벌 평균은 2.5~3% 수준이며, AI·자동화 기반 개인화 기능을 도입한 브랜드는 업계 평균 대비 25~30% 높은 전환율을 달성한다는 데이터가 있다. 번들/크로스셀만으로도 이커머스 전체 매출의 10~30%가 발생한다.

    1.2 목표¶

    • 셀러가 별도 써드파티 앱 없이 기본 탑재 기능만으로 전환율을 높일 수 있는 환경 제공
    • 카페24·Shopify 대비 더 낮은 셋업 복잡도로 동급 이상의 기능 제공
    • GMV 증가 → 플랫폼 수수료 수익 비례 증가라는 셀메이트CM의 수익 구조와 이해관계 일치

    1.3 범위¶

    본 기획서는 아래 세 범주를 다룬다.

    범주 설명
    머천다이징 상품·배너 예약 게시, 진열 순서 수동·자동 관리
    개인화 추천 구매·조회 이력 기반 추천, 함께 본 상품 블록
    전환율 기타 희소성 UI, 타임세일, 소셜 프루프, 위시리스트, 번들 할인

    2. 머천다이징 기능¶

    2.1 상품·배너 예약 게시¶

    2.1.1 문제 정의¶

    셀러는 시즌 프로모션(블랙프라이데이, 설날 기획전 등)을 준비할 때 지정 시각에 맞춰 상품과 배너를 동시에 노출해야 한다. 현재 수동으로 게시 시각에 맞춰 관리자 어드민에 접속해 수동 게시하는 방식은 다음 문제를 야기한다.

    • 야간·새벽 프로모션 오픈 시 운영자가 직접 대기해야 함
    • 실수로 게시가 누락되거나 예상 시각보다 늦어짐
    • 상품 게시와 배너 변경 타이밍이 엇갈려 UX 불일치 발생

    2.1.2 기능 정의¶

    상품 예약 게시

    항목 내용
    게시 예약 날짜·시각(분 단위) 지정 후 자동 노출
    게시 해제 예약 프로모션 종료 시각에 자동 비공개 전환
    배치 예약 다수 상품 일괄 예약 (CSV 업로드 또는 목록 선택)
    상태 표시 관리자 어드민 상품 목록에 "예약" 뱃지 표시
    예약 목록 예정된 예약 건 한눈에 관리하는 예약 캘린더 뷰

    배너 예약 게시

    항목 내용
    배너 예약 메인 슬라이더·팝업·공지 배너 노출 시각 예약
    다중 배너 스케줄 여러 배너를 기간별로 순서 지정 (기간 겹치면 우선순위 번호로 결정)
    롤백 설정 예약 기간 종료 후 이전 배너로 자동 복원

    2.1.3 UX 플로우¶

    어드민 > 상품 > 상품 등록/수정
  └─ "공개 설정" 섹션
       ├─ [즉시 공개]
       └─ [예약 공개]
            ├─ 공개 일시: 2026-04-01 00:00
            └─ 자동 비공개 일시: 2026-04-03 23:59 (선택)

    2.1.4 기술 요건¶

    • cron job 또는 이벤트 스케줄러로 예약 실행 (정확도 ±1분 이내)
    • 대용량 배치 예약 시 큐 처리 (동시 노출 수천 건도 안정 처리)
    • 타임존 처리: 셀러의 스토어 타임존 기준으로 예약 (한국 KST, 일본 JST 등)

    2.2 상품 진열 관리¶

    2.2.1 진열 방식 개요¶

    셀메이트CM은 수동 진열과 자동 진열을 혼합 운용할 수 있도록 설계한다. 셀러가 특정 상품을 고정 상위 노출하면서 나머지는 알고리즘에 맡기는 "핀 고정 + 자동 진열" 혼합 방식이 실무에서 가장 많이 사용된다.

    2.2.2 수동 진열¶

    항목 내용
    드래그&드롭 어드민 상품 목록에서 드래그로 순서 변경
    핀 고정 특정 상품을 1~N번 위치에 고정 (자동 진열이 덮어쓰지 않음)
    분류별 진열 카테고리별 독립적인 진열 순서 설정
    메인 진열 메인 홈의 추천 섹션 상품 수동 지정

    2.2.3 자동 진열 알고리즘¶

    자동 진열은 최소 4가지 기준 중 셀러가 선택하여 적용한다. 기준은 진열 영역(전체 스토어 / 특정 카테고리 / 메인 홈)별로 달리 설정 가능하다.

    알고리즘 기준 지표 정의 갱신 주기
    판매량순 최근 N일 기준 판매 수량 누적 1시간
    조회수순 최근 N일 기준 상품 상세 페이지 조회 수 30분
    전환율순 상품 조회수 대비 주문 전환율 (주문수 ÷ 조회수) 1시간
    최신순 상품 등록일 기준 (최신 상품 우선 노출) 즉시
    클릭가치순 클릭 1회당 기대 주문금액 (조회수 × 평균 주문 단가) 1시간
    복합 스코어 판매량·전환율·최신성을 가중 합산한 종합 점수 1시간

    카페24 "세라(CERA)" 빅데이터 자동 진열 참고: 주문수·주문율·조회수·클릭가치 4가지 지표를 분석해 24시간 자동 갱신하는 방식. 셀메이트CM은 이에 전환율 기준과 복합 스코어를 추가해 차별화한다.

    자동 진열 설정 UX

    어드민 > 진열 관리 > 자동 진열
  ├─ 적용 범위: [전체 스토어 / 특정 카테고리 선택]
  ├─ 기준: [판매량 / 조회수 / 전환율 / 최신순 / 클릭가치 / 복합 스코어]
  ├─ 기간: 최근 [7 / 14 / 30 / 90]일 기준
  ├─ 핀 고정 상품: [목록 선택] → 고정 위치 자동 제외
  └─ [적용] → 즉시 반영, 이후 설정 주기로 자동 갱신

    2.2.4 진열 예약¶

    날짜·시각 기반으로 진열 기준을 자동 전환할 수 있다. 예: 평일에는 전환율순, 주말에는 최신순으로 자동 교체.

    3. 개인화 추천 기능¶

    3.1 배경 및 효과¶

    개인화 추천 엔진을 도입한 스토어는 전환율이 도입 전 대비 4~5배 높아진다는 업계 연구 결과가 있다. Shopify 자체 데이터에서도 추천 엔진이 전체 매출의 최대 31%를 견인하는 것으로 나타났다. 현재 Shopify는 개인화 추천을 "Sidekick Pulse"를 통해 네이티브화하고 있으며, 카페24는 앱스토어 연동에 의존하고 있다.

    셀메이트CM은 기본 내장 추천 블록 형태로 제공하여 앱 설치 없이 즉시 사용 가능하게 한다.

    3.2 추천 블록 유형¶

    블록명 추천 로직 노출 위치
    이 상품을 본 사람이 함께 본 상품 동일 상품 조회 세션의 co-view 상품 추출 상품 상세 페이지 하단
    최근 본 상품 현재 구매자의 세션/로그인 기반 최근 조회 이력 상품 상세·장바구니
    이 상품과 자주 함께 구매하는 상품 동일 주문에 포함된 상품 쌍 분석 (장바구니 분석) 상품 상세·장바구니
    당신을 위한 추천 구매·조회 이력 기반 협업 필터링 (로그인 유저 한정) 메인 홈·카테고리 상단
    인기 상품 최근 7일 구매량·조회수 기반 (신규 방문자 폴백) 메인 홈·카테고리

    3.3 데이터 수집 범위¶

    추천 엔진이 동작하려면 아래 행동 데이터를 수집해야 한다.

    이벤트 수집 내용
    상품 조회 상품 ID, 세션 ID, 유저 ID (로그인 시), 타임스탬프
    장바구니 담기 상품 ID, 수량, 세션 ID
    주문 완료 주문 내 상품 목록 (같은 주문에 담긴 상품 쌍 추출)
    검색어 검색 키워드 + 클릭한 상품 ID

    3.4 콜드 스타트 처리¶

    신규 스토어(데이터 없는 초기 상태)에서의 폴백 전략:

    • 신규 스토어 오픈 0~30일: "인기 상품" 블록(전체 플랫폼 트렌드 기반)으로 대체
    • 조회 이력 없는 방문자: 스토어 전체 판매량순 추천으로 폴백
    • 로그인 유저: 30일 이내 조회·구매 이력이 있으면 개인화 추천 활성화

    3.5 추천 블록 설치 UX (셀러 관점)¶

    셀러는 코딩 없이 AI 프론트빌더에서 드래그&드롭으로 추천 블록을 페이지에 삽입한다.

    AI 프론트빌더 > 섹션 추가
  └─ [상품 추천] 블록 선택
       ├─ 추천 유형: [함께 본 상품 / 함께 구매 / 당신을 위한 추천 / 인기 상품]
       ├─ 노출 상품 수: [4 / 6 / 8]
       └─ 타이틀 텍스트 커스터마이징

    4. 전환율 향상 기타 기능¶

    기능 설명 우선순위
    희소성 UI — 재고 N개 재고 N개 미만일 때 "재고 N개 남음" 자동 표시. 임계값 셀러 설정 (기본 10개 이하) P1
    희소성 UI — 실시간 조회 N명 현재 해당 상품 페이지를 보고 있는 실시간 방문자 수 표시 P1
    타임세일 타이머 특정 가격이 유효한 종료 시각까지의 카운트다운 타이머 (시:분:초) P1
    소셜 프루프 — 최근 구매 알림 "방금 홍길동님이 이 상품을 구매했습니다" 팝업·바 형태 알림 P1
    위시리스트 구매자가 상품을 저장·관리. 로그인 유저는 영구 보관, 비로그인은 로컬 스토리지 임시 저장 P1
    장바구니 저장 세션 종료 후 재방문 시 장바구니 아이템 유지 (로그인 유저는 서버 저장) P0
    번들 할인 지정 상품 세트를 함께 구매 시 할인 적용. 셀러가 번들 구성·할인율 직접 설정 P1
    장바구니 업셀 / 크로스셀 장바구니 페이지에서 "이 상품도 함께 구매하세요" 추천 + 원클릭 추가 P2
    구매 후 업셀 결제 완료 직후 화면에 추가 구매 유도 오퍼 노출 P2

    4.1 희소성 UI 상세¶

    재고 N개 남음

    • 셀러 어드민에서 표시 임계값 설정 (기본: 재고 10개 이하)
    • 상품 상세 페이지 구매 버튼 주변에 "재고 8개 남음" 형태 텍스트 표시
    • 재고 0이 되면 자동으로 "품절" 상태로 전환

    실시간 조회 N명

    • 현재 동일 상품 페이지를 조회 중인 방문자 수를 WebSocket 또는 30초 폴링으로 표시
    • 최소 3명 이상일 때만 표시 (인위적 과장 방지 및 희소성 효과 유지)
    • 수치를 약간 라운딩 처리: 예, 실제 7명 → "7명 이상이 보고 있어요"

    4.2 타임세일 타이머 상세¶

    • 셀러가 상품별·카테고리별로 세일 기간(시작~종료 일시)과 할인가 설정
    • 상품 상세·목록 페이지에 카운트다운 타이머 자동 표시
    • 세일 종료 후 자동으로 정가 복원 (예약 게시 기능과 연동)
    • 셀러 어드민에서 타이머 디자인(색상·위치) 설정 가능

    4.3 소셜 프루프 알림 상세¶

    • 최근 N시간 이내 실제 구매 발생 시 좌하단 팝업으로 "n분 전 구매" 알림 표시
    • 구매자 개인정보 보호: 이름은 앞 2글자만 표시 ("홍길*"), 지역 표시만 허용
    • 셀러 설정: 표시 빈도(초), 알림 메시지 형식, 노출 페이지 범위

    4.4 위시리스트 상세¶

    • 상품 목록·상세 페이지에 하트 아이콘 버튼 노출
    • 비로그인: 브라우저 로컬 스토리지에 저장, 로그인 시 서버 위시리스트에 병합
    • 로그인 유저: 서버 저장, 어드민에서 "위시리스트에 담긴 수" 상품별 통계 확인 가능
    • 위시리스트 담은 상품 가격 하락 시 이메일·푸시 알림 발송 옵션 (Phase 2)

    4.5 번들 할인 상세¶

    • 셀러가 상품 A + 상품 B를 "번들" 구성 → 할인율(%) 또는 정액 할인 지정
    • 상품 상세 페이지에 "함께 구매하면 N% 할인" 섹션으로 표시
    • 번들 전용 랜딩 URL 자동 생성 (SNS 공유용)
    • 최대 번들 구성 상품 수: 5개

    • 앱당 월 $10~$100 수준의 추가 비용 발생 → 소규모 셀러에게 실질적 부담
    • 자동 진열 알고리즘이 기본 정렬 옵션(인기순·최신순·가격순) 수준에 머물러 있어 데이터 기반 최적화는 외부 앱 필요
    • Winter 2026 업데이트에서 A/B 테스트·네이티브 개인화(Sidekick Pulse) 등 내재화 방향으로 이동 중
    • 결론: Shopify의 앱 생태계 의존 구조는 셀러의 도구 파편화와 비용 증가를 야기함. 셀메이트CM이 내재화로 차별화할 수 있는 핵심 영역

    카페24

    • 상품 예약 게시·수동 진열·자동 진열은 네이티브로 잘 갖춰져 있음
    • "세라(CERA)" 자동 진열은 2021년부터 운영된 검증된 빅데이터 기반 기능
    • 타임세일·희소성 UI·소셜 프루프는 앱스토어 유료 앱에 의존
    • 개인화 AI 추천은 2025년 기준 ChatGPT 앱 연동 등 외부 의존 구조
    • 결론: 진열 관련 기능은 강하나, 전환율 향상 기타 기능은 앱 의존. 셀메이트CM은 카페24 진열 기능 수준을 유지하면서 나머지를 내재화하면 충분한 차별점 확보 가능

    아임웹

    • 타임세일 타이머·위시리스트·장바구니는 네이티브로 지원
    • AI 빌더(2025년 출시)로 스토어 제작 경험은 개선 중
    • 자동 진열 알고리즘·개인화 추천·소셜 프루프는 미지원 또는 앱 의존
    • 결론: UI/UX·디자인 친화적이나 데이터 기반 머천다이징·개인화 기능이 약함. 셀메이트CM의 자동 진열 + 개인화 추천은 아임웹 대비 명확한 우위

희소성·조회수  추천 엔진·
실시간 표시   자동 진열
    • 자동 진열·추천 엔진용 분석 DB는 analytics-spec.md의 데이터 수집 범위와 통합 설계
    • 실시간 조회 N명 표시용 카운터는 Redis TTL 기반으로 처리 (세션당 1카운트, 30분 만료)

    7.2 멀티 스토어 격리¶

    셀메이트CM은 멀티테넌시 구조이므로 각 스토어의 행동 데이터는 스토어 ID 기준으로 완전히 격리되어야 한다. 추천 엔진이 A 스토어의 데이터로 B 스토어 추천을 생성하는 구조는 불가. 단, 신규 스토어 콜드 스타트 시 플랫폼 전체 집계 트렌드는 참고 가능.

    7.3 개인정보 처리¶

    • 행동 이벤트 수집 시 구매자 동의 필요 (개인정보처리방침 고지)
    • 소셜 프루프 알림 구매자 이름 표시 시 개인정보보호법 준수 (이름 마스킹, 실명 미표시)
    • 실시간 조회 수 표시는 개인을 식별하지 않으므로 동의 불필요
    • 일본 스토어 적용 시 개인정보보호법(APPI) 준수 별도 검토 필요

    7.4 AI 프론트빌더 연동¶

    추천 블록은 헤드리스 API로 노출되며, AI 프론트빌더에서 드래그&드롭으로 삽입 가능한 표준 섹션 컴포넌트로 제공한다. 셀러가 커스텀 스킨을 사용하더라도 추천 블록 삽입이 가능하도록 설계한다.

    7.5 A/B 테스트 연계¶

    전환율 향상 기능 도입 효과를 측정하기 위해 analytics-spec.md의 A/B 테스트 인프라와 연계한다. 희소성 UI·소셜 프루프·타이머 노출 여부를 A/B 테스트로 효과 검증 후 디폴트 설정을 결정한다.
