    기존 몰 마이그레이션 기능 기획서¶

    • 문서 버전: v1.0
    • 작성일: 2026-03-06
    • 태스크: [10-3]

    1. 개요 및 목표¶

    1.1 배경¶

    셀메이트CM로 전환을 검토하는 판매자의 가장 큰 진입 장벽은 기존 데이터 이전 부담이다. 카페24, 메이크샵, 고도몰 등 기존 플랫폼에서 수년간 쌓인 상품 데이터, 주문 이력, 회원 DB를 잃는 것에 대한 두려움이 전환 결정을 막는 핵심 마찰 요인이다.

    마이그레이션 기능은 이 마찰을 제거하고 전환 비용을 최소화함으로써, 셀메이트CM의 신규 판매자 획득(Acquisition) 퍼널에서 핵심 전환 촉진 도구로 작동한다.

    1.2 목표¶

    목표 정의
    데이터 손실 제로 판매자가 정의한 마이그레이션 대상 데이터의 손실률 0%
    전환 마찰 최소화 마이그레이션 시작 → 첫 테스트 주문 완료까지 소요 시간 72시간 이내 (평균)
    판매자 자립 실행 개발자 없이 판매자 혼자 마이그레이션 완료 가능 (마법사 UX 기반)
    병행 운영 지원 기존 몰 운영을 중단하지 않고 셀메이트CM 준비 가능

    1.3 성공 지표 (KPI)¶

    • 마이그레이션 완료율 ≥ 85% (시작한 판매자 대비)
    • 마이그레이션 후 30일 내 첫 실매출 발생율 ≥ 60%
    • 마이그레이션 관련 CS 문의 비율 ≤ 15% (완료 건 대비)
    • 데이터 정합성 검증 통과율 ≥ 99%

    2. 마이그레이션 대상 데이터 정의¶

    2.1 데이터 유형별 우선순위¶

    데이터 유형 필수/선택 우선순위 비고
    상품 (이름, 설명, 가격, 카테고리) 필수 P0 핵심 영업 자산
    상품 이미지 필수 P0 S3 직접 복사
    상품 옵션 (색상, 사이즈 등) 필수 P0 옵션 구조 변환 필요
    재고 수량 필수 P0 실시간 정확도 중요
    배송비 정책 (기본·조건부·무료) 필수 P1 정책 유형별 매핑
    고객 회원 데이터 (이메일·이름·주소) 선택 P1 개인정보 동의 처리 필수
    주문 이력 (완료 주문) 선택 P1 운영 참고용, 신규 주문과 혼용 금지
    쿠폰 정책 선택 P2 구조 차이로 수동 재입력 권장
    적립금 잔액 선택 P2 판매자 선택 이전 여부 결정
    상품 리뷰 선택 P2 플랫폼 간 구조 차이 큼
    카테고리 트리 필수 P1 상품과 연계 이전 필요

    2.2 마이그레이션 제외 대상¶

    아래 데이터는 초기 버전(Phase 1)에서 마이그레이션을 지원하지 않는다. 판매자에게 명확히 사전 안내한다.

    • 비밀번호 해시 (플랫폼별 암호화 방식 상이 → 재설정 안내)
    • 기존 몰의 앱·플러그인 설정 데이터
    • 결제 수단별 저장된 카드 정보 (PCI DSS 준수, 이전 불가)
    • 세금계산서·현금영수증 발행 이력 (법적 원본 보관 원칙)

    3. 지원 소스 플랫폼별 데이터 추출 방식¶

    3.1 카페24 (1순위)¶

    카페24는 Admin REST API (OAuth 2.0) 와 엑셀 대용량 다운로드 두 가지 방식을 모두 지원한다.

    API 방식 (권장)

    • 엔드포인트: https://developers.cafe24.com/docs/api/admin/
    • 인증: OAuth 2.0 Access Token (판매자가 카페24 개발자센터에서 앱 등록 후 발급)
    • 주요 리소스: products, variants, orders, customers, categories, shippingpolicies
    • 주의사항: 주문 데이터는 최근 6개월분만 API로 추출 가능. 6개월 이전 주문은 별도 엑셀 다운로드 병행 필요.
    • API Rate Limit: 초당 2회 (Burst 최대 10회) → 대용량 처리 시 큐잉 필수

    엑셀 다운로드 방식 (보조)

    • 카페24 관리자 → 엑셀 파일 요청 메뉴 이용
    • 지원 데이터: 주문, 상품, 품목, 제조사, 회원, 적립금
    • 생성 한도: 일반 파일 즉시 생성, 대용량 파일 1일 5회 제한 (익일 생성)
    • 셀메이트CM 포맷으로 자동 변환하는 매핑 엔진 제공

    카페24 전용 처리 사항

    • xans- 기반 상품 상세 HTML → 셀메이트CM 에디터 포맷으로 변환
    • 카페24 배송비 정책 유형 (기본 / 지역별 / 무게별 / 조건부 무료) → 셀메이트CM 정책 스키마 매핑
    • 카페24 회원 등급 → 셀메이트CM 회원 그룹으로 매핑

    3.2 메이크샵 (2순위)¶

    메이크샵은 OpenAPI를 제공한다 (https://openapi.makeshop.co.kr/). CSV/XML 내보내기 방식도 병행 지원.

    API 방식 (OpenAPI)

    • 엔드포인트: https://openapi.makeshop.co.kr/
    • 인증: 메이크샵 개발자센터에서 API KEY 발급
    • 주요 리소스: 상품, 주문, 회원, 재고
    • 셀메이트CM은 메이크샵 OpenAPI를 우선 활용하고, API 미지원 데이터는 CSV/XML 엑셀 방식으로 보완

    추출 경로

    데이터 메이크샵 추출 방법
    상품 OpenAPI 우선 / 상품관리 → 전체상품 통합관리 → 엑셀양식관리 → XML/CSV 다운로드
    주문 OpenAPI 우선 / 주문관리 → 통합주문리스트 → 양식 생성 후 엑셀 다운로드
    회원 OpenAPI 우선 / 회원관리 → 회원/CRM 관리하기 → 엑셀양식관리 → 다운로드

    주의사항

    • 통합옵션 사용 상점: XML 형식 출력 → 셀메이트CM XML 파서 제공
    • 통합옵션 미사용 상점: CSV 형식 출력
    • 상품 이미지는 URL 형식으로 포함 → 셀메이트CM이 직접 다운로드 후 S3 업로드

    셀메이트CM 제공 도구

    • 메이크샵 OpenAPI 연동 모듈 + XML/CSV → 셀메이트CM 임포트 포맷 자동 변환기
    • 업로드 UI: 드래그&드롭 파일 업로드 → 파싱 결과 미리보기 → 확인 후 일괄 등록

    3.3 고도몰 (3순위)¶

    고도몰은 NHN 커머스 운영으로 데이터 이전 서비스를 별도 제공한다.

    추출 경로

    데이터 고도몰 추출 방법
    상품 상품관리 → 엑셀 다운로드 (표준 포맷)
    주문 주문관리 → 주문 엑셀 다운로드
    회원 회원관리 → 회원 엑셀 다운로드 (암호화 필드 포함)

    주의사항

    • 고도몰 5 기준 엑셀 포맷 사용 (고도몰 4와 컬럼 구조 상이)
    • HS코드 필드 없음 → 해외배송 연동 시 판매자 직접 입력 필요
    • 고도 프로그램센터를 통한 데이터 이전 대행 서비스 존재 → 파트너십 검토

    3.4 Shopify (4순위)¶

    Shopify는 Admin REST API / GraphQL API 모두 지원하며, Matrixify 앱과 같은 서드파티 마이그레이션 도구 생태계가 활성화되어 있다.

    API 방식

    • Shopify Admin API: 상품, 주문, 고객, 컬렉션, 할인 코드 전체 지원
    • 인증: Private App (Access Token) 또는 OAuth
    • Bulk Operation API 지원으로 대용량 데이터 일괄 추출 가능

    참조 도구

    • Matrixify: Shopify → 타 플랫폼 마이그레이션 표준 도구. 엑셀 기반 변환 지원.
    • 셀메이트CM은 Shopify 엑셀 포맷 직접 임포트 기능 제공 (Matrixify 내보내기 파일 호환)

    주의사항

    • Shopify 상품 메타필드 → 셀메이트CM 추가 속성으로 변환 (손실 최소화)
    • 해외 몰 운영 판매자 비중이 높음 → 다국가 가격, 다통화 처리 고려

    3.5 CSV 직접 업로드 (5순위, Universal Fallback)¶

    위 플랫폼 외 솔루션(Cafe24 모바일, 임의 솔루션 등) 또는 자체 DB를 직접 보유한 판매자를 위한 범용 업로드 방식.

    제공 내용

    • 셀메이트CM 표준 CSV 템플릿 다운로드 (상품용, 주문용, 회원용 각각)
    • 컬럼 매핑 UI: 업로드 후 판매자가 소스 컬럼을 셀메이트CM 필드에 드래그로 매핑
    • 유효성 검사: 필수 필드 누락, 가격 포맷 오류, 이미지 URL 접근 불가 등 사전 감지

    3.6 아임웹 (Imweb)¶

    아임웹은 공개 API(Imweb OpenAPI)를 제공한다.

    API 방식 (권장)

    • 엔드포인트: https://docs.imweb.me/ (Imweb OpenAPI)
    • 인증: API Key 기반 (아임웹 개발자 콘솔에서 발급)
    • 주요 리소스: 상품, 주문, 회원, 카테고리
    • API Rate Limit: 확인 후 큐잉 처리

    주의사항

    • 아임웹 템플릿 기반 디자인은 마이그레이션 대상에서 제외 (셀메이트CM 테마 신규 선택)
    • 아임웹의 디지털 상품 카테고리는 셀메이트CM 상품 유형으로 매핑 필요

    3.7 식스샵 (Sixshop)¶

    식스샵은 CSV 내보내기 방식을 지원한다.

    추출 경로

    데이터 식스샵 추출 방법
    상품 상품관리 → 엑셀 다운로드
    주문 주문관리 → 엑셀 다운로드
    회원 고객관리 → 엑셀 다운로드

    주의사항

    • 식스샵의 상품 구조(단순형/옵션형)를 셀메이트CM 옵션 스키마로 변환
    • 이미지 URL 형식으로 포함 → 셀메이트CM이 직접 다운로드 후 S3 업로드

    4. 마이그레이션 프로세스 설계¶

    4.1 사전 분석 (Pre-flight Analysis)¶

    마이그레이션 시작 전, 판매자 데이터를 사전 분석하여 예상 소요 시간과 잠재 문제를 파악한다.

    분석 항목

    항목 분석 방법 출력
    상품 수 API 또는 파일 파싱 예상 처리 시간
    이미지 총 용량 URL 헤더 검사 (Content-Length) S3 전송 예상 시간
    옵션 복잡도 옵션 조합 수 집계 변환 오류 가능성 알림
    주문 이력 기간 최초/최종 주문일 6개월 초과 여부 경고
    회원 수 카운트 집계 개인정보 처리 동의 안내
    누락 필드 필수 필드 공백 감지 보완 요청 목록

    출력 리포트 예시

    [사전 분석 결과]
- 상품: 1,247개 (예상 처리 시간: 약 45분)
- 이미지: 3,891장 / 총 8.2 GB (예상 전송 시간: 약 2시간)
- 주문: 12,450건 (6개월 이전 데이터 4,200건 → 엑셀 업로드 필요)
- 회원: 5,832명 (개인정보 처리 동의서 작성 필요)

[주의 사항]
- 상품 중 옵션 조합 100개 이상인 상품 12개 → 수동 검토 권장
- 이미지 URL 응답 불가 23건 → 수동 재업로드 필요

    4.2 병행 운영 전략 (Parallel Run)¶

    판매자가 기존 몰 운영을 중단하지 않고 셀메이트CM를 준비할 수 있도록 설계한다.

    병행 운영 단계

    [기존 몰 운영 중]
    │
    ▼
[Phase A: 준비]  ← 1~3주
  - 셀메이트CM 계정 생성
  - 데이터 마이그레이션 실행 (상품·배송정책·디자인)
  - 테스트 결제 확인
  - 도메인 연결 준비 (서브도메인 사용: test.판매자도메인.com)

[Phase B: 병행 운영]  ← 1~2주
  - 기존 몰: 주문 계속 수령
  - 셀메이트CM: 소규모 트래픽 테스트 (SNS 링크, 지인 테스트)
  - 주문 재고 수동 동기화 (단방향)
  - 정합성 점검 완료

[Phase C: 전환]
  - 도메인 DNS 전환 (TTL 낮춰 사전 준비)
  - 기존 몰 수주 중단 (또는 주문 임시 차단)
  - 셀메이트CM 라이브 전환

[Phase D: 사후 모니터링]  ← 72시간
  - 오류 모니터링
  - 기존 몰 유지 (롤백용, 최소 2주)

    병행 운영 원칙

    • 재고 동기화 원칙: 병행 운영 중 기존 몰과 셀메이트CM은 각각 독립된 재고 수량을 관리한다. 기존 몰의 OMS/WMS에서 재고 수량이 관리되고 있다면, 판매자는 셀메이트CM에 별도로 재고 수량을 입력하고 수시로 수동 동기화한다. 셀메이트CM은 수시 동기화 개념으로 운영되며, 자동 실시간 연동은 Phase 2 이후 OMS 연동 기능 제공 시점에 지원한다.
    • 기존 몰과 셀메이트CM를 동시에 수주하면 이중 판매 위험 발생 → Phase B에서는 셀메이트CM를 테스트 전용으로 사용할 것을 권장하고 UI에서 명확히 안내한다.

    4.3 전환 시점 결정 기준 (Go/No-Go Checklist)¶

    판매자가 전환 전 반드시 확인해야 할 체크리스트를 UX에서 제공한다.

    체크 항목 확인 방법
    상품 전체 등록 완료 상품 수 일치 확인
    대표 상품 구매 테스트 성공 테스트 주문 완료 확인
    배송비 정책 동작 확인 조건별 시뮬레이션
    결제 수단 연동 완료 PG사 정상 응답 확인
    고객 회원 이전 완료 (선택) 샘플 로그인 테스트
    도메인 연결 완료 DNS 전파 확인
    기존 몰 주문 차단 또는 안내 페이지 설정 기존 몰 상태 확인

    전환 당일 권장 시간대: 화~목 새벽 2~4시 (트래픽 최저점, 글로벌 기준은 플랫폼 분석 기반으로 조정)

    4.4 데이터 정합성 검증¶

    마이그레이션 완료 후 자동으로 정합성 검증을 실행하고 결과를 판매자에게 리포트한다.

    검증 항목

    검증 유형 방법 기준
    상품 수 일치 소스 카운트 vs 셀메이트CM 카운트 100% 일치
    이미지 접근 가능 S3 URL 헤드 요청 200 응답률 100%
    가격 정확성 소스 가격 vs 이전 가격 샘플 1% 비교 오차 0%
    옵션 조합 수 소스 SKU 수 vs 셀메이트CM SKU 수 100% 일치
    배송비 계산 주요 배송비 시나리오 3개 이상 테스트 일치
    회원 이메일 중복 이메일 기준 중복 감지 중복 처리 방침 안내

    5. UX 설계¶

    5.1 마이그레이션 마법사 (단계별 안내)¶

    판매자가 개발자 없이 마이그레이션을 완료할 수 있도록 6단계 마법사 UI를 제공한다.

    [Step 1] 소스 플랫폼 선택
  - 카페24 / 메이크샵 / 고도몰 / Shopify / 아임웹 / 식스샵 / CSV 직접 업로드
  - 선택 시 해당 플랫폼 추출 방법 가이드 자동 노출

[Step 2] 데이터 연결 또는 파일 업로드
  - (카페24/Shopify) API 인증 → OAuth 리다이렉트 또는 Access Token 입력
  - (메이크샵/고도몰/CSV) 파일 업로드 (드래그&드롭)

[Step 3] 사전 분석 결과 확인
  - 자동 분석 실행 후 리포트 표시 (상품 수, 이미지 용량, 주의 사항)
  - 판매자가 마이그레이션 범위 선택 (전체 / 상품만 / 주문 포함 등)

[Step 4] 이전 대상 및 옵션 설정
  - 회원 데이터 이전 여부 선택 (개인정보 처리 동의서 첨부 필수)
  - 주문 이력 이전 여부 선택
  - 적립금·쿠폰 처리 방침 선택

[Step 5] 마이그레이션 실행 및 진행 상태 확인
  - 실시간 진행바 (상품 N/M, 이미지 N/M, 회원 N/M)
  - 예상 완료 시간 표시
  - 백그라운드 처리 → 브라우저 닫아도 계속 진행

[Step 6] 검증 리포트 확인 및 완료
  - 정합성 검증 결과 표시
  - 오류 항목 목록 및 수정 방법 안내
  - 다음 단계 안내 (테스트 주문 / 도메인 연결 / 전환 체크리스트)

    5.2 오류 처리 및 사용자 안내¶

    오류 유형 사용자 안내 메시지 처리 방식
    이미지 URL 접근 불가 "23개 이미지를 가져올 수 없습니다. 해당 상품 목록을 다운로드하여 직접 업로드해 주세요." 오류 목록 CSV 제공
    API 인증 실패 "카페24 API 키가 만료되었거나 권한이 부족합니다. [가이드 보기]" 재인증 링크 제공
    옵션 변환 오류 "옵션 조합이 1,000개를 초과하는 상품이 있습니다. 해당 상품은 수동으로 등록해 주세요." 대상 상품 목록 제공
    중복 상품 감지 "이미 동일한 상품명·SKU가 존재합니다. 덮어쓰기 또는 건너뛰기를 선택해 주세요." 판매자 선택
    파일 포맷 오류 "파일 형식이 맞지 않습니다. 셀메이트CM CSV 템플릿을 사용해 주세요." 템플릿 다운로드 링크

    오류 처리 원칙

    • 단일 오류가 전체 마이그레이션을 중단시키지 않는다 (부분 성공 허용).
    • 오류 항목은 별도 큐에 보관 → 수동 보완 후 재시도 가능.
    • 모든 오류 이력은 마이그레이션 로그에서 조회 가능.

    6. 기술 요건¶

    6.1 데이터 변환 규칙 (소스 → 셀메이트CM 스키마)¶

    상품 스키마 매핑 (주요 필드)

    소스 필드 (카페24 예시) 셀메이트CM 필드 변환 규칙
    product_name title 직접 매핑
    summary_description summary 직접 매핑
    description body_html HTML 정제 (스크립트 제거)
    selling_price price Integer (원화)
    stock_quantity inventory.quantity Integer
    category_no category_id 카테고리 매핑 테이블 참조
    variants[] options[] + variants[] 옵션 구조 분리·재조합
    list_image images[0] (대표 이미지) S3 재업로드
    detail_image[] images[1..] (상세 이미지) S3 재업로드

    주문 스키마 매핑 (주요 필드)

    소스 필드 셀메이트CM 필드 변환 규칙
    order_id external_id 참조용 보존, 셀메이트CM 신규 ID 발급
    order_status status 플랫폼별 상태 코드 → 셀메이트CM 표준 상태
    buyer_name shipping.name 직접 매핑
    payment_method payment.method_label 텍스트 변환 (이전 이력 참조용)

    배송비 정책 매핑

    카페24 배송비 유형 셀메이트CM 정책 유형
    기본배송비 flat_rate
    조건부 무료 (금액 기준) free_over_amount
    무게별 차등 weight_based
    지역별 추가배송비 region_surcharge

    │
    ▼
[3] 포맷 변환 (WebP 변환 — 선택, 판매자 설정)
    │
    ▼
[4] S3 업로드 (셀메이트CM S3 버킷)
    │  키: /merchants/{merchant_id}/products/{product_id}/{filename}
    ▼
[5] CDN URL 생성 → 상품 레코드에 저장
    │
    ▼
[6] 원본 URL 참조 보존 (external_image_url 필드 — 롤백 대비)

    처리 규격

    • 병렬 다운로드: 동시 8개 스트림 (네트워크 부하 고려, 설정 가능)
    • 단일 이미지 최대 크기: 50MB (초과 시 스킵 + 오류 목록 등록)
    • 지원 포맷: JPEG, PNG, WebP, GIF (GIF는 첫 프레임 스틸로 변환 — Phase 1)

    6.3 대용량 처리 (비동기 백그라운드 작업)¶

    아키텍처

    [마이그레이션 요청]
    │
    ▼
[API Gateway] → 작업 ID 즉시 반환
    │
    ▼
[Task Queue (Celery / Redis)]
    │
    ├── [Worker A] 상품 데이터 변환 및 DB 삽입
    ├── [Worker B] 이미지 다운로드 및 S3 업로드
    └── [Worker C] 회원·주문 데이터 처리
    │
    ▼
[진행 상태 업데이트] → WebSocket 또는 SSE로 실시간 전달
    │
    ▼
[완료 알림] → 이메일 + 어드민 대시보드 알림

    처리 규모 기준

    규모 상품 수 예상 처리 시간
    소형 ~500개 ~30분
    중형 ~5,000개 ~3시간
    대형 ~50,000개 ~1일 (야간 처리 권장)

    신뢰성 보장

    • 작업 단위 체크포인트 저장 → 중단 시 재개 가능
    • 개별 레코드 오류는 큐에서 분리, 전체 작업 중단 없음
    • 완료 후 30일간 마이그레이션 로그 보존

    6. 법률 검토 사항¶

    6.1 회원 개인정보 이전 시 동의 수집 시점 검증¶

    기존 플랫폼(카페24 등)에서 수집한 회원 개인정보를 셀메이트CM로 이전할 때의 법적 요건:

    한국 (개인정보보호법)

    • 개인정보를 제3자(셀메이트CM)에게 이전하려면 정보 주체의 별도 동의 필요 (개인정보보호법 제17조)
    • 기존 플랫폼의 이용약관·개인정보처리방침에 "서비스 이전 시 제3자 제공" 조항이 명시된 경우에도 별도 동의가 원칙
    • 권장 방식: 기존 몰에서 이전 전 고객에게 동의 이메일 발송 → 동의자만 이전

    이전 후 본인인증 시 동의 수집 가능 여부

    • 이전 후 셀메이트CM 첫 로그인 시 동의를 받는 방식도 가능하나, 이전 자체를 위해서는 이미 데이터가 이전된 상태 → 법적 시간 순서 문제 발생 가능
    • 법률 자문 필요: 이 방식의 적법성은 명확하지 않으므로, 실제 서비스 시작 전 법률 전문가 검토 권장

    ⚠️ 미결 법률 검토 항목: "이전 후 본인인증 시 동의 수집" 방식의 개인정보보호법상 적법성 → 런칭 전 법무 검토 필수

    일본 (個人情報保護法 - APPI)

    • 일본 APPI 제23조: 제3자 제공 시 본인 동의 필요 (Opt-in 방식)
    • 예외: 공동이용(共同利用) 조항 — 사전에 공동이용 목적, 항목, 관리책임자를 공개한 경우 별도 동의 불필요
    • 셀메이트CM의 경우 기존 플랫폼과 공동이용 관계가 아니므로 별도 동의 필요

    권장 프로세스

    1. 판매자가 마이그레이션 신청 시 "회원 데이터 이전 동의 요청 이메일 발송" 옵션 선택
    2. 셀메이트CM이 기존 몰 회원에게 동의 요청 이메일 자동 발송
    3. 동의한 회원만 셀메이트CM로 이전
    4. 미동의 회원은 셀메이트CM에서 신규 가입으로 유도
