# 입점형 마켓플레이스 기능 기획

> 작성일: 2026-03-19 | 버전: v0.2
> 우선순위: P2 (2026 H2 선택 활성화)
> 태스크: 15-4, 18-6

---

## 요약 (결론 먼저)

**누리메이트는 StoreX 사업의 피봇(Pivot) 플랫폼으로 포지셔닝된다.**

- 누리메이트의 자사몰(독립몰) 기능이 기본이되, 입점형 마켓플레이스 운영 기능을 **옵셔널 모듈로 제공**한다.
- 셀러가 원할 경우 자신의 스토어를 "멀티브랜드 마켓플레이스"로 확장할 수 있다.
- StoreX의 핵심 가치(입점 브랜드 관리·정산 자동화)를 누리메이트 아키텍처 위에서 더 가볍게 구현한다.

---

## 1. 전략적 배경

### 1.1 누리메이트 ↔ StoreX 관계

| 항목 | StoreX (현재) | 누리메이트 (차세대) |
|------|-------------|-----------------|
| 포지션 | 입점형 마켓플레이스 구축 SaaS | 자사몰 + 선택적 입점형 |
| 타겟 | 월 매출 10억+ 대기업·중견기업 | 스몰브랜드~중형 브랜드 |
| 진입 비용 | 도입비 300만 + 월 100만 이상 | 슬라이딩 수수료 0~2% |
| 글로벌 지원 | 일본·글로벌 PG 연동 가능 | 한국·일본 네이티브, 동남아 확장 |
| AI 기능 | 미지원 | AI 온보딩·스토어빌더·채팅 내장 |

**핵심 논리**: StoreX 기존 고객(대기업 입점형 마켓 운영사)은 계속 StoreX로 서비스하되, 신규 시장(스몰~중형 브랜드가 자체 마켓플레이스를 만들고 싶은 수요)은 누리메이트의 옵셔널 입점형 모듈로 흡수한다.

### 1.2 입점형 수요가 존재하는 이유

- "지그재그·에이블리처럼 자체 앱 마켓플레이스를 만들고 싶은" 소규모 패션 셀러
- 브랜드 컨설턴트가 여러 신진 브랜드를 한 플랫폼에 큐레이션해서 운영하는 구조
- 카페24에서 "멀티샵" 형태로 여러 브랜드를 운영 중인 기존 판매자
- 오프라인 편집샵이 온라인 편집샵으로 전환하는 수요

## 2. 기능 정의: M-17 입점형 마켓플레이스 모듈

### 2.1 모듈 개요

| 항목 | 내용 |
|------|------|
| 모듈 코드 | **M-17** |
| 분류 | 선택 활성화 (Optional) — 기본 OFF |
| 과금 | Premium (별도 수수료 또는 상위 플랜) |
| 의존 모듈 | M-01, M-02, M-03, M-04, M-05, M-06 |
| 개발 우선순위 | **P2 (2026 H2)** |

### 2.2 핵심 기능 목록

| # | 기능 | 설명 | Phase |
|---|------|------|-------|
| 1 | **마켓플레이스 모드 활성화** | 셀러 어드민에서 "입점형 마켓 활성화" 토글 ON → 슈퍼관리자 기능 전환 | 2 |
| 2 | **브랜드 초대·입점 관리** | 이메일 초대 → 입점 신청 → 운영자 승인 워크플로우 | 2 |
| 3 | **브랜드별 어드민 계정** | 입점 브랜드에 전용 어드민 계정 발급 (상품·주문·정산 제한 권한) | 2 |
| 4 | **상품 검수·승인** | 입점 브랜드 상품 등록 → 운영자 검수 → 노출 승인 | 2 |
| 5 | **공급률 기반 정산** | 브랜드별 공급률 설정 → 판매 후 정산서 자동 생성 | 2 |
| 6 | **마켓 전용 디자인 레이어** | 입점 브랜드별 스토어 페이지 + 통합 마켓 홈 구성 | 3 |
| 7 | **마켓 전용 프로모션** | 입점 브랜드 전체 또는 선택 브랜드 대상 기획전 설정 | 3 |
| 8 | **입점 브랜드 통계** | 브랜드별 매출·주문·전환율 분리 통계 | 3 |
| 9 | **브랜드 수수료 자동 차감** | 정산 시 플랫폼 수수료 + 브랜드 수수료 자동 분리 처리 | 3 |
| 10 | **글로벌 마켓플레이스 지원** | 입점형 마켓에서도 일본·동남아 결제·언어 적용 | 3 |

## 3. 포지셔닝: 자사몰 + 마켓플레이스 이중 모드

```
누리메이트 셀러 (자사몰 모드 — 기본)
    ├── 자신의 브랜드 상품 판매 (기존 방식 유지)
    └── [옵셔널] 마켓플레이스 모드 활성화
            ├── 내 스토어 = 독립몰 운영 (계속 유지)
            ├── 마켓 홈 페이지 = 큐레이션된 멀티브랜드 마켓
            └── 입점 브랜드 B, C, D 상품 함께 판매
                    └── 브랜드별 정산 자동 처리
```

### 3.1 단계별 운영 구조

| 단계 | 구조 | 설명 |
|------|------|------|
| **Step 1** | 자사몰만 운영 | 기본 상태. 입점형 모듈 OFF |
| **Step 2** | 마켓플레이스 모드 ON | 내 스토어 + 마켓 홈 동시 운영 |
| **Step 3** | 브랜드 입점 시작 | 1~2개 브랜드 초대·승인 |
| **Step 4** | 멀티브랜드 마켓 운영 | 5~20개 브랜드 체계적 관리 |

### 3.2 StoreX와의 차별화

| 항목 | StoreX | 누리메이트 M-17 |
|------|--------|---------------|
| 진입 방식 | 별도 솔루션 계약 (도입비 300만) | 기존 누리메이트 어드민에서 토글 ON |
| 자사몰 병행 | 불가 (입점형 전용) | 가능 (자사몰 + 마켓 동시 운영) |
| AI 기능 | 미지원 | M-09 AI 채팅·M-10 운영 어시스턴트 연동 |
| 글로벌 | 연동 가능 | 네이티브 한국·일본 결제 그대로 사용 |
| 대상 | 대기업·중견기업 | 스몰~중형 브랜드 큐레이터 |

## 4. 셀러 어드민 UX 변화 범위

### 4.1 마켓플레이스 모드 활성화 전 (기본)

- 기존 어드민과 동일. 변화 없음.

### 4.2 마켓플레이스 모드 활성화 후

| 영역 | 추가되는 UI |
|------|-----------|
| 사이드바 | "마켓 관리" 섹션 추가 (브랜드 관리 / 입점 신청 / 정산 관리) |
| 스토어 설정 | "마켓 홈 페이지" 설정 (통합 홈 디자인 별도 구성 가능) |
| 상품 관리 | 브랜드 필터 추가 (내 상품 / 브랜드 A / 브랜드 B 분류) |
| 주문 관리 | 주문별 브랜드 출처 표시, 브랜드별 주문 필터 |
| 정산 관리 | 브랜드별 정산 현황 + 정산서 발행 |
| 통계 | 브랜드별 매출·전환율 비교 뷰 |

**중요**: 자사몰 운영 흐름은 변경 없음. 마켓 관련 UI는 별도 섹션으로 분리 추가.

### 4.3 입점 브랜드 어드민 (제한 권한)

입점 브랜드는 별도 어드민 로그인으로 자신의 브랜드 영역만 접근:
- 상품 등록·수정 (검수 대기 상태로 저장)
- 내 브랜드 주문 조회·처리
- 내 브랜드 정산 내역 조회
- 운영자 승인 전 노출 불가

## 5. 수수료 구조

### 5.1 운영자(슈퍼관리자) 수수료 레이어

```
구매자
  → 마켓플레이스 결제 처리 (누리메이트 PG)
    → 누리메이트 플랫폼 수수료 (자사몰 슬라이딩 0~2% 동일 적용, 운영자 GMV 기준)
    → 운영자(슈퍼관리자) 정산
      → 입점 브랜드 정산 (공급률 기반 자동 차감)
```

### 5.2 수수료 설정 구조

| 항목 | 설정 주체 | 기준 |
|------|---------|------|
| 누리메이트 플랫폼 수수료 | 누리메이트 | 운영자 총 GMV 기준, 슬라이딩 0~2% |
| 브랜드 수수료(공급률) | 운영자(슈퍼관리자) | 브랜드별 개별 설정 (예: 70% 공급, 30% 마진) |
| PG 결제 수수료 | PG사 | 거래별 정률 (기존과 동일) |

### 5.3 판매자 수용성

- 운영자 입장: 입점 브랜드 수수료 수익 - 누리메이트 플랫폼 수수료 = 순수익
- 입점 브랜드 입장: 자사몰 별도 구축 비용 0원, 운영자 플랫폼에 상품만 공급
- 누리메이트 입장: 운영자 GMV 성장이 곧 수수료 수익 확대

## 6. 구현 우선순위

### 6.1 P2 (2026 H2) — Phase 2 핵심 기능

| 기능 | 구현 내용 |
|------|---------|
| 마켓플레이스 모드 토글 | 어드민 설정에서 활성화 |
| 브랜드 초대 · 계정 발급 | 이메일 초대 → 입점 브랜드 어드민 계정 생성 |
| 상품 검수 · 승인 | 입점 상품 운영자 검수 워크플로우 |
| 공급률 정산 자동화 | 브랜드별 공급률 설정 + 정산서 자동 발행 |

### 6.2 P3 (2027) — Phase 3 고도화

| 기능 | 구현 내용 |
|------|---------|
| 마켓 전용 디자인 레이어 | 브랜드별 미니샵 + 통합 마켓 홈 |
| 마켓 전용 프로모션 | 기획전·쿠폰 멀티브랜드 대상 설정 |
| 브랜드 통계 분리 | 입점 브랜드별 성과 대시보드 |
| 글로벌 마켓플레이스 | 일본·동남아 결제 연동된 마켓플레이스 |

### 6.3 우선순위 조정 근거

P2 채택 이유:
- 누리메이트가 StoreX 피봇 플랫폼이므로, 입점형 기능은 전략적으로 필수
- P0·P1 기능(자사몰 MVP) 완성 후 3~6개월 내 추가 가능한 범위
- 초기 StoreX 고객 유입 창구로 활용 가능 ("StoreX 계약 없이 누리메이트에서 바로 시작")
