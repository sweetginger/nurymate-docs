    1. 개요¶

    1.1 목적¶

    누리메이트 AI 운영 어시스턴트는 판매 데이터를 자동으로 분석하여 "지금 무엇을 해야 하는가"를 판매자에게 직접 전달한다. 데이터를 읽는 능력이 없어도, GA4를 몰라도, 매일 아침 어드민에 접속하면 오늘 가장 중요한 운영 액션을 알 수 있다.

    1.2 핵심 가치 제안¶

    "통계를 보는 시간을 줄이고, 판매자가 실제 운영에 집중할 수 있도록 한다."

    기존 방식 AI 어시스턴트 도입 후
    여러 통계 화면을 직접 탐색 어드민 홈에서 인사이트 3개 즉시 확인
    숫자를 보고 스스로 해석 AI가 원인과 맥락까지 설명
    개선 방향을 스스로 찾아야 함 구체적 실행 버튼(CTA) 제시
    문제를 놓치는 경우 발생 임계값 도달 시 즉시 알림

    1.3 analytics-spec.md와의 관계¶

    AI 어시스턴트는 analytics-spec.md에 정의된 수집 데이터를 소비하는 최상위 인터페이스 레이어다. 어시스턴트가 판단을 내리는 모든 근거는 analytics-spec.md에 수집되는 지표에서 나온다. AI 어시스턴트 단독으로는 새로운 데이터를 생성하지 않는다.

    [데이터 수집 레이어]   analytics-spec.md 정의 데이터
        ↓
[집계/분석 레이어]     배치 및 실시간 집계 엔진
        ↓
[AI 어시스턴트 레이어] 패턴 감지 → 자연어 인사이트 생성 → CTA 연결
        ↓
[판매자 인터페이스]    어드민 홈 카드, 이메일, 채팅

    2. 분석 대상 통계 항목 (analytics-spec.md 연계)¶

    2.1 어시스턴트가 모니터링할 지표 목록¶

    카테고리 지표 analytics-spec 위치 모니터링 방식
    매출 net_sales (일별 순 매출) 4.1.1 매출 개요 전일/전주 대비 이상 감지
    매출 gross_sales 4.1.1 전일/전주 대비 이상 감지
    매출 avg_order_value (AOV) 4.1.1 7일 이동 평균 대비 편차
    매출 returns (환불 금액) 4.1.1 환불율 급등 감지
    고객 new_customers (신규 고객 수) 4.2.1 주간 추이 분석
    고객 returning_customer_rate (재구매율) 4.2.1 월간 추이 분석
    고객 customer_segment (RFM 기반) 2.1.3 고객 데이터 이탈 위험 고객 증가 감지
    상품 return_rate (상품별 반품율) 4.3.1 상품 성과 급등 상품 감지
    상품 cart_abandonment_rate (장바구니 이탈율) 4.3.3 장바구니 분석 임계값 초과 감지
    상품 days_of_inventory_remaining (재고 소진 예상일) 4.3.2 재고 분석 7일 이하 진입 감지
    마케팅 conversion_rate_by_channel (채널별 전환율) 4.4.1 채널별 분석 채널별 급락 감지
    마케팅 roas_by_channel (채널별 ROAS) 4.4.1 임계값 미달 감지
    퍼널 checkout_abandonment_rate (체크아웃 이탈율) 4.5.2 체크아웃 단계 임계값 초과 감지
    퍼널 퍼널 단계별 이탈율 4.5.1 표준 구매 퍼널 단계별 이상 감지

    2.2 임계값 설정 기준¶

    임계값은 어시스턴트가 알림형 인사이트를 생성하는 트리거 조건이다.

    지표 기본 임계값 감지 조건
    일 매출 전주 동요일 대비 -20% 이상 하락 3일 연속 하락 시 알림
    AOV 7일 이동 평균 대비 ±15% 이탈 단 1일만 발생해도 감지
    재구매율 전월 대비 -5%p 이상 하락 월간 리포트 생성 시 체크
    장바구니 이탈율 80% 초과 7일 평균 기준
    체크아웃 이탈율 70% 초과 7일 평균 기준
    반품율 (특정 상품) 전체 반품율의 2배 초과 판매 수량 10건 이상인 상품 한정
    재고 소진 예상일 7일 이하 상위 20% 판매 상품 한정
    ROAS 1.0 미만 (광고비 > 매출) 3일 이상 지속 시 알림

    임계값 커스터마이징: - 판매자는 어드민 → AI 어시스턴트 → 알림 설정에서 각 지표별 임계값을 직접 수정할 수 있다. - 임계값 미설정 시 위 기본값이 적용된다.

    3. 인사이트 생성 유형¶

    3.1 알림형 (임계값 기반, Rule-based)¶

    임계값을 초과하거나 미달했을 때 즉시 생성되는 인사이트. 정확성이 가장 중요하므로 LLM 생성보다 규칙 기반(rule-based) 감지를 우선 적용하고, 자연어 문장은 LLM이 보조 생성한다.

    구조: 

    [감지된 이상 수치] + [전후 맥락] + [유력한 원인 후보]

    예시: - "지난 주 대비 전환율이 12% 하락했습니다. 이번 주 체크아웃 이탈율이 함께 상승한 것을 확인했습니다. 결제 단계에서 문제가 발생했을 수 있습니다." - "오늘 오전 10시~12시 동안 주문이 0건입니다. 스토어 접근성을 확인해보세요." - "상품 '레이어드 니트'의 반품율이 34%로, 전체 평균(12%)의 3배 수준입니다."

    생성 시점: 임계값 도달 즉시 생성 → 어드민 알림(앱 푸시 / 이메일) 발송

    3.2 추천형 (AI 자연어 생성, LLM 기반)¶

    정기적으로 생성되는 인사이트. 단순 수치 변화를 보고하는 것이 아니라 맥락 + 원인 + 실행 방향을 함께 제시한다.

    구조: 

    [현재 상태 요약] + [업계 대비 위치 (benchmark-report.md 연계)] + [권장 액션]

    예시: - "이번 달 재구매율이 18%로, 동종 업계 중앙값(25%)보다 낮습니다. 첫 구매 후 7일 이내 쿠폰 발송을 고려해보세요. 비슷한 카테고리 스토어에서 효과적인 전략입니다." - "이번 주 인스타그램 채널 전환율이 전주 대비 22% 상승했습니다. 최근 게시물 중 어떤 콘텐츠가 효과적인지 확인해보세요." - "AOV가 업계 상위 25% 수준입니다. 번들 상품 기획이나 무료배송 기준 조정으로 더 높이는 것을 고려할 수 있습니다."

    생성 시점: 일 1회(오전 8시), 주 1회(월요일 오전)

    3.3 예측형 (트렌드 분석, ML 모델 + LLM)¶

    과거 데이터 패턴을 분석하여 미래 수치를 예측하고 선제적으로 알린다.

    구조:

    6. CTA (Call-to-Action) 설계¶

    6.1 인사이트 → 실행 버튼 연결 원칙¶

    모든 인사이트는 반드시 1개 이상의 실행 버튼(CTA)과 함께 제공된다. CTA가 없는 인사이트는 노출하지 않는다.

    CTA 설계 원칙: 1. 즉시 실행 가능성: 클릭 한 번으로 해당 설정 화면으로 이동하거나 즉시 액션이 실행됨 2. 단일 클릭 완료: 쿠폰 발송, 가격 조정 등은 확인 팝업 후 즉시 실행 3. 맥락 유지: CTA 클릭 시 해당 인사이트의 맥락(기간, 상품, 세그먼트)이 미리 설정된 상태로 이동

    6.2 주요 CTA 목록¶

    인사이트 유형 CTA 버튼 연결 화면
    재구매율 하락 쿠폰 발송하기 쿠폰 발송 팝업 (대상 고객 사전 선택됨)
    재구매율 하락 자동화 설정하기 마케팅 자동화 → 첫 구매 후 N일 쿠폰
    재고 소진 임박 발주 요청하기 재고 관리 → 해당 상품 발주
    재고 소진 임박 재고 알림 설정 상품 → 품절 시 알림 신청 기능 ON
    반품율 급등 상품 상세페이지 수정 상품 관리 → 해당 상품 편집
    반품율 급등 반품 사유 상세 보기 반품 분석 리포트
    ROAS 하락 광고 캠페인 확인 마케팅 → 채널별 광고 설정
    이탈 고객 증가 재활성 메시지 발송 고객 → 해당 세그먼트 → 메시지 발송
    체크아웃 이탈 체크아웃 분석 보기 통계 → 전환 퍼널 → 체크아웃 단계
    배송 지연 지연 고객 안내 발송 주문 → 지연 주문 목록 → 일괄 문자
    인기 상품 노출 부족 메인 추천에 추가 스토어 설정 → 추천 상품 → 해당 상품 추가
    가격 조정 추천 가격 조정하기 상품 → 해당 상품 → 가격 편집

    6.3 액션 완료 피드백¶

    CTA 클릭 후 액션 완료 시 어시스턴트에 결과를 피드백하여 다음 인사이트 생성에 반영한다.

    예시:

    "쿠폰 발송하기" 클릭 → 312명에게 쿠폰 발송 완료 → 7일 후 자동 추적: "지난주에 발송한 재활성 쿠폰의 사용율이 11.2%(35명)입니다. 업계 평균(8%) 대비 좋은 성과입니다."

    7. 개인화 수준¶

    7.1 카테고리별 맞춤 메시지¶

    같은 지표 하락이라도 카테고리에 따라 원인과 권장 액션이 다르다.

    카테고리 전환율 하락 시 추천 액션
    패션/의류 사이즈 가이드 추가, 360도 이미지, 무료 반품 정책
    뷰티/코스메틱 성분 정보 보강, 피부 타입별 필터, 샘플 증정 이벤트
    식품/건강 유통기한 정보 강조, 정기구독 옵션 추가
    라이프스타일/홈 공간별 코디 이미지, 치수 상세 표기
    전자기기 호환성 정보 보강, 보증 정책 강조

    7.2 판매 규모별 맞춤 메시지¶

    규모 특성 메시지 톤
    Micro (1~50건/월) 운영 리소스 부족, 데이터 노이즈 큼 단순하고 즉시 실행 가능한 액션 우선
    Small (51~300건/월) 성장 단계, 채널 다변화 시작 채널 최적화 + 재구매 강화 중심
    Medium (301~1,000건/월) 운영 효율화 필요 자동화, 세그먼트 마케팅 강조
    Large (1,001건+/월) 데이터 활용 역량 있음 상세 분석, 코호트, LTV 기반 인사이트

    7.3 판매 국가별 맞춤 메시지¶

    국가 특이사항
    한국 카카오채널, 네이버페이, 네이버 검색광고 기반 인사이트
    일본 PayPay, 콘비니 결제, 라쿠텐/야후 비교 관점, 일본어 메시지
    글로벌 Stripe, PayPal 기반, 영어 메시지, 글로벌 벤치마크 기준

    8. 기술 아키텍처¶

    8.1 LLM 선택¶

    권장: Claude API (Anthropic)

    항목 내용
    모델 Claude 3.5 Sonnet 또는 Claude 3 Haiku (비용 최적화 필요 시)
    호출 목적 규칙 기반으로 감지된 인사이트를 자연어로 변환, 채팅 질의응답
    프롬프트 전략 시스템 프롬프트에 판매자 컨텍스트(카테고리, 규모, 국가) 및 최근 지표 요약 포함
    토큰 비용 절감 알림형 인사이트는 템플릿 기반으로 LLM 미호출, 추천형/예측형만 LLM 사용

    Claude API 선택 근거: - 누리메이트 백엔드 기술 스택(Python FastAPI)과 통합 용이 (Claude SDK for Python) - 한국어 자연어 품질 우수 - 구조화된 출력(structured output) 지원으로 CTA 버튼 정보를 JSON으로 함께 반환 가능

    8.2 데이터 파이프라인 연결¶

    [analytics-spec.md 정의 데이터]
        ↓
[집계 DB (읽기 전용)]
        ↓

  ├── insight_gen.py    ← 추천형/예측형: LLM 호출
  └── scheduler.py     ← 매일 오전 8시, 매주 월요일 실행
        ↓

        ↓

    8.3 인사이트 생성 서비스 설계¶

    도메인 구조 (analytics-spec.md의 backend/domain/ 패턴 준용):

    backend/domain/ai_insight/
├── model.py          ← SQLModel: AiInsight, InsightType, InsightStatus
├── service.py        ← 인사이트 생성 오케스트레이션
├── repository.py     ← 인사이트 CRUD
├── rule_engine.py    ← 알림형 임계값 감지 규칙
├── llm_client.py     ← Claude API 호출 래퍼
└── scheduler.py      ← APScheduler 기반 배치 실행

    인사이트 데이터 모델:

    class AiInsight(SQLModel):
    id: UUID
    store_id: UUID
    insight_type: InsightType  # ALERT / RECOMMENDATION / PREDICTION
    category: str              # 매출 / 고객 / 상품 / 마케팅 / 배송
    title: str                 # 카드 제목 (1줄)
    body: str                  # 인사이트 본문
    cta_buttons: list[CtaButton]  # CTA 버튼 목록
    metrics_snapshot: dict     # 근거 수치 스냅샷
    generated_at: datetime
    status: InsightStatus      # ACTIVE / DISMISSED / ACTED
    expires_at: datetime       # 인사이트 유효 기간

    8.4 개인정보 보호 처리¶

    처리 항목 방안
    LLM 프롬프트에 포함되는 데이터 집계 수치만 포함 (개별 고객 ID, 이름, 연락처 미포함)
    Claude API 데이터 전송 Anthropic 데이터 처리 약관 준수 (모델 학습에 미사용 API 사용)
    채팅 대화 기록 판매자 본인 어드민 계정으로만 접근 가능, 암호화 저장
    인사이트 데이터 보존 생성 후 90일, 이후 자동 삭제 (집계 수치는 별도 보존)

    9. 구현 우선순위 (Phase별)¶

    Phase 1 — 알림형 인사이트 (analytics-spec.md Phase 2 시점, 출시 후 3~6개월)¶

    목표: 판매자가 중요한 이상 신호를 놓치지 않도록 한다. LLM 없이 규칙 기반으로 구현하여 비용과 복잡도를 최소화한다.

    기능 설명 구현 방식
    임계값 기반 알림 5개 매출 급락, 재고 소진 임박, 반품율 급등, 체크아웃 이탈율 급등, ROAS 하락 Rule-based (LLM 미사용)
    어드민 홈 카드 위젯 인사이트 1~3개 표시 프론트엔드 카드 컴포넌트
    CTA 연결 인사이트별 기본 1개 CTA 어드민 내부 링크
    이메일 알림 알림형 인사이트 발생 시 즉시 발송 이메일 발송 서비스 연동

    Phase 1 제외 항목: - LLM 기반 자연어 생성 (Phase 2) - 채팅 질의응답 (Phase 2) - 주간 이메일 리포트 (Phase 2) - 예측형 인사이트 (Phase 3)

    Phase 2 — 추천형 인사이트 + 채팅 (출시 후 6~12개월)¶

    기능 설명
    LLM 기반 자연어 인사이트 Claude API 연동, 추천형 인사이트 생성
    주간 이메일 리포트 매주 월요일 자동 발송
    채팅 질의응답 기본 질문 유형 10개 지원
    benchmark-report.md 연계 업계 비교 기반 인사이트 (벤치마크 Phase 1 완료 후)
    개인화 (카테고리별) 카테고리별 맞춤 메시지 적용
    인사이트 피드백 "도움이 됐나요?" 평점 수집, 모델 개선 데이터 축적

    Phase 3 — 예측형 + 고도화 (출시 후 12개월 이상)¶

    기능 설명
