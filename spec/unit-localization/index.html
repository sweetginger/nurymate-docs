    작성일: 2026-03-06 버전: v0.1 (초안) 관련 태스크: [10-1]

    1. 개요¶

    1.1 배경¶

    국가별로 사용하는 물리적 단위 체계가 상이하다. 판매자가 상품을 등록할 때 입력한 단위 정보를 구매자가 위치한 국가의 단위로 자동 변환하지 않으면 다음과 같은 문제가 발생한다.

    문제 유형 구체 사례
    구매 이탈 한국 사이즈 표기(cm)를 모르는 미국 구매자가 사이즈 선택 포기
    반품 증가 단위 오해로 인한 사이즈 불일치 반품
    신뢰 하락 배송비 계산에 사용된 무게 단위(g/lb)가 달라 예상 금액 불일치
    법적 문제 일부 국가는 특정 표시 단위를 법적으로 요구

    1.2 기능 목표¶

    1. 판매자는 마스터 단위(판매자가 선택한 국가 기준)로 상품 정보를 한 번만 입력한다.
    2. 구매자가 위치한 국가에 따라 해당 국가 표준 단위로 자동 변환·표시된다.
    3. 판매자 어드민에서도 국가별 단위 변환값을 미리 확인할 수 있다.
    4. 변환 과정에서의 부동소수점 오류·반올림 불일치를 방지하는 로직을 내재한다.

    1.3 적용 범위¶

    • 상품 상세 페이지 (구매자 화면)
    • 상품 옵션 선택 (사이즈, 중량, 용량 등)
    • 배송비 계산 (상품 중량 기반)
    • 판매자 어드민 — 재고 관리·상품 목록
    • 판매자 어드민 — 배송 정책 설정

    2. 지원 단위 목록¶

    2.1 국가별 단위 기준 요약¶

    카테고리 한국 기준 일본 기준 미국 기준 비고
    길이 (소) cm cm inch 의류 사이즈, 가구 치수 등
    길이 (중) cm, m cm, m ft, inch
    무게 (경량) g g oz 식품, 화장품
    무게 (중량) kg kg lb 배송비 계산 기준
    부피 (소) ml ml fl oz 음료, 화장품
    부피 (대) L L oz (fl oz) / gal
    온도 ℃ ℃ ℉ 식품·화장품 보관 조건
    화폐 KRW (원) JPY (엔) USD (달러)
    신발 사이즈 mm (발 길이) cm (발 길이) US Size
    의류 사이즈 한국 사이즈 (44/55/66/77) 일본 사이즈 (S/M/L/XL, 숫자) US Size (XS/S/M/L/XL)

    2.2 길이 단위 상세¶

    마스터 단위 변환 대상 변환 공식 예시
    cm inch 1 cm = 0.393701 inch 170 cm → 66.9 inch
    cm mm 1 cm = 10 mm 30 cm → 300 mm
    cm m 1 cm = 0.01 m 150 cm → 1.5 m
    m ft 1 m = 3.28084 ft 2 m → 6.56 ft

    2.3 무게 단위 상세¶

    마스터 단위 변환 대상 변환 공식 예시
    g oz 1 g = 0.035274 oz 300 g → 10.58 oz
    g lb 1 g = 0.00220462 lb 1000 g → 2.205 lb
    kg lb 1 kg = 2.20462 lb 5 kg → 11.02 lb
    kg oz 1 kg = 35.274 oz 0.5 kg → 17.64 oz

    배송비 계산 기준 단위: 누리메이트 내부 배송비 계산은 g 단위를 마스터로 사용한다. 표시 시에만 국가별 단위로 변환한다.

    2.4 부피 단위 상세¶

    마스터 단위 변환 대상 변환 공식 예시
    ml fl oz (US) 1 ml = 0.033814 fl oz 100 ml → 3.38 fl oz
    ml fl oz (UK) 1 ml = 0.035195 fl oz 100 ml → 3.52 fl oz
    L gal (US) 1 L = 0.264172 gal 2 L → 0.528 gal
    L fl oz (US) 1 L = 33.814 fl oz 0.5 L → 16.91 fl oz

    주의: fl oz(US)와 fl oz(UK)는 다른 값이다. 미국 시장 대상 시 US fl oz 사용.

    2.5 온도 단위 상세¶

    마스터 단위 변환 대상 변환 공식 예시
    ℃ ℉ ℉ = (℃ × 9/5) + 32 25℃ → 77℉
    ℉ ℃ ℃ = (℉ − 32) × 5/9 68℉ → 20℃

    2.6 신발 사이즈 단위 상세¶

    신발 사이즈는 물리적 단위 변환이 아닌 사이즈 매핑 테이블 방식으로 처리한다.

    한국 (mm) 일본 (cm) 미국 남성 (US Men) 미국 여성 (US Women) EU
    230 23.0 5 6.5 36
    240 24.0 6 7.5 37-38
    250 25.0 7 8.5 39-40
    260 26.0 8 9.5 41
    270 27.0 9 10.5 42-43
    280 28.0 10 11.5 43-44
    290 29.0 11 12.5 45

    신발 사이즈 매핑은 브랜드별 편차가 있으므로, 판매자가 자체 사이즈 가이드를 추가 입력할 수 있는 기능을 제공한다.

    2.7 의류 사이즈 단위 상세¶

    의류 사이즈도 수치 변환이 아닌 매핑 테이블 방식으로 처리한다.

    한국 사이즈 일본 사이즈 미국 사이즈 실측 가슴 (cm)
    44 (XS) 5 / XS XS 76-80
    55 (S) 7 / S S 81-85
    66 (M) 9 / M M 86-90
    77 (L) 11 / L L 91-95
    88 (XL) 13 / XL XL 96-100
    99 (XXL) 15 / 2XL XXL 101-106

    의류 사이즈의 경우 판매자가 실측 치수(가슴·허리·엉덩이·총장)를 입력하면 구매자 국가에 맞는 사이즈 레이블과 실측값을 병기한다. 실측값 병기가 반품률 감소에 효과적이다.

    3. 마스터 단위 정책¶

    3.1 마스터 단위 정의¶

    마스터 단위란 판매자가 상품 등록 시 사용하는 기준 단위이며, 모든 국가별 표시값은 마스터 단위에서 변환된다.

    카테고리 마스터 단위 선택 이유
    길이 cm 한국·일본 동일, 미터법 기반
    무게 g 한국·일본 동일, 정밀 계산 가능
    부피 ml 한국·일본 동일, 소수점 없는 정수 표기 가능
    온도 ℃ 한국·일본 동일
    화폐 KRW 판매자 기준 통화 (환율 변환은 별도 기능)
    신발 사이즈 mm (발 길이) 가장 정밀한 기준
    의류 사이즈 실측 cm 매핑 테이블의 기준

    3.2 마스터 단위 입력 화면 (어드민)¶

    상품 등록 화면에서 마스터 단위로 입력을 받되, 오른쪽에 실시간 변환 미리보기를 표시한다.

    [상품 정보 입력 — 단위]

무게   [   250  ] g     →  한국: 250g  |  일본: 250g  |  미국: 8.82 oz
길이   [    30  ] cm    →  한국: 30cm  |  일본: 30cm  |  미국: 11.81 inch
부피   [   200  ] ml    →  한국: 200ml |  일본: 200ml |  미국: 6.76 fl oz

    3.3 마스터 단위 변경 정책¶

    • 마스터 단위는 스토어 전체 단위로 고정 (개별 상품별 마스터 단위 변경 불가)
    • 단, 판매자가 미국 기준으로 마스터 단위를 변경하고 싶은 경우 스토어 설정에서 변경 가능
    • 변경 시 기존 입력된 수치는 자동 변환되지 않음 — 재입력 필요 경고 표시
    • 기본값: cm / g / ml / ℃

    4. 단위 변환 오류 방지 로직¶

    4.1 부동소수점 처리 정책¶

    자바스크립트 및 대부분의 프로그래밍 언어에서 부동소수점 연산은 미세한 오차를 발생시킨다.

    예시 문제: 

    // JavaScript
0.1 + 0.2 = 0.30000000000000004  // 오차 발생
1 cm * 0.393701 = 0.393701  // 연속 곱셈 시 누적 오차 발생

    해결 방안: - 내부 계산은 항상 정수형으로 처리 (단위: 소수점 아래 자릿수만큼 곱하여 정수화) - 예: 무게 변환 시 g → oz는 (g * 35274) / 1000000 정수 연산으로 처리 - Python 백엔드에서 decimal.Decimal 타입 사용 (float 미사용) - 프론트엔드(Next.js)에서 표시 전 최종 반올림 처리

    4.2 반올림 정책¶

    카테고리 소수점 처리 예시
    길이 (cm → inch) 소수점 1자리 반올림 170 cm → 66.9 inch
    무게 (g → oz) 소수점 2자리 반올림 250 g → 8.82 oz
    무게 (kg → lb) 소수점 2자리 반올림 1.5 kg → 3.31 lb
    부피 (ml → fl oz) 소수점 2자리 반올림 200 ml → 6.76 fl oz
    온도 (℃ → ℉) 소수점 0자리 반올림 (정수) 25℃ → 77℉
    배송비 계산용 무게 소수점 버림 (g 정수) 내부 계산 시 g 단위 정수 사용

    반올림은 항상 표시 직전 마지막 단계에서만 수행한다. 중간 계산에서 반올림하면 누적 오차가 발생한다.

    4.3 변환 공식 정의 (공식 상수값)¶

    모든 변환 상수는 국제 표준값(SI 기준)을 사용하며, 코드베이스 단일 상수 파일에 정의한다.

    # backend/utils/unit_constants.py

# 길이
CM_TO_INCH = Decimal("0.393701")
INCH_TO_CM = Decimal("2.54")
CM_TO_MM = Decimal("10")
M_TO_FT = Decimal("3.28084")

# 무게
G_TO_OZ = Decimal("0.035274")
G_TO_LB = Decimal("0.00220462")
KG_TO_LB = Decimal("2.20462")
KG_TO_OZ = Decimal("35.274")
OZ_TO_G = Decimal("28.3495")
LB_TO_KG = Decimal("0.453592")

# 부피
ML_TO_FL_OZ_US = Decimal("0.033814")   # US fl oz
ML_TO_FL_OZ_UK = Decimal("0.035195")   # UK fl oz
L_TO_GAL_US = Decimal("0.264172")

# 온도
# ℉ = (℃ × 9 / 5) + 32
# ℃ = (℉ - 32) × 5 / 9

    4.4 단위 표시 이상 감지 로직¶

    입력값이 물리적으로 비합리적인 경우 경고를 표시한다.

    항목 이상 감지 조건 경고 메시지
    의류 무게 1g 미만 또는 50kg 초과 "무게 입력값을 확인해주세요"
    의류 길이 1cm 미만 또는 500cm 초과 "길이 입력값을 확인해주세요"
    온도 -273.15℃ 미만 (절대 영도 이하) "온도 입력값을 확인해주세요"
    신발 사이즈 매핑 테이블 범위 밖 "사이즈 범위 밖입니다. 직접 입력하세요"

    5. UI/UX 반영 범위¶

    5.1 상품 상세 페이지 (구매자 화면)¶

    표시 방식: - 구매자 국가 감지: IP 기반 국가 감지 (기본) → 구매자 선택 국가 (우선) - 단위 자동 변환 후 표시 - 원본 단위와 변환값 병기 옵션 제공 (판매자 설정)

    예시 — 한국 판매자 상품, 미국 구매자 접속 시: 

    [상품 정보]
무게: 8.82 oz (250g)       ← 변환값 + 원본 병기
길이: 11.8 x 7.9 inch      ← 변환값만 표시 (설정에 따라)
       (30 x 20 cm)

    사이즈 가이드 팝업: - 의류·신발 상품에 "사이즈 가이드" 버튼 자동 추가 - 클릭 시 국가별 사이즈 비교 테이블 팝업 표시 - 판매자가 실측 치수를 입력한 경우 실측값 포함 표시

    200ml 200ml 6.76 fl oz
    500ml 500ml 16.91 fl oz

    무게 옵션 — 예시 (식품·건기식):

    5.3 배송비 계산 (중량 기반)¶

    배송비 계산에 사용되는 무게는 마스터 단위(g)로 내부 계산한다.

    [배송비 계산 로직]

1. 상품 등록 시: 무게 = 250g (마스터 단위 저장)
2. 배송비 정책:
   - 500g 이하: 3,000원
   - 501g ~ 1,000g: 4,000원
   - 1,001g ~ 2,000g: 5,000원
3. 배송비 계산: 250g → 500g 이하 구간 → 3,000원
4. 미국 배송 정책:
   - 1 lb 이하: $5.00
   - 1.1 lb ~ 2 lb: $8.00
   내부 계산: 250g → 0.551 lb → 1 lb 이하 구간 → $5.00

    주의: 배송비 구간 경계값도 마스터 단위(g)로 저장하며, 판매자에게는 국가별 단위로 표시한다.

    [배송비 설정 화면 — 판매자 어드민]

■ 한국 배송 정책
  구간 1: ~ 500g (표시: ~ 500g)    →  3,000원
  구간 2: 500g ~ 1,000g            →  4,000원

■ 일본 배송 정책 (동일 수치, 단위 동일)
  구간 1: ~ 500g                    →  ¥600
  구간 2: 500g ~ 1,000g             →  ¥900

■ 미국 배송 정책
  구간 1: ~ 1 lb   (내부: 453g)     →  $5.00
  구간 2: 1 lb ~ 2 lb (내부: 907g) →  $8.00

    • 재고 CSV 내보내기 시 마스터 단위·일본 단위·미국 단위 컬럼 동시 포함 옵션

    6. 구현 우선순위 (Phase별)¶

    Phase 1 — 한국·일본 (MVP, 2026 Q2~Q3)¶

    대상 단위: cm / g·kg / ml·L / ℃

    한국과 일본은 동일한 미터법 단위를 사용하므로 Phase 1에서는 단위 변환 없이 동일 표시가 기본이다. 단, 다음 항목은 Phase 1에서 구현이 필요하다.

    항목 내용 우선순위
    의류 사이즈 매핑 (한국 ↔ 일본) 44/55/66/77 ↔ S/M/L/XL P0
    신발 사이즈 매핑 (한국 mm ↔ 일본 cm) 230mm ↔ 23.0cm P0
    상품 단위 입력 UI (어드민) 마스터 단위 입력 + 변환 미리보기 P0
    배송비 계산 단위 정규화 내부 g 기준 통일 P0
    이상 감지 로직 §4.4 기준 P1

    Phase 2 — 미국·영어권 (2026 Q4~2027 Q1)¶

    대상 단위: inch / lb·oz / fl oz / ℉

    항목 내용 우선순위
    cm → inch 변환 상품 치수 표시 P0
    g → oz / kg → lb 변환 상품 무게 표시 + 배송비 계산 P0
    ml → fl oz (US) 변환 음료·화장품 용량 표시 P0
    ℃ → ℉ 변환 보관 온도 표시 P1
    의류 사이즈 매핑 (한국 ↔ US) 55 ↔ S 등 P0
    신발 사이즈 매핑 (한국 mm ↔ US) 250mm ↔ US 7 P0
    단위 표시 병기 옵션 원본 단위 병기 여부 설정 P1
    어드민 단위 전환 드롭다운 §5.4 P2

    Phase 3 — 동남아·EU (2027 Q2~)¶

    대상 국가·단위: - 동남아시아 (태국·싱가포르·말레이시아·인도네시아): 대부분 SI 단위 사용 → 한국·일본과 동일 적용 - EU (유럽): SI 단위 사용이나 일부 국가 관행 상이 - 영국: fl oz (UK) 별도 처리 필요

    항목 우선순위
    EU 의류 사이즈 매핑 (한국 ↔ EU) P1
    영국 fl oz (UK) 구분 처리 P1
    신발 사이즈 EU 매핑 P1

    7.2 프론트엔드 설계 요건¶

    국가 감지 우선순위: 1. 구매자가 선택한 국가 (명시적 선택 최우선) 2. 로그인 사용자의 배송지 국가 3. IP 기반 감지 국가 4. 스토어 기본 국가 (fallback)

    클라이언트 사이드 처리: - 단위 변환 값은 API 응답에 포함하여 제공 (클라이언트에서 재계산 불필요) - 사이즈 가이드 팝업은 클라이언트 사이드 렌더링 (인터랙션 즉시성 우선)

    i18n 연동: - next-intl 라이브러리에 단위 포맷터 추가 - 단위 문자열 현지화 (예: "oz" vs "온스" 등) 지원

    단위 변환 정확도 테스트 케이스:

    변환 입력 기대 출력 허용 오차
