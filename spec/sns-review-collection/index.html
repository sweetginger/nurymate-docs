    SNS 리뷰 수집·동의·포인트 보상 기능 기획서 [6-9]¶

    작성일: 2026-03-10 | 버전: v0.1 참조: research/sns-review-legal.md (법적 검토)

    1. 개요 및 목표¶

    인스타그램 등 SNS에 게시된 상품 리뷰(UGC)를 자사몰 상품 페이지에 자동 수집·표시하는 기능. 법적 리스크 선제 해소를 위해 원작자 동의 획득 자동화 플로우를 필수로 포함한다.

    1.1 핵심 원칙¶

    • 동의 없이 재게시 절대 금지: 한국·일본 저작권법상 원작자 동의 없는 재게시는 저작권 침해
    • 공식 API만 활용: Instagram Graph API 공식 허용 범위 내에서만 구현
    • 투명한 동의 UI: 원작자가 허락/거부를 명확히 인지하고 응답할 수 있는 인터페이스

    2. 전체 기능 플로우¶

    [SNS 모니터링]
브랜드 해시태그·멘션 자동 탐지 (Instagram Graph API)
         |
         v
[동의 획득 자동화]
자동 DM 발송 -> 허락/거부 응답 대기
         |
    +----+----+
    허락      거부/미응답(7일)
    |              |
    v              v
[포인트 지급]   제외 처리
자동 포인트 지급
+ 고유 링크 발송
         |
         v
[자사몰 게시]
리뷰 섹션 자동 등록
(원작자 인스타 프로필·게시물 링크 표시)

    3. 기능 상세 정의¶

    3.1 SNS 모니터링 — 해시태그·멘션 탐지¶

    활용 API: Instagram Graph API (Hashtag Search API, Mention 웹훅)

    방식 API 설명
    해시태그 탐색 GET /ig-hashtag-search + GET /{hashtag-id}/recent_media 판매자가 등록한 해시태그를 포함한 최근 게시물 탐색
    브랜드 멘션 Mention Webhook (@브랜드계정 멘션 감지) 비즈니스 계정 멘션 시 실시간 알림

    API 제한 사항: - 해시태그 탐색: 하루 30개 해시태그까지 쿼리 가능 (비즈니스 계정 기준) - 최근 게시물은 최대 500개 반환 - 비공개 계정 게시물은 탐지 불가

    판매자 설정: - 추적 해시태그 최대 10개 등록 - 탐지 주기: 6시간마다 자동 스캔

    3.2 동의 획득 자동화 — DM 발송¶

    DM 발송 조건: - 공개 계정 게시물만 대상 (비공개 계정 DM 전송 금지) - 동일 사용자에게 중복 발송 방지 (30일 내 재발송 차단) - 하루 최대 발송 건수 캡 설정 (Meta 정책 준수)

    DM 템플릿 예시: 

    안녕하세요 @{작성자명}님!

[스토어명]에서 고객님의 리뷰 게시물을 발견했습니다.
저희 스토어 페이지에 해당 게시물을 소개하고 싶어요.

동의해 주시면 {포인트금액}포인트를 드립니다!

[허락하기] {동의링크}
원하지 않으시면 이 메시지를 무시해 주세요.

감사합니다 — [스토어명]

    3.3 동의 처리 — 고유 링크 및 포인트 지급¶

    동의 링크 구조: 

    https://{스토어도메인}/review-consent?token={JWT}&post_id={인스타게시물ID}

    동의 플로우: 1. 링크 클릭 → 스토어 로그인/회원가입 2. 동의 확인 페이지: 게시물 미리보기 + 허락/거부 선택 3. 동의 완료 → 포인트 즉시 지급 + 리뷰 큐에 추가

    3.4 리뷰 게시 및 관리¶

    자사몰 표시 원칙: - 원작자 인스타그램 계정명(@username) 필수 표시 - 원본 게시물 링크 표시 - 동의 철회 요청 시 즉시 삭제 처리

    판매자 어드민 관리 상태:

    상태 설명
    탐지됨 DM 미발송 상태
    DM 발송됨 응답 대기 중
    동의 완료 게시 가능
    거부 원작자 거부 (재접촉 불가)
    게시됨 자사몰에 표시 중

    3.5 동의 상태 관리 DB 설계¶

    sns_review_consents
- id
- store_id
- instagram_user_id
- instagram_media_id
- post_url
- status (detected / dm_sent / approved / rejected / published)
- dm_sent_at
- responded_at
- consent_token (JWT, 만료 7일)
- points_granted (boolean)
- published_at
- deleted_at (삭제 요청 시)

    4. 법적 컴플라이언스 조치¶

    4.1 필수 표시 의무¶

    표시 항목 한국 일본
    원저작자 계정명 표시 필수 필수
    원본 링크 표시 필수 필수
    광고성 표시 (포인트 지급 시) 필수 (공정위 지침) 필수 (景表法)
    삭제 요청 대응 48시간 내 48시간 내

    4.2 포인트 지급 시 광고성 표시 의무 검증¶

    리뷰 허락에 대한 대가로 포인트를 지급하는 경우, 해당 리뷰는 경제적 이익 제공을 통해 수집된 리뷰(광고성 리뷰)에 해당할 수 있다. 국가별 법규 요건:

    한국 (공정거래법 추천·보증 등에 관한 표시·광고 심사지침)

    • 경제적 이해관계가 있는 추천·보증에는 광고임을 명확히 표시 의무
    • 포인트를 제공받고 작성된 리뷰는 "이 리뷰는 포인트를 제공받고 작성되었습니다" 등의 문구 표시 필요
    • 배지 형식 예시: [소정의 포인트 제공 리뷰] 배지를 리뷰 상단에 자동 표시

    일본 (景品表示法 - 부당경품류 및 부당표시방지법)

    • 2023년 개정 스텔스 마케팅 규제(ステルスマーケティング規制, 2023년 10월 시행)
    • 경제적 이익을 제공받은 SNS 게시물에 광고임을 명시 의무
    • 게시물에 #PR #광고 #협찬 등 표시 필요
    • 누리메이트에서 수집해 자사몰에 게시하는 리뷰에는 출처 표시(인스타그램 원본 링크) 및 포인트 제공 여부 표시 필요

    글로벌 (FTC Guidelines — 미국)

    • FTC(미연방거래위원회): 물질적 관계(material connection)가 있는 추천에는 명확한 공개 의무
    • 포인트, 무료 상품, 할인 등 경제적 이익 → #ad #sponsored 또는 한국어 #광고 표시

    누리메이트 구현 방향

    항목 구현 방법
    포인트 지급 리뷰 배지 [포인트 지급 리뷰] 배지 자동 표시 (비활성화 불가)
    SNS 원본 링크 인스타그램 원본 게시물 링크 항상 표시
    국가별 언어 한국어 스토어: [소정의 포인트를 제공받은 리뷰입니다], 일본어 스토어: [ポイントを受け取ったレビューです]
    감추기 차단 판매자가 광고성 배지를 숨기는 기능 제공 불가

    ⚠️ 미결 법률 검토 항목: 한국 공정거래위원회 가이드라인상 "자사몰 내 포인트 지급 리뷰"의 광고성 배지 의무화 범위 → 런칭 전 법무 검토 권장

    4.3 삭제 요청 처리 프로세스¶

    1. 삭제 요청 접수 (이메일·어드민 신고 버튼)
    2. 자동 즉시 숨김 처리 (48시간 이내 보장)
    3. 운영팀 검토 후 영구 삭제 처리
    4. 요청자에게 처리 완료 이메일 발송

    5. 구현 로드맵¶

    MVP (2026년 6월 런칭 시)¶

    • [ ] Instagram 해시태그 탐지 (비즈니스 계정 연동)
    • [ ] 동의 획득 DM 발송 (반수동 — 판매자가 대상 선택 후 발송)
    • [ ] 동의 확인 페이지 + 포인트 자동 지급
    • [ ] 자사몰 리뷰 섹션 표시 (원작자명 + 원본 링크)
    • [ ] 삭제 요청 처리 기능

    Phase 2 (2026 Q3)¶

    • [ ] DM 발송 자동화 (탐지 → 발송 자동)
    • [ ] 브랜드 멘션 실시간 웹훅 처리
    • [ ] 동의 상태 관리 대시보드 고도화
    • [ ] 일본 마케팅 법규 대응 (景表法 광고 표시)

    Phase 3 (2026 Q4+)¶

    • [ ] TikTok UGC 수집 (TikTok Creator Marketplace API)
    • [ ] AI 리뷰 품질 평가 (저품질·스팸 자동 필터)

    6. 제한 사항 및 리스크¶

    리스크 설명 대응 방안
    Meta API 정책 변경 Hashtag Search API 기능 제한 이력 있음 판매자 직접 연동 옵션 병행
    DM 발송 제한 대량 DM 시 비즈니스 계정 제한 위험 일일 발송 캡 설정
    동의율 낮음 DM 응답률 10~20% 예상 포인트 금액 조정, DM 문구 A/B 테스트
    저작권 분쟁 동의 증빙 부족 시 분쟁 가능 동의 DB 타임스탬프·IP 로그 보관 (3년)
