    무통장입금 및 가상계좌 결제 기획서¶

    작성일: 2026-03-18

    1. 개요¶

    셀메이트CM은 초기 론칭 시 무통장입금(계좌이체 방식)을 지원하고, PG사 심사 완료 후 가상계좌 방식으로 단계적으로 전환하는 전략을 채택한다. 두 방식 모두 국내 B2C 이커머스에서 기본 결제 수단으로 요구되며, 자동 입금 확인 처리는 운영 효율과 판매자 경험에 직접 영향을 미친다.

    용어 정리¶

    용어 설명
    무통장입금 구매자가 고정 계좌번호로 직접 이체하는 방식. 입금자명과 금액을 수동 대조해야 함
    가상계좌 PG사가 주문별로 1회성 계좌번호를 동적으로 발급하는 방식. 입금 즉시 웹훅으로 자동 확인

    2. 무통장입금 → 가상계좌 전환 전략¶

    2.1 전환이 필요한 이유¶

    무통장입금은 기술적 진입장벽이 낮고 PG 계약 없이 즉시 운영 가능하지만, 운영 규모가 커질수록 아래의 구조적 문제가 드러난다.

    항목 무통장입금 가상계좌
    입금 확인 수동(관리자 대조) 또는 오픈뱅킹 API 연동 필요 PG 웹훅으로 자동
    입금자명 오류 오타·다른 이름 입금 시 미매칭 발생 계좌번호가 주문 고유 → 매칭 오류 없음
    환불 처리 계좌번호 별도 수집 후 수동 송금 PG 환불 API 호출로 자동화 가능
    판매자 운영 부담 고 저
    세금계산서·정산 수동 PG 정산 보고서로 통합 관리

    결론: 일 주문 50건 이상이면 수동 대조 비용이 가상계좌 수수료를 상회한다. 셀메이트CM은 플랫폼 성장 궤도에서 조기에 가상계좌로 전환해야 운영 병목을 예방할 수 있다.

    2.2 가상계좌 심사 기간 및 조건 (PG사별)¶

    아래 정보는 각 PG사 공개 가이드, 업계 공인 사례, 파트너사 기준을 종합한 것이다. 실제 계약 조건은 PG사 영업 담당자를 통해 확인 필요.

    PG사 심사 기간 필요 서류 특이 조건
    토스페이먼츠 영업일 3~7일 (온라인 신청 기준) 사업자등록증, 통신판매업신고증, 대표자 신분증, 법인인감증명서(법인의 경우) 스타트업 친화적. 연 매출 기준 없음. 단, 고위험 업종(성인·도박·여행 등)은 심사 강화
    KG이니시스 영업일 5~10일 사업자등록증, 통신판매업신고증, 대표자 신분증, 사업장 현황(웹사이트 URL), 법인인감(법인) 대형 가맹점 기반. 중소 신규 가맹점도 수용하나 매출 실적 있으면 유리
    나이스페이먼츠(NHN) 영업일 5~10일 위와 동일. 업종에 따라 추가 서류(통신판매업 거래 증빙 등) 요구 가능 쇼핑몰 솔루션 제휴(카페24·고도몰) 연동 특화. 포트원(아임포트)을 통해 간접 연동 가능
    페이플(Payple) 영업일 2~5일 사업자등록증, 통신판매업신고증, 대표자 신분증 소규모 쇼핑몰 특화. 월 최소 매출 요건 없음. 가상계좌 단독 신청 가능
    다날(Danal) 영업일 5~7일 위와 동일 + 홈페이지 사전 심사(결제 플로우 확인) 통신과금서비스 기반. 가상계좌는 부가 서비스로 운영

    공통 필수 요건¶

    1. 사업자등록 (개인사업자 또는 법인): 필수
    2. 통신판매업신고: 「전자상거래법」상 의무. 공정거래위원회 신고 후 신고번호 취득 필요
    3. 웹사이트 운영 상태: PG 심사 시 실제 쇼핑몰 URL을 확인함 → 심사 전 최소 시범 운영 페이지 필요
    4. 에스크로 서비스 가입: 전자상거래법상 소비자 보호를 위해 가상계좌 결제에서 에스크로 적용 의무(일부 업종 면제 있음)

    2.3 단계적 전환 방안¶

    [Phase 1 - 런칭 초기]          [Phase 2 - PG 심사 중]         [Phase 3 - 가상계좌 전환]
무통장입금(고정계좌)             심사 중 무통장입금 유지           가상계좌 온보딩 완료
  + 오픈뱅킹 입금알림            + 부분 자동화(오픈뱅킹)          + PG 웹훅 자동 입금 확인
  수동 확인 병행                 PG 계약·서류 제출               무통장입금 선택 병행 유지
                                                               (단계적 비중 축소)

    구체 실행 계획¶

    단계 기간 주요 작업
    Phase 1 런칭 ~ D+30 고정 계좌 무통장입금 운영. 오픈뱅킹 API 신청(금융결제원). 판매자용 입금 내역 확인 UI 제공
    Phase 2 D+7 ~ D+30 토스페이먼츠 또는 KG이니시스 가상계좌 심사 신청. 서류 준비(통신판매업신고 포함). 웹사이트 심사 대응
    Phase 3 D+30 이후 가상계좌 연동 완료. 신규 주문은 가상계좌 기본 제공. 기존 무통장입금 병행 옵션 유지 6개월

    셀메이트 PG 파트너십 활용¶

    셀메이트가 KG이니시스·토스페이먼츠 파트너십을 보유한 경우, 셀메이트CM은 제휴 가맹점(Sub-merchant) 구조로 심사 기간 단축 및 수수료 우대 협상 가능:

    • 파트너 PG 계약을 통해 심사 기간이 영업일 1~3일로 단축될 수 있음
    • PG 수수료 0.1~0.3%p 우대 조건 협상 여지 있음
    • 단, 이 구조에서는 셀메이트CM이 PG 계약 주체가 아닌 재판매자(ISP) 지위가 될 수 있어 정산 구조 설계 검토 필요

    2.4 수수료 구조 및 마진 수취¶

    PG사별 가상계좌 수수료 구조¶

    PG사 건당 수수료 월정액 비고
    토스페이먼츠 300~400원/건 없음 매출 규모에 따라 협상 가능
    KG이니시스 250~350원/건 없음 대형 가맹점은 200원 이하 협상 사례 있음
    나이스페이먼츠 300~400원/건 없음
    페이플 350원/건 없음 소규모 스타트업 기준

    가상계좌 수수료는 카드 결제(1.5~3%)와 달리 정액제(건당 고정)이므로, 소액 결제가 많은 경우 상대적으로 부담이 클 수 있음.

    셀메이트CM 마진 수취 구조¶

    셀메이트CM이 판매자에게 가상계좌를 제공하는 방식은 두 가지다:

    옵션 A: PG 비용 플랫폼 흡수 - 셀메이트CM이 PG 수수료(건당 300~400원)를 부담하고, 가상계좌 이용을 무료 제공 - 유료 플랜의 부가가치로 포지셔닝 → 전환율·ARPU 제고

    옵션 B: 마크업 수수료 수취 - PG 원가(건당 300원)에 마크업(50~100원) 추가해 판매자에게 건당 350~400원 청구 - 규모 성장 시 PG 협상력 향상 → 마진 폭 확대 가능 - 단, 판매자에게 별도 비용 부담이 될 수 있어 경쟁 플랫폼 대비 가격 매력 저하 우려

    권장: Phase 3 초기에는 옵션 A(플랫폼 흡수) 채택. 월 가상계좌 발급 건수가 1만 건을 초과하면 PG 협상을 통해 단가 인하를 실현하고, 일부를 플랜 가격에 반영하는 방식으로 전환.

    3. 무통장입금 자동 입금확인 구현¶

    3.1 구현 방식 비교¶

    방식 장점 단점 셀메이트CM 권장 여부
    가상계좌 PG 웹훅 완전 자동화. 주문-계좌 1:1 매핑으로 입금자명 매칭 불필요. 구현 난이도 낮음 PG 심사 후에만 사용 가능. 건당 수수료 발생 ✅ Phase 3 이후 기본 방식
    오픈뱅킹 API (금융결제원) 실시간 입금 알림 수신 가능. 무통장입금 자동화에 적합 핀테크 기업 자격 요건. 심사·등록 소요 시간 있음. 전자금융업 등록 여부 검토 필요 ⚠️ Phase 1~2 과도기 검토 대상
    은행 자체 API (기업 인터넷뱅킹) 특정 은행 계좌의 입금 내역 직접 조회 가능 은행별 API 표준이 달라 은행 변경 시 재개발 필요. 실시간성 제한(5~15분 폴링) ⚠️ 소규모 초기에만 임시 운영
    수동 확인 + 관리 UI 개발 없이 즉시 운영 가능 운영 부담 고. 주문량 증가 시 병목. 오류 발생 가능성 ❌ 일 50건 이하 초기에만 허용

    각 방식 상세¶

    가상계좌 PG 웹훅 방식 (권장) 

    [구매자 입금]
    ↓
[은행 → PG사 실시간 통보]
    ↓
[PG사 → 셀메이트CM 웹훅 POST 발송]
    ↓ (orderId, amount, status: "DONE" 포함)
[셀메이트CM 서버 → 주문 상태 "입금확인"으로 업데이트]
    ↓
[판매자·구매자에게 알림 발송]
    - 토스페이먼츠 기준 웹훅 이벤트: VIRTUAL_ACCOUNT_DEPOSIT - KG이니시스 기준: INIstdpay_vbank_result 콜백 URL 방식

    오픈뱅킹 API (금융결제원) 방식 

    [금융결제원 오픈뱅킹 서비스]
  - 제공 API: 잔액 조회, 거래 내역 조회, 입금 알림(입출금 통보 서비스)
  - 이용 대상: 핀테크 기업, 금융회사
  - 신청: openapi.kftc.or.kr → 서비스 신청 → 심사 (2~4주 소요)
  - 참가 기관: 주요 시중은행 + 저축은행 포함 25개 이상

[입금 알림 수신 구조]
  - 계좌 소유자가 오픈뱅킹 서비스 등록 후 알림 구독 필요
  - 실시간 Push 알림(웹훅)으로 입금 이벤트 수신 가능
  - 단, 셀메이트CM 계좌(플랫폼 계좌)에만 적용됨 → 판매자별 개별 계좌는 별도 처리 필요

    기업인터넷뱅킹 API (은행 직접 연동) 방식 - 하나은행·신한은행·기업은행 등 주요 은행이 기업 고객용 Open API 제공 - 입금 내역 조회는 폴링 방식(5~15분 간격) → 실시간성 부족 - 보안 인증(공인인증서·IP 화이트리스트) 요구 - 장기적 의존도가 높아 권장하지 않음

    3.2 입금자명 매칭 로직¶

    무통장입금 방식에서 자동 확인의 핵심 난제는 입금자명 매칭이다.

    입금자명 오류 유형¶

    유형 예시 발생 빈도
    오타 "홍길동" → "홍길둥" 낮음
    이름 순서 변경 "김철수" → "철수김" 낮음
    주문번호 미기재 입금자명만 기재, 주문번호 없음 높음
    대리 입금 주문자 ≠ 입금자 중간
    금액 불일치 부분 입금 또는 배송비 제외 중간

    권장 매칭 로직 설계¶

    Step 1: 완전 일치 (Exact Match)
  - 입금자명 == 주문자명 AND 금액 == 주문금액
  - 즉시 자동 확인 처리

Step 2: 유사도 기반 매칭 (Fuzzy Match)
  - Levenshtein Distance ≤ 1 (한 글자 오차 허용)
  - 금액 일치 조건 유지
  - 매칭 신뢰도 70% 이상 → 자동 확인
  - 신뢰도 50~70% → 관리자 검토 대기열로 이관

Step 3: 금액 기반 단독 매칭
  - 해당 금액의 대기 중 주문이 1건인 경우에만 자동 매칭
  - 2건 이상이면 관리자 수동 확인 요청

Step 4: 미매칭 처리
  - 24시간 내 미확인 → 판매자에게 알림
  - 3영업일 내 미확인 → 구매자에게 재확인 안내 (주문번호 기재 요청)
  - 7일 경과 → 자동 주문 취소 및 반환 안내

    관리자 인터페이스 요구사항¶

    • 미매칭 입금 목록: 입금자명, 금액, 입금 시각 표시
    • 수동 주문 매칭 기능: 드롭다운으로 대기 주문 선택
    • 입금 이력 히스토리 (30일 보관)
    • 판매자별 입금 통계 대시보드

    가상계좌 전환 후 매칭 로직¶

    가상계좌 방식에서는 계좌번호 = 주문번호이므로 입금자명 매칭 로직이 불필요하다. PG 웹훅의 orderId 필드로 주문을 즉시 식별한다. 이것이 가상계좌 전환의 핵심 운영 편익이다.

    3.3 법적 요건¶

    전자금융업 등록 관련¶

    서비스 형태 전자금융업 등록 필요 여부
    단순 PG 연동(토스페이먼츠·이니시스 등) 불필요 (PG사가 전자금융업자)
    직접 계좌 발급·정산 (PG 기능 직접 수행) 전자금융업 등록 필수
    오픈뱅킹 API 연동 (입금 확인 목적만) 불필요 (금융거래 중개 아님)
    셀메이트CM 자체 에스크로 제공 전자금융보조업자 등록 검토 필요

    셀메이트CM 케이스: 기성 PG사(토스페이먼츠·KG이니시스)를 통한 가상계좌 발급이므로 전자금융업 등록 불필요. 단, 플랫폼이 구매대금을 수취하여 판매자에게 지급하는 에스크로 구조를 직접 운영하려면 전자금융업자 등록 또는 기존 에스크로 서비스 제휴가 필요하다.

    기타 법적 준수 사항¶

    • 전자상거래법 제24조: 소비자 청약철회 기간(7일) 동안 에스크로 보호 의무. PG사 에스크로 서비스를 활용하면 의무 이행 가능
    • 개인정보보호법: 입금자명, 계좌번호 등 금융 정보 처리 시 개인정보처리방침 명시 및 안전조치 의무
    • 전자금융거래법 제6조: 거래 기록 5년 보관 의무
    • 특정금융정보법(특금법): 일정 금액 이상 거래 시 고객 확인(KYC) 의무 → 초기 스타트업 단계에서는 대부분 적용 면제이나 서비스 규모 성장 시 재검토

    4. 권장 구현 방향¶

    4.1 Phase별 구현 로드맵¶

    Phase 기간 결제 방식 자동화 수준 핵심 작업
    Phase 1 런칭 ~ D+30 무통장입금 (고정 계좌) 부분 자동화 (오픈뱅킹 검토) 입금 확인 관리 UI 개발. 판매자 알림 시스템 구축. PG 심사 신청
    Phase 2 D+7 ~ D+45 무통장입금 유지 동일 PG 심사 서류 완비. 통신판매업신고 완료. 가상계좌 연동 개발 (Sandbox)
    Phase 3 D+45 이후 가상계좌(기본) + 무통장입금(선택) 완전 자동화 PG 웹훅 연동. 주문-계좌 자동 매핑. 환불 자동화
    Phase 4 D+180 이후 가상계좌 단일화 완전 자동화 무통장입금 지원 종료. 가상계좌 수수료 협상 (볼륨 기준)

    4.2 기술 구현 우선순위¶

    1. [P0] 가상계좌 PG 웹훅 수신 서버: 토스페이먼츠 또는 KG이니시스 웹훅 엔드포인트 구현
    2. [P0] 주문 상태 자동 업데이트: 웹훅 수신 → 주문 상태 WAITING_DEPOSIT → PAID 전환
    3. [P1] 판매자·구매자 알림: 입금 확인 시 알림톡/SMS/이메일 발송
    4. [P1] 입금 확인 관리 UI: 미매칭 건 수동 처리를 위한 관리자 화면
    5. [P2] 오픈뱅킹 API 연동: Phase 1 과도기 자동화를 위한 금융결제원 API 연동
    6. [P2] 환불 자동화: PG 환불 API를 통한 취소·환불 처리

    4.3 PG사 선택 권고¶

    우선순위 PG사 이유
    1순위 토스페이먼츠 심사 기간 최단(3~7일). 개발자 문서 우수. 스타트업 친화적. Sandbox 환경 제공
    2순위 KG이니시스 셀메이트 기존 파트너십 활용 가능성. 대규모 가맹점 기반으로 안정성 검증됨
    보완 포트원(PortOne) 멀티 PG 추상화 레이어. 토스페이먼츠·이니시스·나이스페이 동시 연동 가능. PG사 전환 시 코드 재작성 불필요

    포트원 활용 권장: PG사를 직접 연동하는 대신 포트원을 통해 추상화하면, 향후 PG사 변경이나 추가 시 백엔드 코드 변경 없이 관리 콘솔에서 전환 가능. 초기 개발 비용을 절감하면서도 유연성 확보.

    4.4 의사결정 요약¶

    [무통장입금 운영 중]
  ↓ 오픈뱅킹 API 신청 (입금 알림 자동화)
  ↓ PG 심사 신청 (토스페이먼츠 1순위)
  ↓ 셀메이트 파트너십 여부 확인 → 있으면 KG이니시스 우선 협상
  ↓ 가상계좌 Sandbox 연동 개발
  ↓ Live 전환 후 기본 결제 수단으로 가상계좌 설정
  ↓ 무통장입금은 6개월 병행 후 단계적 종료

    5. 리스크 및 대응¶

    리스크 확률 영향 대응
    PG 심사 지연 (10~15영업일 초과) 중 중 셀메이트 파트너십 활용으로 단축. 병행 기간 연장으로 서비스 연속성 유지
    통신판매업신고 지연 중 고 런칭 전 선행 신청. 신청 확인증으로 심사 진행 가능 여부 PG사 확인
    무통장입금 미매칭 민원 증가 중 중 입금 안내 메시지에 주문번호 기재 요청 명시. 자동 안내 문자 발송
    오픈뱅킹 API 심사 지연 중 저 Phase 1은 수동 확인으로 운영. 오픈뱅킹 없이도 서비스 가능
    가상계좌 발급 실패 (은행 시스템 장애) 저 고 PG 이중화(토스 + 이니시스) 또는 무통장입금 폴백 처리

    참조: 토스페이먼츠 개발자센터(developers.tosspayments.com), 금융결제원 오픈뱅킹(openapi.kftc.or.kr), KG이니시스(inicis.com), 포트원 개발자센터(developers.portone.io)
