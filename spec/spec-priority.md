# 누리메이트 기능 기획서 개발 우선순위

> 작성일: 2026-03-11 | 버전: v1.0
> 팀 리소스: 웹프론트엔드 2명, 백엔드 2명, Flutter 앱 2명, 디자이너 1명. 총 7명 개발·디자인.
> 1차 런칭 목표: **2026년 6월**

---

## 우선순위 기준

| 등급 | 정의 |
|------|------|
| **P0** | 런칭 필수 — 2026년 6월 1차 런칭에 없으면 서비스 자체가 불가능한 기능 |
| **P1** | 런칭 직후 — 런칭 후 3개월 이내(2026년 9월) 출시 필요. 판매자 리텐션에 직결 |
| **P2** | 2026 H2 — 2026년 하반기 목표. 경쟁 우위 강화 |
| **P3** | 장기 로드맵 — 2027년 이후 또는 조건부 실행 |

## Phase별 개발 목표 요약

### P0 목표 (2026년 6월 런칭)

**웹프론트엔드**
- 셀러 스토어 생성 UX (seller-store-creation-ux.md)
- 서브도메인·커스텀 도메인 연결 (domain-architecture.md)
- 한국·일본 기본 스토어 프론트엔드

**백엔드**
- 스토어 기본 모듈 (상품·주문·결제·배송 기본)
- 도메인 라우팅·멀티테넌시 DB 격리
- PG 연동 (이니시스, 토스페이먼츠, GMO-PG Phase 1)
- 마이그레이션 엔진 (카페24·메이크샵)

**디자인**
- 셀러 어드민 UI 시스템
- 스토어 기본 테마 2~3개

### P1 목표 (2026년 9월)

- 통합 정산 대시보드
- 첫 매출 견인 도구 (웰컴 쿠폰, SNS 공유 링크)
- 기본 통계 대시보드
- Meta·TikTok 카탈로그 연동
- 어필리에이트 링크 트래킹

## 주의 사항

- P0 스펙 범위 내에서도 **MVP 범위를 최소화**해야 6월 런칭 가능. 각 스펙의 Phase 1 기능만 우선 구현
- 운영자 어드민(operator-admin.md)은 P0이지만 기본 기능(셀러 조회·상태 변경·PG 심사 모니터링)만 런칭 포함
- Flutter 앱은 P0에서 **웹뷰 기반 MVP**로 시작 가능. 네이티브 앱 기능은 P1~P2
