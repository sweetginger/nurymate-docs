    1. 개요¶

    1.1 모듈화 설계 원칙¶

    누리메이트는 헤드리스 커머스 아키텍처를 기반으로, 백엔드 도메인을 완전히 모듈화하여 필요한 기능만 선택·조합할 수 있도록 설계한다.

    핵심 원칙:

    1. 독립성: 각 모듈은 독립적으로 배포·확장 가능하다. 단, 의존 관계가 있는 경우 의존 모듈이 선행 활성화되어야 한다.
    2. 점진적 활성화: 판매자는 판매 규모와 니즈에 따라 모듈을 단계적으로 활성화할 수 있다.
    3. 멀티테넌시: 모든 모듈은 고객사별 데이터 격리(schema-per-tenant)를 기본으로 한다.
    4. AI-First: 주요 모듈에 AI 보조 기능이 내장되어 수기 작업을 최소화한다.

    1.2 분류 기준¶

    분류 정의 과금
    기본 내장 (Free) 모든 플랜에 기본 제공. 이커머스 운영의 필수 요소 플랜 기본료 포함
    선택 활성화 (Optional) 기본 포함이나 판매자가 ON/OFF 가능. 또는 특정 플랜 이상에서 제공 플랜 기본료 포함 (일부 상위 플랜)
    유료 옵션 (Premium) 별도 과금 또는 상위 플랜 전용 고급 기능 추가 과금 또는 상위 플랜

    6 마켓 전용 디자인 레이어 입점 브랜드별 미니샵 + 통합 마켓 홈 구성 3
    7 마켓 전용 프로모션 입점 브랜드 선택 대상 기획전·쿠폰 설정 3
    8 입점 브랜드 통계 브랜드별 매출·주문·전환율 분리 통계 3
    9 브랜드 수수료 자동 차감 정산 시 플랫폼 수수료 + 브랜드 수수료 자동 분리 처리 3
    10 글로벌 마켓플레이스 지원 입점형 마켓에서도 일본·동남아 결제·언어 적용 3

    M-16. 셀익스 연동¶

    분류: 선택 활성화 (Optional) — 글로벌 시장에 진출할 한국 판매자 대상 의존 모듈: M-03, M-05 개발 우선순위: P2 기반 문서: sellex-integration.md

    핵심 기능 목록:

    # 기능 설명 Phase

    10 셀익스 배송 통계 국가별 배송 현황, 포워더별 성공률 통계 3

    4. 개발 우선순위 매트릭스¶

    평가 기준: 판매자 영향도 (1=낮음, 5=높음) × 개발 난이도 역수 (1=어려움, 5=쉬움) 높은 영향도 × 낮은 난이도 = 우선 개발

    모듈 판매자 영향도 (1-5) 개발 난이도 (1-5, 5=쉬움) 우선순위 점수 우선순위 등급

    M-02 상품 관리 상품 CRUD, 옵션·SKU, 이미지 업로드, AI 설명 생성, 카테고리, 재고

    M-07 마케팅 (기초) 웰컴 쿠폰 자동 발급, SNS 공유 링크, 첫 매출 체크리스트
