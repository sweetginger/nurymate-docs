    라쿠텐·아마존 JP 상품 데이터 API 조사 [4-15]¶

    조사일: 2026-03-10 | 목적: 누리메이트 일본 플랫폼 마이그레이션 도구 설계

    1. 개요¶

    라쿠텐 시장(楽天市場)과 아마존 JP는 일본 EC 시장의 양대 마켓플레이스로, 많은 일본 셀러들이 이 두 플랫폼에 상품을 등록·운영하고 있다. 누리메이트가 마이그레이션 도구를 제공하려면 두 플랫폼의 상품 데이터 API 구조와 접근 권한 체계를 정확히 이해해야 한다.

    핵심 요약¶

    항목 라쿠텐 RMS API 아마존 SP-API
    접근 주체 셀러 본인 또는 위임받은 시스템 개발사 셀러가 OAuth로 권한 위임한 앱
    상품 데이터 export 가능 (상품 API) 가능 (Listings Items API)
    OAuth 위임 셀러가 시스템 개발사에 API 키 발급 방식 OAuth 2.0 + Login with Amazon(LwA)
    이용 비용 무료 (특정 기능 계약 필요) 2026.01.31부터 연간 $1,400 (개발자 등록비)
    정책상 제약 서비스 이용 목적 제한, 제3자 재배포 금지 개인정보·데이터 사용 정책 엄격

    2. 라쿠텐 RMS(Rakuten Merchant Server) API¶

    2-1. RMS API 개요¶

    라쿠텐 RMS는 라쿠텐 시장 출점 셀러가 상품 등록·주문 관리·재고 관리 등을 수행하는 관리 시스템이다. RMS API는 이 관리 기능을 프로그래밍 방식으로 접근할 수 있도록 제공하는 인터페이스다.

    API 접근 주소: https://api.rms.rakuten.co.jp/

    2-2. 상품 데이터 Export 가능 여부¶

    라쿠텐 RMS API를 통해 다음 상품 데이터를 조회·export할 수 있다:

    데이터 항목 API 지원 비고
    상품명 (商品名) ○
    상품 설명·캐치카피 ○
    가격 (税込·税抜) ○
    상품 이미지 URL ○ 이미지 직접 export는 URL 제공 방식
    재고 수량 ○ InventoryAPI 2.0 (SKU 이행 대응)
    상품 카테고리 ○ 라쿠텐 독자 카테고리 코드
    SKU 정보 ○ 2023년 SKU 이행 이후 InventoryAPI 2.0 사용
    리뷰 데이터 △ 공개 리뷰만 가능, API 직접 export는 제한
    주문 데이터 ○ Order API
    CSV 다운로드 ○ RMS 관리 화면에서 직접 CSV export 가능

    상품 CSV 다운로드 방법 (API 외 대안): RMS 관리 화면 → 점포 설정 → 상품 관리 → CSV 다운로드 → 대상 상품 조건 지정 → 원하는 항목 선택 후 다운로드. 대량 상품의 경우 CSV 방식이 가장 현실적인 export 방법이다.

    2-3. 접근 주체: 셀러 vs 플랫폼 사업자¶

    라쿠텐 RMS API의 접근 방식은 두 가지 경로가 있다:

    경로 1: 셀러 직접 접근 - 셀러 본인이 RMS에서 API 라이선스 키 발급 - 인증 방식: serviceSecret:licenseKey를 Base64 인코딩한 Bearer 토큰 - 2025년 6월 12일부터 "RMS 서비스 스퀘어 라이선스 키" 방식으로 인증 방법 변경 의무화

    경로 2: 시스템 개발사(누리메이트 같은 사업자) 경유 - 셀러가 시스템 개발사에 API 사용을 승인(API 承認)하는 절차 필요 - 셀러가 RMS에서 "시스템 개발사로부터의 API 승인 신청"을 수락 - 이후 해당 라이선스 키를 사용해 개발사가 API 호출 가능

    중요: 라쿠텐은 아마존처럼 OAuth 표준 위임 플로우를 제공하지 않으며, 셀러가 직접 RMS 화면에서 개발사에 권한을 부여하는 수동 프로세스다. 이는 UX 마찰을 유발하므로 마이그레이션 도구 설계 시 셀러 온보딩 가이드가 중요하다.

    2-4. API 신청 절차¶

    1. 시스템 개발사(누리메이트): 라쿠텐 파트너 포털에 개발사 등록
    2. API 이용 서비스 신청 (RMS WEB API 서비스 이용 약관 동의)
    3. 개발사가 셀러에게 "API 승인 요청" 발송
    4. 셀러가 RMS 관리 화면에서 개발사 API 사용 승인
    5. 라이선스 키 발급 완료 → API 호출 가능

    2025년 인증 방식 변경 (중요): 2025년 6월 12일부터 기존 "WEB SERVICE의 라이선스 키" 방식이 폐지되고 "RMS 서비스 스퀘어 라이선스 키" 방식으로 의무 전환. 기존 연동 시스템은 반드시 새 인증 방식으로 업데이트 필요.

    2-5. 비용¶

    • RMS API 이용 자체: 무료
    • 단, 특정 기능 사용을 위해 라쿠텐과 옵션 계약이 필요한 경우 있음
    • 예: 상품 API를 통한 대량 상품 관리를 위해서는 "CSV 대량 상품 편집" 계약 필요
    • 라쿠텐 시장 출점 자체의 월정액 비용은 별도 (약 50,000~100,000엔/월 수준)

    3. 아마존 JP SP-API (Selling Partner API)¶

    3-1. SP-API 개요¶

    아마존 SP-API는 아마존의 기존 MWS(Marketplace Web Service) API를 대체하는 현대적 REST API 체계다. 판매자, 공급업체, 물류 제공업체 등이 아마존의 리스팅·주문·재고·보고서 데이터에 프로그래밍 방식으로 접근할 수 있다.

    공식 문서: https://developer-docs.amazon.com/sp-api

    3-2. 상품 데이터 제공 범위¶

    SP-API를 통해 접근 가능한 상품 관련 데이터:

    API 명 제공 데이터 비고
    Listings Items API 상품명, 가격, 이미지, 카테고리, 속성값, ASIN/SKU 셀러 본인 리스팅 데이터
    Catalog Items API 아마존 카탈로그 상품 정보 (공개 데이터) ASIN 기반 조회
    Reports API 재고 보고서, 주문 보고서, 리스팅 보고서 (CSV) 대용량 데이터 export에 최적
    Inventory API 재고 수량, FBA 재고 현황
    Orders API 주문 데이터
    Product Pricing API 가격 정보

    리뷰 데이터: SP-API는 셀러의 자사 상품 리뷰 데이터를 직접 export하는 API를 제공하지 않는다. 아마존은 리뷰를 플랫폼 자산으로 간주하므로 직접 이전은 정책상 불허.

    이미지 데이터: 이미지는 아마존 CDN URL로 제공되며, 직접 다운로드 후 누리메이트 스토리지로 재업로드하는 방식으로 마이그레이션 가능. 단, 이미지 저작권은 셀러 소유이므로 법적 문제 없음.

    3-3. 접근 주체: OAuth 판매자 위임 방식¶

    SP-API는 OAuth 2.0 + Login with Amazon(LwA) 방식으로 판매자 인증 위임을 공식 지원한다.

    위임 플로우:

    누리메이트 (SP-API 앱 개발자 등록)
  │
  ├─ 1. 아마존 개발자 포털에서 앱 등록 (Client ID/Secret 발급)
  │
  └─ 셀러 온보딩 시
       ├─ 2. 누리메이트가 아마존 OAuth 인증 URL로 셀러 리다이렉트
       ├─ 3. 셀러가 아마존에서 로그인 및 권한 허용
       ├─ 4. 아마존이 Authorization Code 반환
       ├─ 5. 누리메이트가 Access Token + Refresh Token 교환
       └─ 6. 이후 Refresh Token으로 셀러 데이터 API 호출 가능

    이는 표준 OAuth 2.0 플로우이므로, 셀러는 ID/Password를 공유하지 않아도 되며 권한 철회도 아마존 설정에서 언제든 가능하다. 라쿠텐의 수동 승인 방식보다 현저히 우수한 UX다.

    앱 유형: - Public Application: 여러 셀러가 사용하는 앱 (누리메이트 같은 SaaS에 적합) - Private Application: 단일 판매자 자체 사용 앱

    3-4. 비용 구조¶

    개발자 등록 및 앱 사용: - 현재 (2026년 1월 31일 이전): 무료 - 2026년 1월 31일부터: 연간 $1,400 USD 구독 요금 부과 (신규 및 기존 개발자 모두 적용) - 이는 SP-API를 사용하는 모든 제3자 개발자에게 적용 - 누리메이트가 SP-API 연동을 설계한다면, 이 비용을 사업 계획에 반영 필요

    API 호출 비용: - API 호출 자체는 별도 과금 없음 (Rate Limit 내에서) - Rate Limit 초과 시 일시적 차단 (비용 아님)

    3-5. 아마존 JP 특이사항¶

    • 아마존 JP는 글로벌 SP-API와 동일한 API 사용 (별도 JP 전용 API 없음)
    • 리전 엔드포인트: https://sellingpartnerapi-fe.amazon.com (Far East — JP, AU, SG 포함)
    • 멀티 마켓플레이스 지원: 셀러가 JP와 US 등 복수 마켓에 출점 시 단일 앱으로 관리 가능

    4. 두 플랫폼 OAuth 위임 방식 지원 비교¶

    항목 라쿠텐 RMS 아마존 SP-API
    표준 OAuth 2.0 ✕ (자체 방식) ○ (OAuth 2.0 + LwA)
    셀러 권한 위임 방식 수동 (RMS 화면에서 개발사 승인) 자동 (OAuth 인증 URL 리다이렉트)

    권한 철회 셀러 또는 개발사가 RMS에서 직접 아마존 계정 설정에서 셀러 직접 철회 가능

    6. 마이그레이션 자동화 구현 시 기술적 제약 및 정책상 허용 범위¶

    6-1. 라쿠텐 정책상 제약¶

    1. 데이터 재배포 금지: 라쿠텐 RMS API를 통해 취득한 데이터를 제3자에게 재판매하거나 공개하는 것은 서비스 이용 약관상 금지. 마이그레이션은 해당 셀러의 자사 데이터 이전이므로 허용 범위.

    2. API 호출 레이트 리밋: 과도한 API 호출은 제한될 수 있으며, 대량 상품(수천 개 이상)의 경우 CSV export 방식이 더 효율적.

    3. 인증 방식 변경 리스크: 2025년 6월 인증 방식 의무 변경처럼 라쿠텐은 API 사양을 비교적 자주 변경. 장기 유지보수 비용 고려 필요.

    4. 계약 필요 기능: 상품 대량 편집 API 사용을 위해 "CSV 대량 상품 편집" 서비스 계약이 별도 필요할 수 있음.

    6-2. 아마존 정책상 제약¶

    1. 데이터 사용 목적 제한: SP-API 개발자 정책에 따라 취득한 데이터는 허용된 사용 사례(Acceptable Use Policy) 내에서만 사용 가능. 셀러 자신의 데이터를 다른 플랫폼으로 이전하는 것은 허용됨.

    2. 리뷰 데이터 이전 불가: 아마존 리뷰는 아마존의 재산으로 간주되어 API 제공 자체가 없으며, 외부 스크래핑도 ToS 위반.

    3. ASIN 종속성: 아마존의 상품 식별자 ASIN은 아마존 플랫폼 전용이므로, 누리메이트 내부에서는 새로운 상품 식별자 체계를 사용해야 함.

    4. 이미지 저작권: 상품 이미지는 셀러 소유이므로 이전 가능. 단, 아마존이 생성한 이미지(A+ 콘텐츠 등)는 아마존 소유일 수 있어 확인 필요.

    5. Throttling (스로틀링): API 호출에 Rate Limit이 있으며, 대량 상품 마이그레이션 시 Reports API(비동기 방식)를 활용하는 것이 효율적.

    6-3. 공통 기술적 제약¶

    • 카테고리 매핑: 라쿠텐/아마존의 독자 카테고리 체계를 누리메이트 카테고리로 변환하는 매핑 테이블 구축 필요
    • 이미지 재호스팅: CDN URL에서 이미지를 다운로드 후 누리메이트 S3 스토리지에 재업로드하는 파이프라인 구축 필요
    • HTML 파싱: 상품 설명이 HTML 형식인 경우 파싱 및 정제 로직 필요
    • SKU 충돌 처리: 복수 플랫폼에서 동일 셀러의 SKU가 중복될 경우 처리 로직 필요

    7. 누리메이트 플랫폼 마이그레이션 도구 설계 시사점¶

    7-1. 권장 아키텍처¶

    셀러 온보딩 화면
├─ 라쿠텐 연동
│   └─ 셀러: RMS에서 누리메이트에 API 키 발급 (가이드 제공)
│       └─ 누리메이트: API 키 저장 후 상품 API 호출
│           └─ 대안: CSV export 가이드 제공 후 업로드
│
└─ 아마존 JP 연동
    └─ OAuth 인증 플로우 (1클릭 연동)
        └─ Access Token으로 Listings Items API + Reports API 호출

    7-2. 마이그레이션 단계 설계¶

    단계 작업 기술 스택
    1. 인증 셀러 OAuth/API 키 수집 OAuth 2.0, 안전한 키 저장 (Vault)
    2. 상품 목록 조회 API 호출 또는 CSV 파싱 Background Job (Celery/ARQ)
    3. 이미지 재호스팅 이미지 다운로드 → S3 업로드 비동기 파이프라인
    4. 카테고리 매핑 규칙 기반 + AI 보조 매핑 ML 분류 또는 규칙 테이블
    5. 셀러 검토 마이그레이션 미리보기 프론트엔드 UI
    6. 최종 게시 누리메이트 상품 DB에 저장 트랜잭션 처리

    7-3. 비용 고려사항¶

    • 아마존 SP-API 연간 $1,400: 2026.01.31부터 의무 부과. 초기 사업 계획에 반영 필요.
    • 라쿠텐 RMS API: 무료이나 셀러별 온보딩 지원 비용(CS) 발생 가능
    • 이미지 스토리지: 대량 이미지 재호스팅 시 S3 비용 발생
    • 개발 공수: 라쿠텐 + 아마존 양사 연동 초기 개발에 약 6~10주 예상

    7-4. MVP 우선순위 제언¶

    1. 아마존 SP-API 먼저 구현 — OAuth 표준 지원으로 셀러 온보딩 UX 우수, API 안정성 높음
    2. 라쿠텐은 CSV 방식으로 먼저 지원 — API 연동보다 CSV 업로드 가이드가 셀러에게 더 친숙할 수 있음
    3. 라쿠텐 API 연동은 2차 단계 — 2025년 6월 인증 방식 변경 안정화 후 구현
