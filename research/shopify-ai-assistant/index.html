    Shopify AI 어시스턴트(Sidekick / Shopify Magic) 사용 사례 조사¶

    조사 기준일: 2026년 3월 관련 태스크: [17-1]

    1. Shopify AI 어시스턴트 개요¶

    1.1 Shopify Magic vs Sidekick 관계¶

    구분 설명
    Shopify Magic Shopify 전체 플랫폼에 내장된 AI 기능 집합 (텍스트 생성, 이미지 편집, SEO 최적화 등)
    Sidekick Shopify Magic을 기반으로 한 대화형 AI 어시스턴트. 관리자 패널 내 채팅 인터페이스

    Sidekick은 Shopify Magic의 대화 인터페이스 레이어다. 판매자가 자연어로 질문하거나 작업 요청을 하면 Shopify Magic의 기능을 실행하거나 안내한다.

    1.2 출시 및 주요 버전 이력¶

    시기 이벤트
    2023년 여름 Shopify Magic 최초 도입 (텍스트 생성, 이미지 배경 제거 등)
    2024년 여름 Sidekick 정식 출시 (Summer '24 Edition)
    2024년 12월 Sidekick 관리자 패널 전면 통합 (Winter '25 Edition)
    2025년 5월 대규모 재아키텍처: 멀티스텝 추론, 고급 분석, 이미지 생성 추가
    2025년 9월 앱 추천·비교·직접 설치 기능 추가
    2025년 12월 Winter '26 Edition — 프로액티브 협력자로 재정의, Agentic Storefronts 동시 발표

    1.3 접근 방법 및 과금 구조¶

    접근 방법¶

    • 진입점: 관리자 패널(admin) 우측 상단 별 모양 아이콘 클릭
    • 직접 URL: https://admin.shopify.com/store/{스토어명}/sidekick
    • 확장 모드: 패널 내 '확장' 버튼으로 전체 화면 전환 가능
    • 모바일: 데스크톱 및 모바일 브라우저 지원 (태블릿·iPad는 현재 미지원)
    • 음성 입력: 모바일에서 보이스 채팅 기능 제공

    과금 구조 요약¶

    Shopify Magic / Sidekick은 별도 유료 구독 없이 Shopify 플랜에 포함된다. 크레딧 구매나 사용량 기반 추가 과금은 현재(2026년 3월 기준) 없다. 단, 일부 고급 기능은 상위 플랜에서만 이용 가능하다.

    플랜별 기능 접근 범위¶

    기능 Starter / Trial Basic ($39/월) Grow ($105/월) Advanced ($399/월) Plus ($2,300+/월)
    Sidekick 대화형 AI - ✅ ✅ ✅ ✅
    콘텐츠 생성 (상품 설명·마케팅 카피·이메일 등) - ✅ ✅ ✅ ✅
    이미지 생성·배경 제거 - ✅ ✅ ✅ ✅
    스토어 설정 변경 (할인·세그먼트·테마·Flow 등) - ✅ ✅ ✅ ✅
    데이터 분석·ShopifyQL 쿼리 - ✅ (기본) ✅ ✅ (고급) ✅ (고급)
    커스텀 앱 생성 (Sidekick Generate Apps) - ⚠️ 임시 허용 (2026년 4월까지) ✅ ✅ ✅
    조직 수준(Organization-Level) 분석 - - - - ✅
    Skills 저장·팀 공유 - ✅ ✅ ✅ ✅

    ⚠️ 커스텀 앱 생성 기능: 정식 지원 플랜은 Grow·Advanced·Plus이며, Basic 플랜은 2026년 4월까지 임시 접근 허용 중. 이후에는 Grow 플랜 이상 필요. 시간당·주간 사용량 제한(limit)이 있으며, 초과 시 Sidekick이 알림을 발송한다.

    플랜별 주요 차이 포인트¶

    1. Starter / 무료 체험(Free Trial)
    2. Sidekick 및 Shopify Magic 이용 불가

    3. 일부 이메일 기능은 유료 플랜 전환 후 사용 가능

    4. Basic ($39/월)

    5. 대부분의 AI 기능 이용 가능
    6. 분석 기능은 기본 보고서 수준에 한정 (고급 보고서 미제공)

    7. 커스텀 앱 생성은 2026년 4월까지 임시 허용

    8. Grow ($105/월) · Advanced ($399/월)

    9. 커스텀 앱 생성 정식 지원
    10. Advanced: 고급 보고서 포함 → Sidekick의 ShopifyQL 기반 분석 쿼리 범위가 넓어짐

    11. 분석 깊이 면에서 Basic 대비 실질적 차이 있음

    12. Plus ($2,300+/월)

    13. 조직 수준(Organization-Level) 분석: 복수 스토어 통합 데이터를 Sidekick으로 조회 가능
    14. 엔터프라이즈 전용 기능 접근 (멀티스토어 대시보드 등)
    15. 커스텀 앱 생성 사용량 한도 상향 적용

    현재 알려진 제한 사항¶

    • 사용량 한도: 커스텀 앱 생성 기능에 한해 시간당·주간 한도 존재. 그 외 일반 대화·콘텐츠 생성은 쿼리 수 제한 없음
    • 서드파티 앱 비용 별도: Sidekick이 추천하거나 설치하는 앱은 해당 앱 자체 과금 정책 적용
    • 분석 깊이 차이: 코호트 분석·멀티변수 통계 등 고급 분석은 Sidekick 범위 외 (플랜과 무관하게 외부 도구 필요)

    2. 지원 기능 전수¶

    2.1 콘텐츠 생성¶

    기능 상세
    상품 설명 생성 프롬프트 기반 고품질 상품 설명문 자동 작성
    기존 설명 개선 선택 텍스트에 대해 재작성·개선 제안
    SEO 최적화 상품 페이지·메타 설명 SEO 문구 생성
    마케팅 카피 광고 문구·SNS 콘텐츠·이메일 본문 생성
    블로그 포스트 스토어 블로그용 글 초안 작성
    다국어 콘텐츠 AI 기반 현지화 및 번역 제안
    이미지 생성 프롬프트 기반 시각 콘텐츠 생성 (2025년 5월 추가)
    이미지 배경 제거 상품 사진 배경 자동 제거 (Shopify Magic)

    2.2 스토어 설정 및 관리¶

    기능 상세
    할인 코드 생성 조건 지정 후 할인 코드 직접 생성
    고객 세그먼트 생성 자연어 조건으로 고객 그룹 생성
    고객·거래처 생성 요청 내용 기반으로 고객/거래처 레코드 생성
    테마 편집 자연어로 버튼·색상·폰트·레이아웃 변경 지시
    이메일 편집 트랜잭션 이메일 내용 수정
    자동화 Flow 생성 자연어로 Shopify Flow 워크플로 생성
    태그 관리 상품·고객·주문 태그 일괄 관리

    2.3 데이터 조회 및 분석¶

    기능 상세
    ShopifyQL 쿼리 작성 웹 성과·결제·주문완료·지급 데이터 쿼리 자동 생성
    매출 분석 "지난주 매출이 왜 줄었나?" 등 자연어 질문 응답
    베스트셀러 조회 "이번 달 상위 10개 상품" 등 순위 데이터 제공
    채널별 성과 마케팅 채널별 매출 기여도 분석
    재고 현황 재고 수준·잘 팔리는 상품 상태 조회
    수요 예측 과거 데이터 기반 매출·방문자 예측

    2.4 앱 생태계¶

    기능 상세
    앱 추천 판매자 상황에 맞는 앱 추천 및 비교
    앱 직접 설치 채팅 인터페이스에서 앱 설치 완료 (2025년 9월 추가)
    커스텀 앱 생성 자연어 프롬프트로 Shopify 관리자용 커스텀 앱 생성 (Winter '26)

    2.5 Sidekick Skills (Winter '26 신기능)¶

    • 자주 사용하는 프롬프트를 Skills로 저장·재사용
    • 팀원과 공유 가능 (팀 단위 반복 작업 자동화)
    • 커뮤니티 공유 기능 (다른 판매자의 Skills 활용 가능)

    3. 실제 사용 사례¶

    3.1 설정 변경 사례¶

    사례 A — 할인 코드 생성 - 판매자 요청: "재방문 고객에게만 이번 주말 한정 15% 할인 코드 만들어줘" - Sidekick 동작: 조건 설정 후 할인 코드 생성, 실행 전 확인 요청 - 결과: 판매자 승인 시 즉시 적용

    사례 B — 테마 스타일 변경 - 판매자 요청: "공지 바 배경색을 짙은 초록으로 바꾸고 폰트를 좀 더 현대적으로 바꿔줘" - Sidekick 동작: 해당 테마 설정 수정 직접 실행 - 결과: 변경 사항은 판매자가 수동 저장 필요 (자동 저장 아님)

    사례 C — 플래시 세일 준비 - 판매자 요청: "내일 플래시 세일 준비해줘" - Sidekick 동작: 할인 코드 생성 + 테마 업데이트 + 알림 이메일 설정 등 체크리스트 생성 후 각 항목 처리

    사례 D — 자동화 Flow 설정 - 판매자 요청: "재고가 10개 이하로 떨어지면 자동으로 나한테 알림 보내줘" - Sidekick 동작: Shopify Flow 워크플로 자연어로 생성·설정

    3.2 데이터 분석·조회 사례¶

    사례 E — 매출 원인 분석 - 판매자 요청: "지난주 매출이 갑자기 떨어진 이유가 뭐야?" - Sidekick 동작: 복수 데이터 소스 분석 후 원인 후보 정리·제시

    사례 F — 상품 성과 조회 - 판매자 요청: "이번 달 가장 잘 팔린 상품 10개 보여줘" - Sidekick 동작: ShopifyQL 쿼리 자동 작성 후 결과 표 형태로 제시

    사례 G — 채널 성과 비교 - 판매자 요청: "이번 분기 마케팅 채널 중 매출 기여 1위는 어디야?" - Sidekick 동작: 채널별 매출 기여 데이터 파싱·정리 후 명확하게 제시

    3.3 콘텐츠 생성 사례¶

    사례 H — 상품 설명 생성 - 판매자 요청: "이 여름 원피스에 어울리는 상품 설명 써줘, 타깃은 20대 여성" - Sidekick 동작: 타깃·톤앤매너 반영한 상품 설명 초안 생성, 수정 반영 가능

    사례 I — 이미지 기반 생성 - 판매자 요청: "이 상품의 라이프스타일 이미지 만들어줘" - Sidekick 동작: 프롬프트 기반 시각 콘텐츠 생성

    사례 J — 커스텀 앱 생성 - 판매자 요청: "우리 스토어 전용 재고 트래커 앱 만들어줘" - Sidekick 동작: 자연어 기반 Shopify 관리자용 커스텀 앱 생성 (Winter '26)

    4. 어시스턴트 행동 방식: 직접 처리 vs 화면 안내¶

    작업 유형 방식 상세
    할인 코드 생성 직접 처리 조건 입력 → 확인 요청 → 판매자 승인 후 즉시 생성
    테마 스타일 편집 직접 처리 설정값 수정 실행 (단, 판매자가 수동 저장)
    고객 세그먼트 생성 직접 처리 자연어 조건 → 세그먼트 즉시 생성
    자동화 Flow 생성 직접 처리 Flow 워크플로 자동 생성·설정
    앱 설치 직접 처리 채팅에서 추천 → 판매자 확인 → 설치 완료
    고객·거래처 생성 직접 처리 설명 기반으로 레코드 직접 생성
    이메일 편집 직접 처리 트랜잭션 이메일 내용 수정
    데이터 분석 조회 직접 처리 ShopifyQL 쿼리 자동 작성 및 결과 제시
    복잡한 설정 변경 화면 안내 현재 위치 파악 후 해당 설정 페이지로 단계별 안내
    서드파티 앱 설정 화면 안내 Shopify 관리 범위 외 → 앱 페이지로 안내
    고급 코드 수정 화면 안내 복잡한 커스텀 코드 작업 → 개발자 권장
    외부 도구 연동 화면 안내 Klaviyo·Mailchimp·Zendesk 등 → 해당 도구 안내

    4.1 핵심 원칙: "Human-in-the-Loop"¶

    Sidekick은 판매자 승인 없이 절대 스토어를 변경하지 않는다. 모든 변경 사항은 실행 전 옵션 제시·검토·승인 단계를 거친다. 이 원칙은 Shopify가 명시적으로 공표한 설계 가이드라인이다.

    4.2 진화 방향: 안내 중심 → 실행 중심¶

    • 초기 Sidekick (2024년): 주로 설명·안내 제공. "어디에 가면 이 설정을 할 수 있습니다" 방식
    • 재아키텍처 후 (2025년 5월~): 직접 실행 비중 대폭 증가. "대신 해드릴게요, 확인해 주세요" 방식
    • Winter '26: 프로액티브 모드 추가 — 판매자가 요청하기 전에 기회·문제를 먼저 감지하고 제안

    5. 처리 한계 및 오류 처리¶

    5.1 처리 불가 작업¶

    한계 설명
    고객 대화 불가 판매자 스토어의 고객과 직접 소통하거나 CS 대응 불가
    타 판매자 데이터 접근 불가 다른 Shopify 판매자의 데이터 공유 불가
    서드파티 도구 제어 불가 Klaviyo, Google Ads, Mailchimp, Zendesk, Gorgias 등 외부 도구 직접 조작 불가
    이미지 파일 직접 편집 불가 테마·상품 이미지 파일 직접 수정 불가 (배경 제거는 Shopify Magic 별도 기능으로 가능)
    커스텀 코드 작성 불가 복잡한 Liquid 코드·API 통합 등 개발 작업 불가 (Winter '26에서 커스텀 앱 생성은 가능해짐)
    외부 지식베이스 접근 불가 Confluence·Google Docs 등 외부 KB 참조 불가
    모바일 앱 미지원 Shopify Mobile App 및 태블릿(iPad) 미지원

    5.2 오류 처리 방식¶

    • 타임아웃: 지나치게 긴 지시나 복잡한 요청 시 응답 중단·타임아웃 발생 가능
    • 테마 호환성: Dawn 테마 11.0.0 버전 미만에서는 일부 블록 렌더링 오류 발생 가능
    • 복잡한 블록: 특정 복잡한 블록은 정상 렌더링되지 않을 수 있으며 사이트 전체에 영향 가능
    • 범위 초과 요청: 처리 불가 요청에 대해서는 "이 작업은 지원하지 않습니다" 응답 후 대안 안내

    5.3 확인 절차¶

    • 모든 스토어 변경 작업은 실행 전 판매자 확인 요청
    • 테마 변경 시 자동 저장되지 않으며 수동 저장 필요
    • 저장된 정보 업데이트 시 확인 메시지 제공
    • 복수 작업(예: 플래시 세일 준비) 시 체크리스트 형태로 각 항목 순차 승인 유도

    6. 누리메이트 AI 어시스턴트 비교 및 시사점¶

    6.1 Shopify Sidekick vs 누리메이트 AI 어시스턴트 비교¶

    비교 항목 Shopify Sidekick 누리메이트 AI 어시스턴트 (기획 방향)
    타깃 판매자 글로벌 영어권 중심, 기술 친숙도 높음 한국·일본 중소 셀러, 비기술 사용자 비중 높음
    결제 도메인 지식 Shopify Payments 중심, 한국·일본 로컬 결제 미흡 한국 PG(이니시스·토스페이먼츠)·일본 결제(콘비니·PayPay) 완전 지원
    언어·현지화 영어 우선, 다국어 지원 제한적 한국어 우선, 일본어 병행 — 로컬 문화 이해 내장
    직접 실행 범위 할인·세그먼트·테마·Flow 등 폭넓음 초기 단계에서는 안내 중심, 점진적 확장 권장
    데이터 분석 깊이 자체 ShopifyQL 기반 강력한 쿼리 초기에는 기본 매출·재고 조회에 집중
    앱 생태계 연동 앱 스토어 직접 연동·설치 누리메이트 내 기능 모듈 연동에 집중
    확인 절차 모든 변경 전 승인 요구 동일 원칙 필수 적용
    서드파티 제어 불가 (명시적 한계) 초기에는 동일 한계 수용

    6.2 누리메이트가 차별화할 수 있는 영역¶

    1. 로컬 결제 컨텍스트 내장
    2. "이번 달 카카오페이 결제 비율이 어떻게 돼?"
    3. "콘비니 결제 고객 재구매율이 카드 결제 고객보다 높아?"

    4. Shopify Sidekick이 제공할 수 없는 한국·일본 결제 수단별 분석

    5. 한국 이커머스 워크플로 특화

    6. 네이버 스마트스토어·쿠팡 크로스채널 재고 동기화 가이드
    7. 국내 배송사(CJ대한통운·한진택배) 연동 설정 안내

    8. 세금계산서 발행·부가세 신고 보조

    9. 비기술 사용자 친화 UX

    10. Shopify Sidekick은 영어 기반 기술 문서에 익숙한 사용자 전제

    11. 누리메이트는 PC 온라인 쇼핑몰 처음 개설하는 소상공인 수준의 사용자도 쉽게 이해할 수 있는 응답 설계

    12. 일본 현지 규정 가이드

    13. 일본 특정상거래법 표기 자동 생성
    14. 일본 소비세(JCT) 표시 방식 가이드
    15. 이 영역은 Shopify Sidekick이 완전히 공백 상태

    6.3 Sidekick 설계에서 차용할 원칙¶

    1. Human-in-the-Loop 필수: 모든 스토어 변경은 판매자 확인 후 실행. 누리메이트에서도 동일 원칙 적용
    2. 컨텍스트 파악: Sidekick은 판매자가 현재 어느 페이지에 있는지를 인식하고 맥락 기반 응답. 누리메이트도 현재 메뉴·화면 컨텍스트 주입 필요
    3. Skills 저장 기능: 자주 쓰는 프롬프트를 저장·재사용하는 기능은 반복 작업이 많은 중소 셀러에게 특히 효과적
    4. 점진적 에이전트화: 초기에는 안내 중심 → 사용자 신뢰가 쌓이면 직접 실행 범위 확대. Shopify가 실제로 걸어온 경로와 동일

    6.4 Sidekick의 실패에서 배우는 교훈¶

    • 초기 저조한 사용률의 원인: 안내만 해주고 실제로 대신 해주지 않았기 때문
    • 누리메이트 시사점: AI 어시스턴트의 핵심 가치는 "정보를 알려주는 것"이 아니라 "대신 해주는 것". 판매자가 수행해야 할 단계 수를 줄이는 것이 KPI
    • 도입 초기 설계: 작은 작업(할인 코드 생성, 상품 설명 생성)부터 직접 실행 가능하게 설계해야 사용자가 어시스턴트의 가치를 빠르게 체감
