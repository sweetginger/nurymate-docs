    일본 콘비니 픽업 배송 수령 연동 방식 조사 [4-12]¶

    조사일: 2026-03-10 | 목적: 셀메이트CM 일본 배송 옵션 설계

    1. 개요: 주소 입력만으로 콘비니 수령이 가능한가?¶

    결론부터 말하면 주소 입력만으로는 콘비니 수령이 불가능하며, 배송사 API 또는 시스템 연동이 필수이다.

    일본의 콘비니 픽업 수령은 크게 두 가지 구조로 운영된다.

    방식 설명 필요 연동
    배송사 경유 방식 야마토 운수·일본우편 등 배송사가 콘비니와 계약 후 배달, EC 사업자는 배송사 API 연동 배송사 API (야마토/일본우편)
    콘비니 직접 연동 방식 EC 플랫폼이 세븐일레븐·패밀리마트 등과 직접 계약, 해당 콘비니 네트워크에 배송 지시 각 콘비니 체인 시스템

    실제로 Amazon·메르카리 등 대형 EC는 배송사 경유 방식을 채택하고, 콘비니 직접 연동은 콘비니 결제(コンビニ決済)와 혼동되는 경우가 많다. 셀메이트CM 수준의 신규 플랫폼에서는 배송사 API 연동 경로가 현실적이다.

    2. 배송사별 콘비니 수령 API 상세¶

    2-1. 야마토 운수 — EC 자택 외 수령 API¶

    야마토 운수는 "EC 자택 외 수령 API(EC自宅外受け取りAPI)" 를 공개하고 있으며, 이는 EC 사업자가 자사 사이트에 직접 통합할 수 있는 REST API이다.

    지원 수령 거점: - 전국 약 25,000개 이상의 거점 - 콘비니 (패밀리마트·로손·미니스톱 등) - 오픈형 택배 로커 "PUDO 스테이션" - 야마토 영업소

    참고: 세븐일레븐은 야마토 운수의 콘비니 수령 거점에서 제외된다. 세븐일레븐은 자체 물류망(세븐넷 쇼핑 등)을 운영하기 때문에 외부 EC와의 수령 연동이 원칙적으로 불가하다.

    비용: - API 이용 자체는 무료 - EC 사업자에게 추가 수수료 없음 - 단, 야마토 운수와 택배(宅急便) 월별 후불 계약(掛売契約) 및 B2 클라우드 이용이 필수

    연동 조건: - 야마토 비즈니스 멤버스(ヤマトビジネスメンバーズ) 가입 필요 - B2 클라우드: 야마토의 EC 사업자용 운송장 발행·배송 관리 시스템 - 연동 대상: 일반 택배 발불(一般宅急便 発払い)만 지원 (착불·냉장냉동 등 일부 상품 제외) - 개발은 EC 사업자 측이 자체 수행 (야마토는 API 사양서·테스트 환경 제공)

    API 주요 기능: - 수령 가능 거점 목록 조회 (현재 위치 기반 반경 검색 포함) - 수령 거점 지정 및 운송장 정보 연동 - 고객에게 수령 준비 완료 통지 (야마토 → 고객 직접 SMS/이메일)

    2-2. 일본우편 — 콘비니·우체국 창구 수령 서비스¶

    일본우편(日本郵便)은 EC 사업자를 대상으로 "コンビニ・郵便局窓口・はこぽす受取サービス" 를 제공한다.

    지원 수령 거점: - 로손, 패밀리마트, 미니스톱 (세븐일레븐 제외) - 전국 우체국 창구 - 하코포스(はこぽす): 우체국 내 무인 택배 로커

    서비스 형태:

    서비스명 사전 절차 대상 상품
    e수취 어시스트(e受取アシスト) 고객 사전 등록 필요 모든 ゆうパック
    コンビニ・郵便局窓口受取 사전 등록 불필요 제휴 통판 사이트 발송 상품만

    EC 사업자가 일본우편의 통판 클라우드 시스템(通販クラウドシステム) 또는 호환 시스템(ecbeing 등)과 연동하면, "ゆうパックプリントR" 소프트웨어를 통해 콘비니 수령용 운송장 발행 및 데이터 연동이 가능하다.

    제휴 연동 절차: 1. 일본우편과 ゆうパック 계약 체결 2. 통판 클라우드 시스템 또는 연동 솔루션 도입 3. 운송장 발행 소프트웨어 설정 4. 고객 수령 거점 선택 UI 구현 (EC 사업자 자체 개발)

    2-3. 사가와 급편(佐川急便)¶

    사가와 급편은 콘비니 수령 서비스를 사실상 미지원하며, 편의점 배달(コンビニ置き配)과 헷갈리지 않도록 주의가 필요하다. 사가와는 주로 자사 영업소 수령 방식을 지원한다.

    3. 각 콘비니 체인별 직접 API 제공 여부¶

    체인 외부 EC API 직접 제공 배송사 경유 수령 가능 비고
    세븐일레븐 없음 (비공개) 불가 자체 7NOW·세븐넷 전용
    패밀리마트 없음 (비공개) 가능 (야마토·일본우편) 야마토 PUDO 연계
    로손 없음 (비공개) 가능 (야마토·일본우편) Uber Eats 등 자체 연동은 배달 전용
    미니스톱 없음 (비공개) 가능 (일본우편) AEON 계열

    핵심 결론: 콘비니 체인이 외부 EC 사업자에게 직접 픽업 API를 공개하는 사례는 현재 없다. 모든 연동은 야마토 운수 또는 일본우편을 경유하는 간접 방식이다. 세븐일레븐은 자체 생태계로 완전히 폐쇄되어 있어 외부 연동이 불가하다.

    4. 콘비니 픽업 UX 구현 방식¶

    4-1. 수령 거점 선택 인터페이스¶

    야마토 운수 API를 활용한 표준 UX 플로우:

    주문 완료 전 화면
 └─ 배송 방법 선택
      ├─ 자택 배송 → 주소 입력
      └─ 콘비니/로커 수령
           └─ 지도 또는 목록에서 거점 검색
                ├─ 현재 위치 기반 검색 (GPS)
                ├─ 우편번호/주소 기반 검색
                └─ 거점 선택 확정
                     └─ 주문 완료

    지도 연동: 야마토 API가 반환하는 거점 좌표(위도·경도)를 Google Maps API 또는 Yahoo!地図 API에 표시하는 방식이 일반적이다. 야마토 API 자체에 지도 UI는 포함되지 않으므로 EC 사업자가 별도 지도 라이브러리를 구현해야 한다.

    필수 표시 항목: - 콘비니명 및 체인 로고 - 주소 및 영업시간 - 보관 기간 (야마토 기준: 수령 준비 완료 후 7일) - 거리 (현재 위치로부터)

    4-2. 고객 통지 흐름¶

    야마토 EC 자택 외 수령 API 사용 시: 1. EC 사업자 → 야마토: 수령 거점 지정 운송장 발행 2. 야마토 → 콘비니: 배달 3. 야마토 → 고객: 수령 준비 완료 알림 (SMS/이메일, 야마토가 자동 발송) 4. 고객 → 콘비니: 인증번호 제시 후 수령

    5. 경쟁사 콘비니 픽업 지원 방식 벤치마크¶

    5-1. Shopify Japan¶

    2024년 3월, Shopify Japan과 야마토 운수가 공식 파트너십을 발표했다. 미쓰이물산(三井物産)의 "Plus Shipping" 앱을 통해 Shopify 스토어에 야마토·일본우편·사가와 연동이 가능하며, 배송 처리 공정을 10단계에서 3단계로, 처리 시간을 15분에서 1분으로 단축하는 성과를 보였다.

    • 콘비니 수령 지원: 야마토 API 경유로 지원 가능 (앱 설치 필요)
    • 코비니 결제(決済)와 수령(受取)는 별개로 관리됨

    5-2. BASE¶

    BASE는 배송사 연동을 직접 제공하지 않으며, 기본 배송 방식은 자택 배송 중심이다. 별도 앱(プラグイン)을 통해 야마토·사가와 운송장 발행이 가능하나, 콘비니 수령 선택 UI는 공식 지원하지 않는다. 소규모 개인 셀러 타겟으로 복잡한 배송 옵션 미지원이 포지셔닝이다.

    5-3. STORES¶

    STORES도 BASE와 유사하게 자택 배송 중심이며, 콘비니 수령 기능을 공식 지원하지 않는다. 실점포와 POS 연동을 강점으로 내세우는 서비스이므로, 클릭&콜렉트(매장 수령) 방향으로 발전할 가능성은 있으나, 콘비니 픽업은 미지원이다.

    5-4. 카페24 (일본 진출 시나리오)¶

    카페24는 한국에서 CJ대한통운·롯데택배 API 연동 경험이 있으나, 일본 시장에서 야마토·일본우편 콘비니 수령 API 연동 사례는 아직 없다.

    6. 셀메이트CM 설계 시 시사점¶

    권장 구현 경로¶

    1. 1차 MVP: 야마토 운수 EC 자택 외 수령 API 연동
    2. 패밀리마트·로손·미니스톱 + PUDO 로커 약 25,000개 거점 확보
    3. API 이용료 무료, 야마토 택배 계약만 필요

    4. 세븐일레븐 미지원 리스크 고객에게 사전 고지 필요

    5. 2차 확장: 일본우편 통판 클라우드 연동 추가

    6. 우체국 창구 수령 옵션 추가로 접근성 향상

    7. 세븐일레븐 대응: 현재로서는 공식 연동 불가. 세븐일레븐 자체 EC인 세븐넷 쇼핑 입점 방식으로만 접근 가능하며, 독립 EC 플랫폼에서의 연동은 기술적·계약적으로 막혀 있음.
