    1. 제품 비전¶

    1.1 미션 & 비전¶

    내용
    미션 누구나 쉽게 글로벌 커머스를 시작하고 성장할 수 있도록 한다
    비전 아시아에서 시작해 세계 표준이 되는 헤드리스 커머스 플랫폼
    포지션 한국 최적화 × 글로벌 확장성 — 둘 다 가진 이커머스 플랫폼

    1.2 핵심 일정¶

    마일스톤 시점 비고
    1차 런칭 (한국 베타) 2026년 6월 Phase 1 완료 기준
    Phase 2 본격 온보딩 2026 Q3 한국 확산 + 일본 공식 출시
    Phase 3 스케일업 2027+ 동남아 진입

    ⚠️ 6월 런칭을 위해 3월 중 착수 필수: Stripe Atlas 미국 법인 설립 (1~2주 소요) + GMO-PG 제휴 협상 착수 (심사 2~3개월)

    1.3 핵심 가치 제안¶

    • For 카페24 사용자: 익숙한 한국 환경 + 글로벌 진출 사다리
    • For Shopify 사용자: 글로벌 자유도 + 한국 결제·세금·CS 완전 지원
    • For 신규 창업자: AI가 도와주는 30분 셋업, 가입 즉시 판매 시작

    1.3 기존 StoreX와의 관계¶

    누리메이트는 StoreX의 차세대 진화 버전이다. StoreX를 리뉴얼하는 것이 아니라, 완전히 새로운 독립 플랫폼으로 처음부터 구축한다.

    StoreX (기존 버전 — 현행 유지)      누리메이트 (차세대 플랫폼 — 신규 독립 개발)
    │                                         │
    │  기존 고객 안정 운영                      │  신규 고객 전량 온보딩
    │  독립 판매 채널로 계속 운영                │  글로벌·일본 시장 공략
    │  (누리메이트 백엔드와 API 연동 없음)        │  완전히 새로운 코드베이스
    │                                         │
    │  단계적 마이그레이션 ────────────────────▶│
    │                                         │
    └──────────── 향후 법인 분리 검토 ───────────┘
    • StoreX: 기존 버전을 그대로 유지. 독립 판매 채널로 운영되며, 누리메이트와 기술적으로 통합되지 않음 (별도 시스템)
    • 누리메이트: StoreX를 전신(predecessor)으로 하는 차세대 독립 플랫폼. 코드베이스 완전 분리, 새로 구축
    • 신규 고객은 전량 누리메이트로 온보딩
    • 기존 고객(STO, DBDB)은 안정성 검증 후 단계적 마이그레이션

    2. 시장 분석¶

    2.1 일본 시장 — 2순위 공략 (Phase 2)¶

    왜 일본인가 - Shopify가 글로벌 1위지만 일본 로컬 최적화 부족 - 일본 특유 결제 구조(콘비니·은행이체·JCB)를 Shopify가 제대로 지원 못함 - 한국과 유사한 이커머스 문화 (빠른 배송, 상세페이지 중시, 리뷰 의존) - 한국 기업이 일본 시장에서 기술·서비스 신뢰를 쌓아온 선례 다수 - 일본 BtoC EC 시장 2024년 26.1조 엔 (전년비 +5.1%), D2C 연성장률 15~20% (출처: japan-korea-ecommerce.md) - 일본 내 Shopify 스토어 도쿄 단독 18,142개, 전년 대비 +16% 성장 — 이탈·전환 가능한 판매자 기반 확인됨 (출처: japan-korea-ecommerce.md)

    Shopify 일본 내 약점 (수치 근거 포함) - 일본 로컬 결제 수단 연동 복잡: komoju 계정 개설~실사용까지 최소 3~5 영업일 + 수작업 1~2시간 소요 (출처: shopify-jp-payment-ux.md) - 이중 수수료 구조: komoju 결제 수수료 ~3% + Shopify 외부 거래 수수료 최대 2% = 실질 부담 최대 5% (출처: shopify-jp-payment-ux.md) - 일본어 CS 부재 → 기술 문의·주말 문의 수 영업일 소요, Shopify Magic AI 기능 2024년 기준 일본어 미지원 (출처: shopify-japan-weaknesses.md) - 일본 내 세금 처리(소비세, 인보이스 제도) 자동화 미흡 — 인보이스 제도 대응 별도 앱 10종 이상 필요 (출처: shopify-japan-weaknesses.md) - 콘비니 결제 환불 시 건당 ¥1,500 수수료 추가 발생 (출처: shopify-jp-payment-ux.md) - 2025년 1월 Amazon Pay 연동 종료 — Amazon Pay 유무에 따라 매출 최대 40% 차이 발생, MakeShop 이전 사례 증가 (출처: shopify-japan-weaknesses.md)

    우리의 공략 방향 - 일본어 완전 지원 (UI + CS + 공식 문서) - 주요 일본 결제 수단 기본 탑재 (콘비니, PayPay, JCB, 은행이체) - 일본 소비세·인보이스 제도 자동 처리 - GMO-PG, SoftBank Payment 제휴로 현지 카드 매입 - 현지 파트너(에이전시·EC 컨설턴트) 채널 활용

    타겟 세그먼트 (일본) - 현재 Shopify 사용 중인 중소 규모 EC 사업자 - 한국 브랜드의 일본 진출 스토어 - D2C 전환을 고려하는 일본 중소 제조사

    2.2 한국 시장 — 1순위 공략 (Phase 1)¶

    한국 시장 규모 및 경쟁 지형 - 한국 이커머스 총 거래액 2024년 242조 원 (출처: korea-ecommerce-platforms.md) - 카페24: 시장 솔루션 점유율 약 70%, 누적 스토어 200만 개 이상 (출처: korea-ecommerce-platforms.md) - 쿠팡 시장 점유율 39.7%, 네이버 쇼핑 17~22.5% — 마켓플레이스 양강 구도 (출처: japan-korea-ecommerce.md)

    카페24 약점 분석 (수치 근거 포함) - UI/UX 노후화 — 관리자 화면 복잡도 높음 - 글로벌 진출 지원 제한적 - 앱 생태계 빈약, 디자인 자유도 낮음 - AI 활용 기능 부재 - 데이터 분석 기능 약함: 기본 통계 1시간 지연 업데이트, 코호트 분석·실시간 퍼널 분석 미지원, 간편결제 비중 70~80% 환경에서 GA4 전환 추적 누락 빈발 (출처: cafe24-analytics.md)

    우리의 공략 방향 - 글로벌 진출 준비 중인 한국 D2C 브랜드 선점 - AI 기반 운영 자동화로 인력 의존 탈피 - 헤드리스 구조로 디자인 완전 자유화

    타겟 세그먼트 (한국) - 카페24 사용 중이나 해외 진출 니즈 있는 브랜드 (Phase 1 거점) - 월 거래액 1천만~5억원 성장 중인 D2C 셀러 - 오프라인 기반 제조사의 D2C 전환 수요

    2.3 글로벌 확장 방향¶

    • Phase 1: 한국 집중 (카페24 대체)
    • Phase 2: 일본 (Shopify 대체)
    • Phase 3: 동남아 (한국 브랜드 진출 수요 多)
    • Phase 4: 미국·유럽 (Shopify 본진, 장기 과제)

    3. 제품 방향성¶

    3.1 헤드리스 커머스 아키텍처¶

    [프론트엔드 레이어]        [백엔드 도메인 레이어]
  - 기본 템플릿 다수        - 상품 관리
  - 외부 에이전시 자유 구성  - 주문 관리 (OMS)
  - 완전 독립 배포          - 결제
                           - 회원·CRM
                           - 재고·WMS
                           - 마케팅
                           - 분석·리포트
                           - PLM, POS, SellEx 연동
    • 백엔드 도메인 완전 모듈화 → 필요한 모듈만 선택·조합
    • 프론트엔드 독립: 디자인 에이전시도 자유롭게 구성 가능
    • StoreX는 기존 독립 판매 채널로 유지 (누리메이트와 기술 통합 없음)

    3.2 인프라 원칙¶

    • 멀티테넌시 + 오토스케일링: 공용 서버에 다수 고객사, 트래픽 자동 확장
    • 고객사별 개별 DB 스키마: 데이터 분리, 마이그레이션, 복구 전략 적용
    • VIP 고객사 전용 서버: 대형 고객사 완전 분리 운영
    • DB: PostgreSQL (MySQL → 마이그레이션, 대용량 처리·경량화)
    • 클라우드: AWS vs Azure 비용·성능 비교 후 결정 (담당: 은준, 성배)

    3.3 AI-First 제품 철학¶

    목표: 수기 작업과 시간 투자를 최소화. 사용자가 어려움 없이 쉽고 빠르게 운영

    영역 AI 기능
    온보딩 업종·상품 입력 → 스토어 자동 구성 제안
    상품 등록 이미지 업로드 → 제목·설명·태그 자동 생성
    마케팅 데이터 기반 광고 소재 자동 생성 → 매체 연동·송출 자동화
    운영 분석 매출·고객 행동 AI 리포트 자동화
    고객 CS AI 챗봇 1차 대응
    기능 개발 고객 요청 → AI가 Jira 티켓 자동 등록 → 투표 → 우선순위 결정

    핵심 목표: 가입 후 30분 안에 첫 판매 가능한 스토어 완성

    4. 결제 인프라 전략¶

    4.1 전체 구조¶

    결제 레이어는 시장별로 독립된 구조로 운영된다. 한국 법인은 Stripe 계정을 직접 개설할 수 없으므로 (2026년 3월 기준 한국은 Stripe 가맹 국가 제외), 국내결제·크로스보더·일본 로컬결제를 별도 채널로 분리한다.

    [국내결제 — 한국 원화]        [크로스보더 — 해외 구매자]     [일본 로컬결제]
한국 PG사                     글로벌 결제 솔루션              일본 현지 PG사
KG이니시스 / 토스페이먼츠       Payoneer 또는                  GMO Payment Gateway
├── 한국 카드                  Paymentwall 검토               SoftBank Payment
├── 가상계좌(무통장)            (Stripe는 한국 법인 기준         ├── 콘비니 결제
├── 휴대폰 소액결제              계정 개설 불가)                 ├── PayPay / JCB
└── 에스크로                                                   └── 은행이체
         │                           │                              │
         └───────────────────────────┴──────────────────────────────┘
                                     │
                              포트원(PortOne) 미들웨어 (통합 검토)
                                     │
                              누리메이트 정산 대시보드
                              (채널별 정산 통합 집계)

    ⚠️ 주의: 한국 PG사와 일본 PG사는 Stripe Connect 정산 구조 안에 포함되지 않는다. 각각 별도 계약·별도 정산 채널로 운영된다. 크로스보더 레이어는 Stripe 활용 시 해외 법인(미국·싱가포르 등) 명의 계정 개설이 전제조건이다.

    4.2 Stripe Connect — 수익 모델 핵심 인프라¶

    • 역할: 다국가 크로스보더 정산 + 누리메이트 PG 마진 수취
    • 수익 구조: 머천트 거래액 × 플랫폼 수수료율 = 누리메이트 매출
    • 필수 전제조건: 미국 법인 보유

    ⚠️ 즉시 착수 필요 — Stripe Atlas 미국 법인 설립 - 방법: Stripe Atlas (Delaware C-Corp) - 비용: 약 $500 - 처리 기간: 약 1-2주 - 완료 후: Stripe Connect 플랫폼 심사 신청 (추가 수 주 소요) - 6월 출시 목표 시 3월 중 착수하지 않으면 일정 위협

    4.3 시장별 현지 PG 전략¶

    한국 - 토스페이먼츠, KG이니시스, NHN KCP - 간편결제: 카카오페이, 네이버페이, 삼성페이 - 국내 카드사 직매입 지원

    일본 (Phase 2) - GMO Payment Gateway — 일본 최대 PG, 콘비니 결제 포함 - SoftBank Payment — 이동통신 청구 결합 결제 - PayPay — 일본 1위 QR 결제 - JCB 현지 매입 - ⚠️ PG 심사 기간 2-3개월 → 3월 착수 필수

    글로벌 - Stripe: Stripe Connect와 통합, 135개국 커버 - Adyen: 현지 매입망 넓음, 대형 머천트 적합

    4.4 정산 현지화¶

    • 각국 통화로 정산 지원 (엔화, 원화, USD 등)
    • 정산 주기: D+1 목표
    • 환전 수수료 투명하게 공개

    4.5 PG 심사 전 결제 지원 방안¶

    문제: 스토어 개설 즉시 PG사 심사로 인해 카드결제 즉시 개시가 불가하다. 일반적으로 심사에 1~3 영업일이 소요되어, 신규 판매자의 초기 경험을 저해하고 이탈을 유발할 수 있다.

    해결 옵션 (우선순위 순)

    옵션 설명 구현 난이도 비고
    계좌이체 결제 심사 불필요, 개설 즉시 활성화 가능 낮음 한국 소비자 무통장입금 수요 높음
    카페24식 사전 심사 구조 회원가입 시 PG 심사를 미리 진행 → 개설 즉시 카드결제 가능 중간 PG사 협의 필요, 온보딩 플로우 설계 변경
    이니시스 바로오픈 카드결제 KG이니시스의 조건부 즉시 승인 상품 활용 중간 이니시스 제휴 조건 확인 필요
    심사 대기 중 테스트 결제 시뮬레이션 PG 심사 완료 전 샌드박스 환경에서 결제 플로우 체험 제공 낮음 실제 판매는 불가하나 셋업 완성도 체감 가능

    권장 접근: 단기적으로 계좌이체 즉시 활성화 + 테스트 결제 시뮬레이션을 기본 제공하고, 중기적으로 사전 심사 구조를 PG사와 협의해 "가입 즉시 카드결제 가능" 경험을 목표로 한다.

    6. 운영 방안¶

    6.1 신규 고객 온보딩¶

    • Self-serve 기본, 대형 고객은 전담 온보딩 지원
    • 일본 고객: 일본어 온보딩 가이드 + 일본어 CS 지원 (한국 본사 기반 운영, 필요 시 현지 방문)
    • 첫 30일 이내 이탈 방지: AI 기반 가이드·알림 자동화

    6.2 기존 StoreX 고객 마이그레이션¶

    • 신규 고객은 전량 누리메이트로 온보딩
    • 기존 고객은 안정성 검증 후 단계적 마이그레이션
    • 마이그레이션 도구 자동화 (상품·주문·고객 데이터 이전)
    • 전환 기간 중 병행 운영 지원

    6.3 VIP 고객사 운영¶

    • 대형 고객사: 전용 서버 완전 분리
    • 전담 계정 매니저 배정
    • SLA 별도 계약
