# 현지 PG사 직접 계약 필요성·조건·난이도 조사 [8-7]

> 조사일: 2026-03-10 | 목적: 누리메이트 시장별 결제 인프라 전략 수립

---

## 1. 현지 PG사 직접 계약이 필요한 이유

글로벌 PG(Stripe, PayPal 등) 단독 운영만으로는 아시아 주요 시장에서 결제 전환율을 최적화하기 어렵다. 현지 PG사 직접 계약이 필요한 주요 이유는 다음과 같다.

### 1-1. 현지 결제수단 지원

| 이유 | 설명 |
|------|------|
| 현지 주력 결제수단 미지원 | Stripe는 한국의 카카오페이·네이버페이·토스페이·계좌이체를 일부만 지원. 한국 발행 신용카드 직접 처리 불가 |
| 일본 편의점 결제(콘비니) | Stripe도 지원하나, 현지 PG(SBPS, GMO-PG)를 통할 때 더 많은 편의점 네트워크 커버 |
| 동남아 e-wallet | GoPay, OVO, TrueMoney, GCash 등은 해당 국가 현지 PG/결제 앱과만 직거래 가능 |
| 한국 신용카드 발급사 직접 처리 | 해외 법인 기반의 Stripe로는 한국 발행 카드 처리 시 수수료 상승 및 승인률 저하 |

### 1-2. 수수료 최적화

| 구분 | 글로벌 PG 단독 | 현지 PG 직접 계약 |
|------|---------------|-----------------|
| 카드 수수료 구조 | 크로스보더 수수료 포함 (일반적으로 1~2% 추가) | 국내 처리 수수료 적용 (더 낮음) |
| 환전 수수료 | 별도 발생 | 현지 통화 정산으로 최소화 |
| 최저 수수료율 | 협상 여지 제한적 | 거래량 기반 협상 가능 |

### 1-3. 규제 대응

- 일부 국가(인도네시아, 베트남 등)는 외국 PG사의 현지 결제 처리를 제한하거나, 현지 금융 라이선스를 요구
- 한국의 전자금융거래법은 PG업 등록 없이 자금을 경유·보유하는 구조를 제한
- 일본의 자금결제법(資金決済法)은 결제 서비스 사업자에 대한 등록·공탁 요건 부과

### 1-4. 소비자 신뢰 및 전환율

- 한국·일본 소비자는 낯선 결제 브랜드보다 **신뢰하는 현지 PG(이니시스, 토스페이먼츠, GMO-PG 등)** 결제창에서 전환율이 높음
- 현지 언어 결제창, 현지 CS 대응 가능 여부가 전환율에 직접 영향

### 2-1. 한국

#### 주요 현지 PG사

| PG사 | 시장 점유율 | 특징 |
|------|-----------|------|

| 토스페이먼츠 | 급성장 | 간편결제 통합, API 친화적 |
| NHN KCP | 상위권 | 카드·간편결제·해외 결제 |
| Nice Payments | 상위권 | 카드·계좌이체 |
| KSNET | 중위권 | 카드 중심 |

#### 계약 요건

| 항목 | 요건 |
|------|------|
| 법인 요건 | **국내 법인 원칙적 필요** (단, 외국 법인도 국내 영업소 설치 또는 위탁 대리인을 통해 계약 가능한 경우 있음) |
| 사업자등록증 | 필수 (국내 사업자 또는 외국법인 국내 지점) |
| 통신판매업 신고증 | 필수 (이커머스 사업자의 경우) |
| 개인정보처리방침 | 사이트에 게시 필수 |
| 보안 서버(HTTPS) | 필수 |
| 서비스 운영 실적 | 일부 PG사는 오픈 예정 쇼핑몰도 계약 가능 |
| 가입비 | PG사별 상이 (카페24 등 쇼핑몰 솔루션 통하면 면제·할인 가능) |
| 수수료율 | 카드 약 2~3%, 거래량 기반 협상 |
| 보증보험 또는 에스크로 | 선불 상품·고가 상품 취급 시 요구 가능 |

#### 난이도 평가: ★★☆☆☆ (낮음)

한국은 PG 가입 절차가 체계화되어 있고, 카페24·아임포트(포트원) 등 연동 솔루션이 발달해 있어 **국내 법인 기준으로는 진입 장벽이 낮다.** 단, 외국 법인의 직접 계약은 절차가 추가된다.

### 2-2. 일본

#### 주요 현지 PG사

| PG사 | 특징 |
|------|------|
| GMO Payment Gateway (GMO-PG) | 일본 1위, 매출 기준 압도적 점유율 |
| SB Payment Service (SBPS) | 소프트뱅크 자회사, Shopify Payments 일본 파트너 |
| Epsilon (GMO Epsilon) | 중소 쇼핑몰 특화, 후불 결제 강점 |
| Veritrans (DGフィナンシャルテクノロジー) | 정기결제·구독 강점 |
| KOMOJU | 일본·한국 결제 통합, 외국 사업자 친화적 |

#### 계약 요건

| 항목 | 요건 |
|------|------|
| 법인 요건 | **일본 현지 법인 원칙적 필요** (이사 1인 이상 일본 주소 보유 필요) |
| 법인등기부 등본 | 필수 |
| 신분증 (대표자) | 여권 또는 일본 국적자는 마이넘버 |
| 사업 계획서 | GMO-PG, SBPS 등 대형 PG는 사업 검토 필요 |
| 사이트 심사 | 운영 중인 사이트 또는 테스트 사이트 제출 필요 |
| 자금결제법 등록 | PG사가 등록된 사업자와 계약해야 하므로, 가맹점 자체 등록은 불필요 |
| 최소 거래량 | 대형 PG(GMO-PG, SBPS)는 최소 거래량 기준 또는 초기 기본료 부과 |
| 수수료 | 신용카드 약 3.0~3.6%, 콘비니 약 5~7% |
| 에스크로 | 선불 서비스·디지털 콘텐츠의 경우 선불 전자결제 등록 별도 필요 |

#### 외국 법인 대안

- **KOMOJU:** 일본·한국 결제를 모두 지원하며, 외국 법인도 계약 가능한 경우가 많음 (Shopify 앱으로도 연동 가능)
- **KG이니시스 일본 법인:** 2021년 일본 법인 설립 + GMO-PG 파트너십, 한국 사업자가 일본 결제를 처리하는 브릿지 서비스 제공

#### 난이도 평가: ★★★☆☆ (중간)

일본 현지 법인이 없으면 GMO-PG, SBPS 등 주요 PG와 직접 계약이 어렵다. 단, KOMOJU 같은 중간자 서비스를 활용하면 외국 법인도 일본 결제수단을 처리할 수 있다.

## 4. Stripe/PayPal 글로벌 PG 단독 운영의 한계와 현지 PG 병행의 구조적 복잡도

### 4-1. 글로벌 PG 단독 운영 한계

| 한계 | 세부 내용 |
|------|-----------|
| 국가 커버리지 갭 | 한국·베트남·말레이시아 등 주요 아시아 시장에서 사업자 계정 미지원 |
| 로컬 결제수단 미비 | e-wallet, QR 결제, 편의점 결제 등 현지 주력 수단 미지원 |
| 승인률 저하 | 크로스보더 처리 시 현지 은행 거부율 상승 |
| 높은 수수료 | 크로스보더 + 환전 수수료 이중 발생 |
| 규제 비대응 | 현지 금융 규제(Bank Indonesia, SBV 등) 미충족 |

### 4-2. 현지 PG 병행 시 구조적 복잡도

현지 PG 병행 운영 시 아래와 같은 복잡도가 추가된다:

| 복잡도 요소 | 설명 |
|------------|------|
| 다중 PG 통합 개발 | PG사별 API가 상이 → 개발 비용 증가 |
| 정산 일정 불일치 | PG사마다 정산 주기 다름 (D+1, D+2, D+7 등) |
| 복수 통화 정산 | KRW, JPY, THB 등 다통화 관리 |
| 이중 모니터링 | 결제 오류·사기 탐지를 PG사별로 별도 운영 |
| 계약·법무 관리 | 국가별 PG 계약서, 현지 법률 검토 필요 |
| 고객 서비스 복잡도 | 결제 오류 시 담당 PG 식별 및 CS 처리 분산 |

### 4-3. 통합 솔루션을 통한 복잡도 경감 옵션

| 솔루션 | 설명 | 커버 국가 |
|--------|------|-----------|
| **포트원(PortOne/아임포트)** | 한국·일본·동남아 다중 PG 통합 API | 한국, 일본, 인도네시아, 태국 등 |
| **Xendit** | 동남아 통합 결제 API (인도네시아·필리핀·베트남 등) | 동남아 6개국 이상 |
| **2C2P** | 동남아·한국·일본 결제 통합 | 아시아 전반 |
| **KOMOJU** | 일본·한국 결제 통합 (Shopify 앱 포함) | 일본, 한국 |
| **Paymentwall** | 200개국 150+ 결제수단 통합 | 글로벌 (로컬 결제 특화) |

## 5. 카페24·Shopify의 현지 PG 연동 방식 벤치마크

### 5-1. 카페24의 현지 PG 연동 전략

카페24는 단일 플랫폼 내에서 **다중 PG 병렬 연동**을 기본 아키텍처로 채택한다.

| 전략 요소 | 내용 |
|-----------|------|
| 국내 PG 다중 선택 | KG이니시스, 토스페이먼츠, KCP, KSNET, 다날, KG모빌리언스 등 동시 지원 |
| 해외 결제 전용 PG | 카페24 글로벌 페이먼츠, Paymentwall, Eximbay 별도 운영 |
| API 추상화 | 카페24 플랫폼 레이어에서 PG별 API를 추상화 → 가맹점은 단일 설정 인터페이스 |
| 가입 간소화 | 카페24를 통한 PG 신청 시 가입비 면제·심사 간소화 |

**글로벌 확장 구조:**
```
카페24 쇼핑몰 (한국 가맹점)
      ↓
카페24 글로벌 (해외 쇼핑몰 솔루션)
      ↓
카페24 글로벌 PG (해외 결제수단 처리)
  + Paymentwall (로컬 결제 보완)
```

### 5-2. Shopify의 현지 PG 연동 전략

Shopify는 **자체 Shopify Payments(Stripe 기반) + 앱 마켓플레이스를 통한 외부 PG 앱** 구조를 채택한다.

| 전략 요소 | 내용 |
|-----------|------|
| 1차 결제 레이어 | Shopify Payments (Stripe 기반, 40여개국 지원) |
| 2차 결제 레이어 | Shopify 앱 마켓의 외부 결제 앱 (KOMOJU, KG이니시스, Eximbay 등) |
| 수수료 구조 | Shopify Payments 사용 시 Transaction Fee 없음, 외부 PG 사용 시 0.5~2% 추가 |
| 한국 전략 | Shopify Payments 미지원 → KOMOJU/KG이니시스 앱으로 한국 결제 처리 |
| 일본 전략 | Shopify Payments(SBPS 기반) 기본 + PayPay/Rakuten 앱 추가 |

**Shopify 일본 결제 구조:**
```
Shopify 체크아웃
      ├── Shopify Payments (SBPS 기반) → 카드, 콘비니
      ├── PayPay 앱 → PayPay
      ├── Rakuten Pay 앱 → 라쿠텐 포인트 결제
      └── GMO Epsilon 앱 → 후불 결제
```

### 5-3. 누리메이트를 위한 벤치마크 교훈

| 교훈 | 카페24/Shopify 사례 | 누리메이트 적용 방안 |
|------|-------------------|---------------------|
| 플랫폼 레이어 추상화 | PG별 API를 플랫폼 레이어에서 통합 → 가맹점 부담 최소화 | 포트원·Xendit 같은 통합 PG API 레이어 도입 |
| 1차+2차 PG 이중 구조 | 글로벌 PG(기본) + 현지 PG(보완)의 투트랙 구조 | Stripe(글로벌 카드) + 현지 PG(로컬 결제수단) |
| 앱 생태계 활용 | 외부 결제 앱으로 PG 다양성 확보 | 누리메이트 파트너 PG 목록 확장 |
| 국가별 최소화 전략 | 모든 국가에 현지 PG 계약 대신 필요 국가만 집중 | Phase 1: 한국·일본 집중, Phase 2: 동남아 |

### 단계별 현지 PG 계약 전략

| 단계 | 국가 | 방법 | 비고 |
