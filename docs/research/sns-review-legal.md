# SNS 리뷰 수집 법적 검토 [6-9 조사]

> 조사일: 2026-03-10 | 목적: 누리메이트 SNS 리뷰 수집 기능 법적 리스크 사전 해소

## 1. SNS 게시물 무단 수집·재게시의 저작권법 위반 여부

### 1-1. 한국 저작권법

**원칙: 이용허락 없는 재게시 = 저작권 침해**
- 한국 저작권법은 이용허락을 대원칙으로 함
- SNS 게시물(사진, 영상, 텍스트)은 창작성이 인정될 경우 저작물로 보호
- 원저작자의 명시적 허락 없이 타 플랫폼에 재게시·임베딩하면 저작권 침해 성립
- **출처 표시만으로는 면책 불가** — 이는 광범위하게 오해되는 사항

**리그램(Regram) / 리포스트**
- 인스타그램 게시물을 무단 리그램하는 행위는 저작권 침해
- "좋아서 공유했다"는 사유는 법적 항변 사유가 되지 않음 (nepla.ai 판례 해설 참조)
- 원저작자의 댓글 허락, DM 허락 등 명시적 동의 확인 후 재게시 가능

**상업적 목적의 리뷰 수집·재게시**
- 누리메이트가 SNS 리뷰를 수집하여 상품 상세페이지에 게재하는 경우: 원저작자 동의 필수
- 저작물의 공정이용(저작권법 제35조의5) 항변 가능성 낮음 — 상업적 이익 취득 목적이 명백

**형사 처벌**
- 저작권법 제136조: 저작재산권 침해 시 5년 이하 징역 또는 5천만원 이하 벌금

### 1-2. 일본 저작권법

- 저작권법(著作権法) 제23조: 공중송신권. SNS 게시물을 무단 재게시·임베딩 시 위반
- 스크린샷 인용도 원칙적으로 인용 요건(제32조) 충족 여부 판단 필요
  - 인용 요건: 공표된 저작물, 정당한 범위, 출처 명시, 종속적 관계(주/종)
- 상업적 목적의 리뷰 수집은 인용 요건 충족 어려움

### 1-3. 글로벌 (미국·EU 기준)

**미국**
- 법원 판례: "공개 데이터 스크래핑은 계약 위반이지 범죄는 아님" (캘리포니아 연방법원, Meta 관련 판결)
- 퍼블릭 데이터(로그인 없이 접근 가능한 데이터) 스크래핑의 합법성 인정 추세
- 단, **저작권(Copyright)** 침해 여부는 별개 — 공개 데이터라도 저작물 재게시는 허락 필요
- DMCA(디지털 밀레니엄 저작권법): 저작권자의 삭제 요청(Takedown Notice) 시 즉각 대응 의무

**EU (GDPR)**
- 공개 SNS 게시물도 개인 식별 정보 포함 시 GDPR 적용
- 마케팅 목적의 수집은 명시적 동의(Explicit Consent) 필요

## 2. 플랫폼 ToS(이용약관)상 콘텐츠 크롤링·재게시 허용 범위

### 2-1. Instagram (Meta)

**스크래핑 금지 조항**
- Meta 이용약관 명시: "You can't attempt to create accounts or access or collect information in unauthorized ways. This includes creating accounts or collecting information in an automated way without our express permission."
- 봇·크롤러를 통한 자동 수집 = 약관 위반

**공개 데이터 스크래핑의 법적 위치**
- 약관 위반은 민사 계약 분쟁 (형사 범죄 아님)
- 로그인 불필요한 공개 데이터: 법적으로 수집 가능성 있으나, 약관 위반으로 계정 차단 위험 존재
- 인증 우회 및 기술적 보호조치 무력화: 위법 (CFAA 해당, 한국은 정보통신망법 적용)

**공식 Graph API 활용의 경우**
- Meta가 허용한 공식 API를 통한 수집은 약관 준수
- 단, API 이용 약관상 수집 데이터를 제3자 서비스에서 재게시하는 것은 별도 제한 적용

### 2-2. X (Twitter)

**API 정책 변경 (2023~현재)**
- 2023년 이후 API 유료화 및 대규모 스크래핑 사실상 차단
- Basic API($100/월): 읽기 10,000건/월. 상업적 리뷰 수집에는 불충분
- 트윗 재게시: X의 임베딩 기능(공식 oEmbed API)을 통한 임베딩은 허용. 직접 콘텐츠 복사·재게시는 약관 위반

### 2-3. TikTok

**콘텐츠 크롤링 금지**
- TikTok 이용약관: 자동화된 방식의 데이터 수집 금지
- 공식 Content Posting API는 콘텐츠 게시용이며, 타인 게시물 수집 API는 별도 없음
- TikTok 영상을 누리메이트 페이지에 임베딩하는 경우: 공식 임베딩 코드 사용 시 허용

### 2-4. 플랫폼 ToS 위반의 실질적 리스크

| 위반 행위 | 민사 리스크 | 형사 리스크 | 계정 리스크 |
|-----------|------------|------------|------------|
| 무단 크롤링 | 손해배상 청구 가능 | 정보통신망법 위반 가능 | 계정·IP 차단 |
| 재게시 (허락 없음) | 저작권 침해 손해배상 | 저작권법 형사처벌 | 게시물 삭제 요청 |
| 공식 API 이용 약관 위반 | API 액세스 취소 | 해당 없음 | 앱 삭제 |

## 3. 법적 리스크 완화 방책

### 3-1. 원작자 동의 획득 방식

**방법 1: 해시태그 기반 자발적 동의 (Hashtag UGC Permission)**
- 특정 해시태그(예: `#누리메이트리뷰`) 사용 시 리뷰 재게시에 동의함을 약관에 명시
- 캠페인 랜딩페이지에서 약관 동의 후 해시태그 안내
- 법적 효력: 약관 동의 + 행위(해시태그 사용)의 조합으로 암묵적 동의 성립 가능하나, **명시적 동의보다 법적 효력 약함**

**방법 2: DM/댓글 기반 명시적 동의**
- 브랜드 관련 게시물 작성자에게 DM을 통해 재게시 동의 요청
- 동의 메시지 스크린샷 또는 로그 보관
- 가장 확실하지만 운영 비용 높음

**방법 3: 공식 임베딩만 사용**
- 원본 SNS 게시물을 복사하지 않고 플랫폼 공식 oEmbed API로 임베딩
- 저작권 침해 리스크 최소화 (원본 플랫폼에서 콘텐츠 제공)
- 단, 게시물 삭제 시 임베딩 콘텐츠 함께 소실되는 문제 존재

**방법 4: 리뷰 자발적 기고 시스템 구축**
- 누리메이트 자체 인터페이스에서 SNS 게시물 URL을 사용자가 직접 제출
- 제출 시 이용약관 동의로 재게시 권한 확보
- 법적으로 가장 안전한 방법

### 3-2. 출처 표시 의무

- 재게시 시 반드시 원작자 계정 태그 및 원본 URL 링크 포함
- 출처 표시가 법적 면책 사유는 아니나, 리스크 경감 및 원작자 우호 관계 유지에 기여

### 3-3. 삭제 요청 대응 프로세스

**필수 운영 절차**
1. 원작자 또는 플랫폼으로부터 삭제 요청 접수 채널 마련 (이메일, 신고 버튼)
2. 접수 후 **24시간 이내 검토, 72시간 이내 처리** 원칙 (DMCA Notice 대응 기준 준용)
3. 처리 결과 요청자에게 회신
4. 삭제 처리 내역 로그 보관 (분쟁 시 증거)

**한국 정보통신망법 제44조의2 준수**
- 타인의 정보 삭제·차단 요청 시 즉시 임시조치 의무
- 미이행 시 서비스 제공자의 법적 책임 가중

## 4. Instagram Graph API 활용 범위

### 4-1. 브랜드 해시태그 기반 게시물 탐지

**기능 및 한계**
- 해시태그 검색 API: 특정 해시태그의 최근 공개 게시물 탐지 가능
- **주간 30개 해시태그** 제한 (7일 주기 리셋)
- 시간당 200 요청/계정 제한
- 과거 데이터 접근 제한: 최근 수일~수주 이내 게시물만 조회 가능

**활용 방법**
```
GET /ig-hashtag-search?user_id={IG_USER_ID}&q={hashtag}
→ hashtag_id 획득

GET /{hashtag_id}/recent_media?user_id={IG_USER_ID}&fields=id,media_type,media_url,permalink,timestamp
→ 최근 게시물 목록 수신
```

**중요 제약**: API로 수집한 데이터를 제3자 서비스(누리메이트)에서 재게시하는 것은 Instagram Platform Policy 위반 여지 있음 → 임베딩 방식 권장

### 4-2. 멘션 기반 게시물 탐지

**기능**
- Business Discovery API: 비즈니스 계정의 게시물 탐색
- Mentions API: 브랜드 계정이 태그된 게시물·댓글 수집
- **주간 제한 없음** — 해시태그 API보다 안정적

**활용 시나리오**
- 브랜드 공식 인스타 계정에 태그(@브랜드명)된 게시물 자동 수집
- 수집된 게시물 임베딩 형태로 상품 페이지에 노출
- 원작자 동의 없는 재게시 방지를 위해 임베딩 코드만 사용

## 5. Instagram Messaging API를 통한 DM 발송의 법적·정책적 허용 범위

### 5-1. Meta 정책 기준

**24시간 메시지 창(24-Hour Window)**
- 사용자가 브랜드 계정에 먼저 메시지 발송 시 24시간 창 열림
- 창 내: 프로모션 포함 무제한 메시지 발송 가능
- 창 종료 후: 메시지 태그(Message Tags) 사용 시에만 발송 가능
  - 단, 메시지 태그로는 **프로모션 콘텐츠(쿠폰, 할인 등) 발송 불가**

**리뷰 요청 DM 자동화의 허용 범위**
- 사용자가 먼저 메시지를 보낸 후 24시간 이내: 리뷰 작성 요청 DM 발송 가능
- 사용자 행동(댓글, 특정 키워드 DM) 기반 트리거 자동화: 허용
- 일방적 대량 DM 발송(사용자 사전 접촉 없음): **명백한 정책 위반**

### 5-2. 법적 기준 (한국)

**정보통신망법 제50조 (스팸 금지)**
- 수신자 사전 동의 없는 상업적 정보 전송 금지
- 위반 시: 3천만원 이하 과태료
- DM으로 리뷰 작성 요청 시 해당 사용자가 브랜드 팔로워이거나 구매 이력 있는 경우도 별도 동의 필요 여부 검토 필요

**미국 CAN-SPAM Act / TCPA**
- 자동화 DM이 상업적 목적일 경우 CAN-SPAM Act 적용
- TCPA: 자동화 메시지 발송 전 서면 동의 필요 (소셜 미디어 메시징에 확대 적용 추세)

### 5-3. 누리메이트 DM 자동화 권고 기준

- 리뷰 요청 DM 발송 조건: 사용자가 브랜드 계정에 먼저 연락하거나, 특정 캠페인 참여를 통해 명시적 동의를 확인한 경우에 한정
- "구매 완료 후 자동 DM" 방식은 사전 동의 없는 경우 법적 리스크 존재
- 권장 방식: 주문 완료 페이지에서 "인스타그램으로 리뷰 공유하기" 버튼 제공 → 사용자 자발적 액션 유도

## 6. 한국 정보통신망법·개인정보보호법 관련 유의사항

### 6-1. 개인정보보호법

**SNS 리뷰 수집 시 개인정보 해당 여부**
- 리뷰 작성자의 인스타그램 계정명, 프로필 사진, 위치 정보 등은 개인정보에 해당 가능
- 수집 시 **이용 목적, 항목, 보유 기간, 동의 거부 권리** 고지 및 동의 필요

**2025년 개인정보보호법 개정 핵심사항 (3차 개정안)**
- 개인정보 처리 목적 명확화 의무 강화
- 익명처리 가능한 경우 익명처리 우선 원칙
- 가명처리 시 목적 외 활용 제한

**실무 체크리스트**
- [ ] SNS 리뷰 수집·재게시에 대한 개인정보 처리방침 수립
- [ ] 리뷰어 정보(계정명 등) 수집 동의 획득 절차 마련
- [ ] 개인정보 보유 기간 및 삭제 정책 명시
- [ ] 수집 목적 외 활용 금지

### 6-2. 정보통신망법

**제44조의2 (임시조치 의무)**
- 타인의 권리를 침해하는 정보 삭제 요청 시 즉시 임시조치 실시 의무
- 누리메이트가 플랫폼으로서 UGC를 게시하는 경우 해당

**제50조 (스팸 금지)**
- 수신 동의 없는 영리 목적 정보 전송 금지 (DM 마케팅 관련)

**제48조 (악성 프로그램 유포 금지)**
- 웹 스크래핑 봇 운영이 기술적 보호조치 우회 해당 시 적용 가능

### 6-3. 실무 적용

- SNS 리뷰 임베딩 방식 채택 시: 원본 플랫폼에서 데이터 렌더링되므로 개인정보 직접 수집·저장 최소화
- 리뷰어 계정명 등을 DB에 저장하는 경우: 개인정보처리방침에 명시, 동의 절차 필수

## 7. 일본 개인정보보호법(APPI) 관련 유의사항

### 7-1. APPI 기본 구조

- 일본 個人情報の保護に関する法律 (APPI)
- 적용 대상: 일본 내 정보주체의 개인정보를 처리하는 모든 사업자 (역외 적용)
- 누리메이트가 일본 사용자 SNS 리뷰를 수집하는 경우 APPI 적용

### 7-2. 2022년 개정 강화사항

**개인관련정보(個人関連情報) 신설**
- 쿠키, 행동이력, SNS 계정 연계 정보 등을 개인관련정보로 규정
- 제3자에게 개인관련정보를 제공 시 정보주체 동의 필요
- SNS 게시물을 브랜드 페이지에 연계·표시하는 행위가 해당할 수 있음

**제3자 제공 시 기록 보관 의무**
- 기업 간 데이터 연계 또는 외부 광고 플랫폼 이용 시 기록 관리 의무화
- 누리메이트가 일본 사용자 리뷰를 수집하여 브랜드에 제공하는 경우 해당

### 7-3. 스테마(스텔스 마케팅) 규제 (2023년 10월~)

- 景品表示法(경품표시법) 개정으로 스테마 금지 조항 신설
- 체험단·협찬 후기임을 표시하지 않는 광고 게시: 행정처분 대상
- 한국 공정위 지침과 유사하나 일본은 사업자(광고주) 책임도 직접 규정

### 7-4. 실무 체크리스트 (일본)

- [ ] 일본 사용자 SNS 리뷰 수집 전 개인정보 취급 방침(日本語) 마련
- [ ] 개인관련정보 제3자 제공 동의 절차 수립
- [ ] 제3자 제공 기록 보관 체계 구축
- [ ] 체험단·협찬 표시 「PR」「広告」「#AD」 등 명시 가이드라인 브랜드에 제공
- [ ] 일본 법무 전문가 검토 (APPI 역외 적용 구체적 범위 확인)

## 8. 종합 법적 리스크 매트릭스 및 권고사항

| 수집 방식 | 저작권 리스크 | 개인정보 리스크 | ToS 리스크 | 권고 |
|-----------|-------------|----------------|-----------|------|
| 무단 크롤링 후 재게시 | 매우 높음 | 높음 | 매우 높음 | 절대 금지 |
| 공식 API 수집 후 재게시 | 중간 | 중간 | 중간 | 임베딩으로 대체 |
| 공식 API 수집 + oEmbed 임베딩 | 낮음 | 낮음 | 낮음 | 권장 |
| 해시태그 캠페인 + 자발적 제출 | 매우 낮음 | 낮음 | 없음 | 가장 권장 |
| 사용자 직접 제출 (URL 입력) | 없음 | 없음 | 없음 | 최우선 권장 |

### 최종 권고사항

1. **SNS 리뷰 수집의 기본 원칙**: "수집"보다 "자발적 기고" 유도로 패러다임 전환
2. **임베딩 우선**: 콘텐츠 복사·저장 없이 원본 플랫폼 임베딩 코드만 사용
3. **해시태그 캠페인 약관 명시**: 브랜드 전용 해시태그 사용 = 재게시 동의로 간주하는 약관 명시 + 법무 검토
4. **삭제 요청 SLA**: 접수 24시간, 처리 72시간 이내 완료 프로세스 표준화
5. **일본 시장**: APPI 역외 적용 대응 및 스테마 표시 가이드라인 브랜드에 의무 제공
6. **법무 자문**: SNS 리뷰 수집 기능 출시 전 한국·일본 법무법인 리뷰 필수
