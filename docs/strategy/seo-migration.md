# 셀메이트CM 마이그레이션 SEO 보존 전략

> 작성일: 2026-04-06 | 버전: v1.0
> 참고 내부 문서: `internal/seo-migration.md`

---

## 개요

카페24 등 기존 이커머스 플랫폼에서 셀메이트CM으로 이전하는 판매자는 **축적된 검색 인덱싱(SEO 자산)을 잃을 위험**이 있다. 이 문서는 마이그레이션 시 SEO 손실을 최소화하는 방법을 정리하고, 셀메이트CM이 지원해야 할 기능 수준을 정의한다.

---

## 1. 핵심 전제: 도메인 소유권

SEO 보존 가능 여부는 **도메인 소유권**에 따라 결정된다.

| 상황 | 처리 주체 | 셀메이트CM 승인 필요 여부 |
|------|---------|----------------------|
| 판매자 자체 도메인 사용 (`myshop.co.kr`) | 판매자 + 셀메이트CM | ❌ 불필요 — DNS 이전 후 셀메이트CM에서 직접 처리 |
| 카페24 서브도메인 (`xxx.cafe24.com`) | 카페24 | ❌ 셀메이트CM 처리 불가 — 카페24 서버를 경유하므로 원플랫폼 협조 필요 |

> **실무 의미**: 대부분의 운영 중인 쇼핑몰은 자체 도메인을 사용하므로, 셀메이트CM에서 100% 처리 가능하다. 카페24 서브도메인만 사용하는 신규 창업자 등은 이전 전 자체 도메인 연결을 유도해야 한다.

## 2. 301 리다이렉트 처리

### 2.1 원리

DNS를 셀메이트CM 서버로 전환하면, 구글·네이버에 인덱싱된 기존 URL 요청이 셀메이트CM 서버로 유입된다. 이 시점에 **301(영구 이동) 응답**을 반환하면 검색엔진이 새 URL로 인덱스를 점진적으로 갱신한다.

```
[구글/네이버 인덱싱 URL]
https://myshop.co.kr/product/나이키-운동화/12345
          ↓ DNS 이전 후
[셀메이트CM 서버가 요청 수신]
          ↓ 301 리다이렉트 반환
https://myshop.co.kr/products/12345
```

### 2.2 카페24 → 셀메이트CM URL 패턴 매핑

| 카페24 URL 패턴 | 셀메이트CM URL 패턴 | 비고 |
|---------------|-----------------|-----|
| `/product/{상품명}/{ID}` | `/products/{handle}` | ID 또는 slug 기반 |
| `/category/{카테고리명}/{ID}` | `/collections/{handle}` | |
| `/board/review/{글번호}` | `/pages/reviews` 또는 `/products/{handle}#reviews` | |
| `/board/faq/{글번호}` | `/pages/faq` | |
| `/member/login.html` | `/account/login` | |
| `/order/order_list.html` | `/account/orders` | |

### 2.3 셀메이트CM 기능 수준 정의

**P0 (마이그레이션 도구 필수 기능):**
- 어드민 UI에서 리다이렉트 규칙 등록 (URL 단위)
- CSV 대량 업로드 지원 (최대 10만 건)
- 정규식 패턴 매칭 — 카페24 URL 구조 자동 파싱 → 규칙 자동 생성

**P1:**
- Google Search Console 연동 → 현재 인덱싱된 URL 자동 import
- 301 적용 후 크롤링 상태 모니터링 (처리율 대시보드)

**서버 구현 방식 (헤드리스 구조):**
```
# Nginx 레벨 리다이렉트 (인프라 처리)
rewrite ^/product/([^/]+)/([0-9]+)$ /products/$2 permanent;
rewrite ^/category/([^/]+)/([0-9]+)$ /collections/$2 permanent;

# Next.js 헤드리스 프론트 (next.config.ts)
redirects: [
  { source: '/product/:name/:id', destination: '/products/:id', permanent: true },
  { source: '/category/:name/:id', destination: '/collections/:id', permanent: true },
]
```

## 3. 메타 데이터 이전

### 3.1 이전 대상

| 메타데이터 항목 | 카페24 → 셀메이트CM 이전 방식 |
|--------------|--------------------------|
| 상품 페이지 title / meta description | 상품 API 마이그레이션 시 필드로 포함 |
| Open Graph 태그 (og:title, og:image) | 상품 이미지·설명 기반 자동 생성 후 수동 편집 가능 |
| 구조화 데이터 (JSON-LD: Product, BreadcrumbList) | 셀메이트CM 표준 템플릿 자동 생성 |
| Canonical URL | DNS 이전 완료 후 새 도메인 기준 자동 설정 |
| hreflang (다국어) | 일본어 스토어 개설 시 `ko` / `ja` 태그 자동 처리 |

### 3.2 마이그레이션 도구 요구사항

**P0:**
- 카페24 API에서 상품별 meta title/description 가져오기 (카페24 API 스펙 확인 필요)
- 셀메이트CM 상품 레코드에 SEO 필드 별도 구분 저장

**P1:**
- 메타 데이터 일괄 편집 UI (어드민 → 상품 목록에서 SEO 미입력 항목 필터)
- AI 자동 생성 옵션 (상품명 + 설명 기반 GPT/Claude 활용)

## 4. Sitemap 처리

### 4.1 기본 원칙

DNS 이전 후 셀메이트CM은 **자동으로 sitemap.xml을 생성·갱신**해야 한다. 검색엔진이 새 URL 구조를 신속히 인덱싱하도록 유도하는 핵심 도구다.

### 4.2 셀메이트CM Sitemap 생성 사양

| 항목 | 내용 |
|------|------|
| 위치 | `https://{판매자도메인}/sitemap.xml` |
| 자동 갱신 | 상품·컬렉션·페이지 추가·수정·삭제 시 자동 반영 |
| 포함 URL 유형 | 상품 상세, 컬렉션, 정적 페이지 (공지, FAQ 등) |
| 제외 URL | 마이페이지, 장바구니, 주문 완료 등 non-indexable 페이지 |
| 업데이트 주기 표시 | `<changefreq>` — 상품 `weekly`, 컬렉션 `monthly` |
| 다국어 스토어 | `<xhtml:link>` 태그로 언어별 URL 연결 |

### 4.3 마이그레이션 단계별 조치

1. **DNS 이전 전**: 기존 sitemap.xml에서 인덱싱된 URL 목록 백업
2. **DNS 이전 직후**: 셀메이트CM에서 신규 sitemap.xml 즉시 생성
3. **Google Search Console 제출**: 새 sitemap URL 수동 제출 (크롤링 가속)
4. **네이버 웹마스터 도구**: 동일하게 sitemap 제출
5. **모니터링 기간**: 이전 후 3개월간 인덱스 갱신 상태 추적

## 5. SEO 마이그레이션 체크리스트

마이그레이션 도구 내 판매자가 직접 확인하는 셀프서비스 체크리스트:

```
□ 자체 도메인 보유 확인 (cafe24.com 서브도메인 사용 중 → 도메인 구입 안내)
□ Google Search Console에서 현재 인덱싱 URL 목록 export
□ URL 패턴 매핑 자동 분석 (마이그레이션 도구 실행)
□ 301 리다이렉트 규칙 검토·확정
□ 상품 메타 데이터 이전 확인 (누락 항목 AI 자동 생성)
□ DNS 이전 실행
□ Sitemap 생성 확인 (https://{도메인}/sitemap.xml 접근)
□ GSC/네이버 웹마스터 도구에 신규 sitemap 제출
□ 주요 URL 301 응답 수동 확인 (상위 10개 페이지)
```

## 6. 경쟁사 비교

| 기능 | Shopify | 카페24 | 아임웹 | 셀메이트CM |
|------|---------|-------|-------|----------|
| 어드민 URL 리다이렉트 UI | ✅ CSV 10만 건 | ❌ | △ 제한적 | ✅ P0 목표 |
| 정규식 패턴 매칭 | △ 앱 필요 | ❌ | ❌ | ✅ P0 목표 |
| GSC 연동 인덱스 import | ❌ | ❌ | ❌ | ✅ P1 목표 |
| Sitemap 자동 생성 | ✅ | ✅ | ✅ | ✅ P0 목표 |
| 메타 데이터 AI 자동 생성 | △ (앱) | ❌ | ❌ | ✅ P1 목표 |
| 마이그레이션 리다이렉트 자동 파싱 | ❌ | ❌ | ❌ | ✅ P0 목표 |

> Shopify의 URL 리다이렉트가 가장 성숙하나 카페24 URL 구조 자동 파싱은 없다. 셀메이트CM은 **카페24 전용 자동 파싱**을 차별점으로 설계한다.

## 7. 주의사항

- **301은 영구 이동 신호**: 검색엔진이 새 URL로 인덱스를 갱신하는 데 **1~3개월 소요**. 마이그레이션 후 최소 3개월간 리다이렉트 유지 필수.
- **카페24 서브도메인 판매자**: 이전 전 자체 도메인 구입·연결 필요. 마이그레이션 도구 시작 단계에서 서브도메인 사용 여부 감지 → 안내 메시지 노출.
- **URL slug 변경 최소화**: 마이그레이션 시 상품 URL handle을 카페24 ID와 동일하게 유지하면 리다이렉트 없이 SEO 보존 가능. 가능하면 slug 유지 권장.
