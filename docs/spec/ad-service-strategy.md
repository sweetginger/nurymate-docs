## 1. 개요 및 전략 방향

### 1.1 왜 광고 부가서비스인가

누리메이트 셀러의 최대 고민은 "만들었는데 어떻게 팔지"다. 쇼핑몰을 개설한 셀러 중 다수가 첫 30일 내 유료 광고를 집행하지 못해 이탈한다. 플랫폼이 광고 실행 진입 장벽을 낮춰주면:

- 셀러 첫 매출 달성률 상승 → 플랫폼 잔존율 개선
- 광고 집행액 증가 → 수수료 수익 신규 스트림 확보
- 셀러가 광고를 플랫폼 안에서 관리 → 데이터 락인(lock-in) 효과

### 1.2 전략 방향: 자체 개발 NO, 파트너 API 연동 YES

| 항목 | 자체 개발 | 파트너 API 연동 (채택) |
|------|---------|-------------------|
| 초기 개발 비용 | 억 단위 | 수천만 원 수준 |
| 각 플랫폼 정책 변경 대응 | 내부 공수 발생 | 파트너 책임 |
| 광고 전문성 | 내재화 필요 | 파트너 활용 |
| 수익 구조 | 광고 마진 100% | 수수료 % 수취 (마진 낮지만 리스크 낮음) |
| 확장성 | 플랫폼별 재개발 | 파트너 추가만으로 확장 |

**결론**: 광고 기술 자체를 소유하지 않고, 광고 대행사·애드테크 파트너에게 누리메이트 데이터 접근 API를 제공하고 수수료를 수취하는 **마켓플레이스 모델**로 운영한다.

### 1.3 광고 부가서비스 전체 구조

```
셀러
  │
  ├─ (A) 앱스토어에서 광고 앱 설치
  │       └─ 앱 = 광고 대행사/애드테크가 만든 앱
  │           ├─ 누리메이트 API로 상품/주문/고객 데이터 수신
  │           └─ Meta·Google·Kakao·Naver에 광고 집행
  │
  └─ (B) 광고 성과 누리메이트 대시보드에서 통합 확인
          └─ ROAS, CPC, 전환 등 채널별 비교

수익 흐름
  셀러 광고비 → 광고 플랫폼(Meta/Google 등)
  셀러 앱 이용료 → 광고 대행사/앱 파트너
  수수료(GMV % 또는 앱 매출 %) → 누리메이트
```

## 2. 광고 대행사 파트너 연동 구조

### 2.1 누리메이트 API 공개 범위

광고 파트너가 접근 가능한 데이터 범위를 API로 제공한다. 셀러가 앱을 설치하면 OAuth 기반 동의를 거쳐 데이터 접근 권한이 부여된다.

| API 범주 | 제공 데이터 | 활용 목적 |
|---------|-----------|---------|
| **상품 피드 API** | 상품명, 이미지, 가격, 재고, 카테고리, SKU | Meta/Google 쇼핑 광고용 카탈로그 자동 동기화 |
| **주문 데이터 API** | 주문 수, 매출액, 상품별 판매량, 전환 이벤트 | 전환 추적, ROAS 계산, 광고 최적화 |
| **고객 세그먼트 API** | 구매 빈도, 평균 객단가, 최근 구매일, 이탈 여부 | 리타겟팅 오디언스 생성, 유사 타겟 광고 |
| **픽셀 이벤트 API** | PageView, AddToCart, Purchase 이벤트 | Meta Pixel·Google Tag 이벤트 자동 심기 |
| **스토어 메타데이터** | 스토어 URL, 로고, 업종 카테고리 | 광고 소재 자동 생성 |

**API 접근 등급**

| 등급 | 접근 범위 | 대상 파트너 |
|------|---------|-----------|
| Basic | 상품 피드 + 스토어 메타데이터 | 무료 앱 파트너 |
| Standard | Basic + 주문 데이터 + 픽셀 이벤트 | 수수료 계약 파트너 |
| Premium | Standard + 고객 세그먼트 | 전략적 파트너 (계약 필수) |

> 개인정보 처리: 고객 세그먼트 API는 개인 식별 정보(이름, 연락처)를 제외한 행동 데이터만 제공. 셀러 동의 및 개인정보처리방침 고지 필수.

### 2.2 수수료 스플릿 구조

광고 파트너와 누리메이트 간 수익 배분 모델은 파트너 유형에 따라 두 가지로 운영한다.

#### 모델 A: 광고 집행액 기준 % 수취 (주력 모델)

```
셀러가 집행한 광고비 100만 원 기준

Meta/Google 등 미디어 플랫폼   →  셀러 광고비의 70~80% (미디어 구매 비용)
광고 대행사/앱 파트너           →  셀러 광고비의 15~25% (서비스 수수료)
누리메이트                     →  파트너 수수료의 10~20% 수취 (파트너 계약에 따라 차등)
```

**적용 조건**:
- 누리메이트 API를 통해 생성·집행된 광고에 한해 추적
- 파트너 앱 내 픽셀 이벤트 기반으로 집행액 산정
- 월 정산, 파트너사 인보이스 기준

#### 모델 B: 앱 월정액 수익 쉐어 (보조 모델)

셀러에게 앱 이용료(월 9,900원~49,000원)를 부과하는 SaaS형 파트너의 경우, 앱 매출의 **20~30%를 누리메이트가 수취**한다 (앱스토어 수수료 모델).

| 앱 월정액 | 누리메이트 수취 | 파트너 수취 |
|---------|-------------|----------|
| 9,900원 | 2,000원 (20%) | 7,900원 |
| 29,000원 | 7,250원 (25%) | 21,750원 |
| 49,000원 | 14,700원 (30%) | 34,300원 |

#### 모델 선택 기준

| 파트너 유형 | 권장 모델 | 이유 |
|-----------|---------|------|
| 퍼포먼스 마케팅 대행사 | 모델 A | 광고 집행액 연동 수익 |
| 애드테크 스타트업 (SaaS형) | 모델 B | 앱 구독 수익 쉐어 |
| 전략적 대기업 파트너 | A+B 혼합 | 규모에 따라 협의 |

### 2.3 대행사 온보딩 프로세스

#### 파트너 신청 ~ 앱 출시까지 (목표: 6주)

```
Week 1-2: 파트너 심사
  ├─ 파트너 신청 (비즈니스 인증, 서비스 소개)
  ├─ 누리메이트 파트너팀 심사 (역량, 레퍼런스, 법적 적격성)
  └─ 수수료 조건 협의 및 파트너 계약 체결

Week 3-4: 기술 연동
  ├─ API 키 발급 (Sandbox 환경)
  ├─ 누리메이트 API 문서 공유 + 기술 지원
  ├─ Sandbox 테스트 및 QA
  └─ Production API 키 발급

Week 5-6: 앱스토어 등록
  ├─ 앱 메타데이터 등록 (설명, 스크린샷, 요금제)
  ├─ 누리메이트 앱스토어 심사 (기능·UI·보안)
  └─ 앱 출시 및 마케팅 협력 시작
```

#### 파트너 지원 자료

- API 레퍼런스 문서 (개발자 포털)
- 파트너 대시보드 (설치 셀러 수, 매출, 수수료 정산 현황)
- 파트너 슬랙 채널 (기술 지원)
- 분기별 파트너 미팅 (로드맵 공유, 피드백 수렴)

## 4. 팀 구조 설계

### 4.1 현재 팀 구조 문제

광고·부가서비스 기능을 커머스 코어 팀이 함께 담당할 경우:
- 커머스 코어(결제, 주문, 배송)의 런칭 일정 압박이 광고 로드맵을 후순위로 밀어냄
- 광고 플랫폼(Meta/Google) API 정책 변경 대응 속도 저하
- 파트너 온보딩·영업 업무가 기술팀 리소스를 잠식

### 4.2 분리 운영 방안: 2팀 체제

#### 팀 A: 커머스 코어 팀

**미션**: 누리메이트 플랫폼 기반 기능 (스토어, 결제, 주문, 배송, AI) 개발·운영

| 역할 | 인원 | 담당 |
|------|------|------|
| 엔지니어링 리드 | 1 | 백엔드 아키텍처, 코어 API 설계 |
| 백엔드 개발자 | 2~3 | 결제·주문·배송·AI 기능 |
| 프론트엔드 개발자 | 1~2 | 셀러 어드민, 스토어 빌더 |
| PM | 1 (유림) | 코어 로드맵 관리 |
| 디자이너 | 1 | UX/UI |

**주요 산출물**: 결제 인프라, 주문 관리, 배송 연동, AI 어시스턴트, 셀러 대시보드

#### 팀 B: 광고·부가서비스 팀

**미션**: 앱스토어 생태계 구축, 광고 파트너 온보딩, 부가서비스 수익화

| 역할 | 인원 | 담당 |
|------|------|------|
| 팀 리드 (개발+비즈) | 1 | 팀 운영, 파트너 전략 |

| BizDev / 파트너 매니저 | 1 | 파트너 발굴·계약·온보딩 |

**주요 산출물**: 파트너 API, 앱스토어 플랫폼, 파트너 대시보드, 수수료 정산 시스템

### 4.3 팀 분리 전환 시점

| 단계 | 시기 | 팀 구조 |
|------|------|---------|
| Phase 0 (현재~6월) | 2026 H1 | 단일 팀 — 코어 개발 집중, 광고 기능 최소화 |
| Phase 1 (6월 런칭 후) | 2026 Q3 | 코어팀에서 1명 광고팀으로 전환 시작 |
| Phase 2 (본격 수익화) | 2026 Q4 | 팀 B 공식 분리, 파트너 매니저 채용 |
| Phase 3 (스케일업) | 2027+ | 팀 B 독립 P&L 운영 가능 수준 확장 |

> Phase 0~1 기간: 코어 런칭이 최우선. 광고 관련 개발은 파트너 API 기반 설계(코어팀 1인 담당)에 집중하여 Phase 2 이후 팀 분리 시 전환 비용 최소화.

## 5. 수익 모델 및 예상치

### 5.1 수익 스트림 정의

| 수익 유형 | 구조 | 수취 시점 |
|---------|------|---------|
| 파트너 API 수수료 (모델 A) | 파트너 앱 수수료 수익의 10~20% | 월 정산 |
| 앱스토어 수수료 (모델 B) | 앱 월정액 매출의 20~30% | 월 정산 |
| 광고 집행액 직접 쉐어 | 파트너 통해 집행된 광고비의 2~5% | 분기 정산 |

### 5.2 대행사 연동 수수료율 상세

| 파트너 규모 | 누리메이트 수취율 | 조건 |
|-----------|---------------|------|
| 소형 파트너 (월 광고집행 5천만↓) | 15% (앱 매출 기준) | 기본 계약 |
| 중형 파트너 (월 광고집행 5천만~3억) | 12% | 전략 파트너 계약 |
| 대형 파트너 (월 광고집행 3억↑) | 8~10% | 개별 협의 (MG 조건 가능) |

> MG(Minimum Guarantee): 대형 파트너의 경우 월 최소 보장 수수료 조건으로 계약 가능

### 5.3 수익 예상치 시뮬레이션 — 셀러 100명 기준

**가정**:
- 활성 셀러 100명 (광고 앱 최소 1개 설치 기준)
- 셀러 월 평균 광고비: 100만 원
- 앱 이용률 분포: 무료 앱 40%, 유료 앱 60%
- 유료 앱 평균 월정액: 19,800원
- 파트너 수수료 중 누리메이트 수취: 평균 15% (앱 매출 기준)

```
[시나리오 1] 앱스토어 수수료 모델 (모델 B 기준)

  유료 앱 사용 셀러: 100명 × 60% = 60명
  월 앱 매출: 60명 × 19,800원 = 1,188,000원
  누리메이트 수취 (20%): 237,600원 / 월

  ※ 초기 단계이므로 미미한 수준 — 셀러 수 증가가 핵심
```

```
[시나리오 2] 광고 집행액 수수료 모델 (모델 A 기준)

  셀러 100명 × 월 광고비 100만 원 = 1억 원 (총 광고집행액)
  파트너 앱 수수료 (광고비의 20%): 2,000만 원
  누리메이트 수취 (파트너 수수료의 15%): 300만 원 / 월
  연간: 3,600만 원
```

```
[시나리오 3] 셀러 규모별 성장 예상 (모델 A 기준)

  셀러 수    월 광고집행액(추정)    누리메이트 월 수익
  ─────────────────────────────────────────────────
  100명      1억 원               300만 원
  500명      5억 원              1,500만 원
  1,000명   10억 원              3,000만 원
  3,000명   30억 원              9,000만 원
  ─────────────────────────────────────────────────
  * 셀러 평균 광고비 100만 원, 파트너 수수료 20%, 누리메이트 수취 15% 기준
```

### 5.4 손익분기점 분석
