## 1. 개요 및 목표

인스타그램 등 SNS에 게시된 상품 리뷰(UGC)를 자사몰 상품 페이지에 자동 수집·표시하는 기능.
**법적 리스크 선제 해소**를 위해 원작자 동의 획득 자동화 플로우를 필수로 포함한다.

### 1.1 핵심 원칙

- **동의 없이 재게시 절대 금지**: 한국·일본 저작권법상 원작자 동의 없는 재게시는 저작권 침해
- **공식 API만 활용**: Instagram Graph API 공식 허용 범위 내에서만 구현
- **투명한 동의 UI**: 원작자가 허락/거부를 명확히 인지하고 응답할 수 있는 인터페이스

---

## 2. 전체 기능 플로우

```
[SNS 모니터링]
브랜드 해시태그·멘션 자동 탐지 (Instagram Graph API)
         |
         v
[동의 획득 자동화]
자동 DM 발송 -> 허락/거부 응답 대기
         |
    +----+----+
    허락      거부/미응답(7일)
    |              |
    v              v
[포인트 지급]   제외 처리
자동 포인트 지급
+ 고유 링크 발송
         |
         v
[자사몰 게시]
리뷰 섹션 자동 등록
(원작자 인스타 프로필·게시물 링크 표시)
```

## 3. 기능 상세 정의

### 3.1 SNS 모니터링 — 해시태그·멘션 탐지

**활용 API:** Instagram Graph API (Hashtag Search API, Mention 웹훅)

| 방식 | API | 설명 |
|------|-----|------|
| 해시태그 탐색 | GET /ig-hashtag-search + GET /{hashtag-id}/recent_media | 판매자가 등록한 해시태그를 포함한 최근 게시물 탐색 |
| 브랜드 멘션 | Mention Webhook (@브랜드계정 멘션 감지) | 비즈니스 계정 멘션 시 실시간 알림 |

**API 제한 사항:**
- 해시태그 탐색: 하루 30개 해시태그까지 쿼리 가능 (비즈니스 계정 기준)
- 최근 게시물은 최대 500개 반환
- 비공개 계정 게시물은 탐지 불가

**판매자 설정:**
- 추적 해시태그 최대 10개 등록
- 탐지 주기: 6시간마다 자동 스캔

---

### 3.2 동의 획득 자동화 — DM 발송

**DM 발송 조건:**
- 공개 계정 게시물만 대상 (비공개 계정 DM 전송 금지)
- 동일 사용자에게 중복 발송 방지 (30일 내 재발송 차단)
- 하루 최대 발송 건수 캡 설정 (Meta 정책 준수)

**DM 템플릿 예시:**
```
안녕하세요 @{작성자명}님!

[스토어명]에서 고객님의 리뷰 게시물을 발견했습니다.
저희 스토어 페이지에 해당 게시물을 소개하고 싶어요.

동의해 주시면 {포인트금액}포인트를 드립니다!

[허락하기] {동의링크}
원하지 않으시면 이 메시지를 무시해 주세요.

감사합니다 — [스토어명]
```

---

### 3.3 동의 처리 — 고유 링크 및 포인트 지급

**동의 링크 구조:**
```
https://{스토어도메인}/review-consent?token={JWT}&post_id={인스타게시물ID}
```

**동의 플로우:**
1. 링크 클릭 → 스토어 로그인/회원가입
2. 동의 확인 페이지: 게시물 미리보기 + 허락/거부 선택
3. 동의 완료 → 포인트 즉시 지급 + 리뷰 큐에 추가

---

### 3.4 리뷰 게시 및 관리

**자사몰 표시 원칙:**
- 원작자 인스타그램 계정명(@username) 필수 표시
- 원본 게시물 링크 표시
- 동의 철회 요청 시 즉시 삭제 처리

**판매자 어드민 관리 상태:**

| 상태 | 설명 |
|------|------|
| 탐지됨 | DM 미발송 상태 |
| DM 발송됨 | 응답 대기 중 |
| 동의 완료 | 게시 가능 |
| 거부 | 원작자 거부 (재접촉 불가) |
| 게시됨 | 자사몰에 표시 중 |

---

### 3.5 동의 상태 관리 DB 설계


## 4. 법적 컴플라이언스 조치

### 4.1 필수 표시 의무

| 표시 항목 | 한국 | 일본 |
|---------|------|------|
| 원저작자 계정명 표시 | 필수 | 필수 |
| 원본 링크 표시 | 필수 | 필수 |
| 광고성 표시 (포인트 지급 시) | 필수 (공정위 지침) | 필수 (景表法) |
| 삭제 요청 대응 | 48시간 내 | 48시간 내 |

### 4.2 포인트 지급 시 광고성 표시 의무 검증

리뷰 허락에 대한 대가로 포인트를 지급하는 경우, 해당 리뷰는 **경제적 이익 제공을 통해 수집된 리뷰(광고성 리뷰)**에 해당할 수 있다. 국가별 법규 요건:

**한국 (공정거래법 추천·보증 등에 관한 표시·광고 심사지침)**

- 경제적 이해관계가 있는 추천·보증에는 **광고임을 명확히 표시** 의무
- 포인트를 제공받고 작성된 리뷰는 "이 리뷰는 포인트를 제공받고 작성되었습니다" 등의 문구 표시 필요
- **배지 형식 예시**: `[소정의 포인트 제공 리뷰]` 배지를 리뷰 상단에 자동 표시

**일본 (景品表示法 - 부당경품류 및 부당표시방지법)**

- 2023년 개정 스텔스 마케팅 규제(ステルスマーケティング規制, 2023년 10월 시행)
- 경제적 이익을 제공받은 SNS 게시물에 **광고임을 명시** 의무
- 게시물에 `#PR` `#광고` `#협찬` 등 표시 필요
- 셀메이트CM에서 수집해 자사몰에 게시하는 리뷰에는 출처 표시(인스타그램 원본 링크) 및 포인트 제공 여부 표시 필요

**글로벌 (FTC Guidelines — 미국)**

- FTC(미연방거래위원회): 물질적 관계(material connection)가 있는 추천에는 명확한 공개 의무
- 포인트, 무료 상품, 할인 등 경제적 이익 → `#ad` `#sponsored` 또는 한국어 `#광고` 표시

**셀메이트CM 구현 방향**

| 항목 | 구현 방법 |
|------|---------|
| 포인트 지급 리뷰 배지 | `[포인트 지급 리뷰]` 배지 자동 표시 (비활성화 불가) |
| SNS 원본 링크 | 인스타그램 원본 게시물 링크 항상 표시 |
| 국가별 언어 | 한국어 스토어: `[소정의 포인트를 제공받은 리뷰입니다]`, 일본어 스토어: `[ポイントを受け取ったレビューです]` |
| 감추기 차단 | 판매자가 광고성 배지를 숨기는 기능 제공 불가 |

> ⚠️ **미결 법률 검토 항목**: 한국 공정거래위원회 가이드라인상 "자사몰 내 포인트 지급 리뷰"의 광고성 배지 의무화 범위 → 런칭 전 법무 검토 권장

### 4.3 삭제 요청 처리 프로세스

1. 삭제 요청 접수 (이메일·어드민 신고 버튼)
2. 자동 즉시 숨김 처리 (48시간 이내 보장)
3. 운영팀 검토 후 영구 삭제 처리
4. 요청자에게 처리 완료 이메일 발송

## 7. X·틱톡·스레드 플랫폼 추가 조사 (관련 태스크: [17-13])

> 조사 기준: 2025~2026년 초 공개 문서 및 공식 API 문서 기준

---

### 7.1 X (구 트위터)

#### API 수집 가능 범위

X API v2의 검색·멘션 수집은 유료 티어에서만 실질적으로 가능하다. 2024년 이후 무료 티어는 사실상 쓰기 전용(월 500포스팅)으로 전락했으며, 트윗 읽기·검색은 기능이 없거나 극히 제한적이다.

**티어별 요금 및 읽기 한도 (2026년 3월 기준):**

| 티어 | 월 요금 | 읽기 한도 | 검색 기능 |
|------|---------|----------|----------|
| Free | $0 | 사실상 없음 (1req/15min) | 미지원 |
| Basic | $200 (연간 $175/월) | 15,000 포스트/월 | 최근 7일 |
| Pro | $5,000 | 1,000,000 포스트/월 | 최근 30일 |
| Enterprise | $42,000+ | 커스텀 | 전체 아카이브 |

- 2026년 2월 공식 출시된 **종량제(Pay-as-you-go)** 옵션도 있으나, 동일 사용량 기준 고정 티어보다 비쌀 수 있음 (Basic 동량 기준 약 $575/월 추산)
- 상품명·해시태그·멘션 기반 수집: `GET /2/tweets/search/recent` 엔드포인트 사용, Basic 이상 필요

#### 임베드 권한 및 표시 정책

- oEmbed API(`publish.twitter.com/oembed`) 또는 X Embedded Tweets 기능은 **별도 API 키 없이 무료 사용 가능**
- 단, X 공식 임베드 위젯 사용이 원칙 — 직접 렌더링 시 표시 요건 필수 준수:
  - 작성자 프로필 사진·@username·표시명 + X 프로필 링크 반드시 노출
  - X 로고를 포스트 우상단 또는 타임라인 상단에 가시적으로 표시
  - 수정된 포스트의 경우 편집 이력 표시 의무

#### 셀러 스토어 리뷰 위젯 약관 준수 사항

- 수집한 트윗을 상업적 목적(리뷰 위젯)으로 재사용 시 X 개발자 약관 내 **상업적 이용 금지 조항** 저촉 위험
- Basic 이상 티어 계약 시에도 데이터 재판매·제3자 플랫폼 배포는 금지
- 개별 포스트 원작자의 별도 동의 획득 권장 (인스타그램 DM 동의 플로우와 동일한 방식 적용)
- 결론: **X는 이커머스 리뷰 위젯 수집 목적에 비용 대비 효율이 낮고 약관 리스크가 높음**

### 7.2 틱톡 (TikTok)

#### API 종류 및 수집 가능 범위

틱톡은 목적에 따라 두 개의 독립된 API 경로를 운영한다.

**① TikTok Research API (연구용)**

- 대상: 비영리 학술기관 연구자 (미국·유럽 소재 대학 중심)
- 비상업적 조직(civil society)은 2025년 1월부터 VCE(Virtual Compute Environment) 방식으로 제한적 접근 허용
- **상업적 사업자(이커머스 플랫폼, 광고주, 브랜드)는 접근 불가**
- 해시태그 기반 영상 메타데이터(설명·조회수·댓글수·해시태그명) 쿼리 가능
- 국가별 제한: 캐나다 영상, 중국 계정 연동 영상 등 일부 데이터 미포함

**② TikTok for Business API (광고·비즈니스용)**

- 주된 용도: 광고 캠페인 생성·관리·최적화
- UGC 해시태그 리뷰 수집 목적으로는 직접 활용하기 어려움
- Hashtag Analytics API, Trending Content API 등 콘텐츠 탐색 API 존재하나, 크리에이터 마케팅 플랫폼 사업자 대상이며 별도 심사 필요

#### TikTok Shop 리뷰 데이터 접근

- TikTok Research API에 `GET /research/tiktokshop/reviews/` 엔드포인트 존재
- 수집 가능 필드: 상품명, 리뷰 텍스트, 표시명, 리뷰 좋아요 수, 생성일, 별점
- 단, 해당 엔드포인트도 **Research API 접근 승인** 필요 (상업 사업자 적용 불가)
- TikTok Shop 셀러용 Partner API는 자신의 스토어 리뷰 데이터에만 접근 가능 (타 셀러 데이터 크로스 수집 불가)

#### 영상 임베드 권한 및 저작권

- 공식 임베드(`developers.tiktok.com/doc/embed-videos`): oEmbed 방식으로 영상 작성자·설명·배경음악 정보 포함하여 외부 웹사이트에 임베드 가능
- UGC 상업적 재사용(이커머스 리뷰 위젯)은 **저작권법상 원작자 허락 필요** — 임베드 기술 허용과 상업 이용 허락은 별개
- 영상에 삽입된 배경음악의 저작권 이슈 별도 존재 (음원 라이선스 미해결 시 takedown 위험)

#### 국가별 접근 제한

- TikTok 자체가 일부 국가에서 서비스 중단·제한 위험 보유 (미국 규제 이슈 지속)
- Research API는 현재 미국·유럽 소재 기관 중심으로 운영; 한국·일본 사업자의 직접 신청 경로 불명확
- 결론: **틱톡 UGC 수집은 현재 비상업적 Research API 경로뿐이며, 이커머스 사업자 직접 사용 불가 — Phase 3에서 TikTok Creator Marketplace API 또는 제3자 솔루션 경유 방식 재검토 필요**

### 7.3 스레드 (Threads)

#### API 공개 여부 및 현재 기능 수준

Meta는 2024년 Threads API를 공식 공개했으며, 2025년에 검색·분석·멘션 웹훅 기능을 대폭 확장했다.

**주요 기능 (2025 기준):**

- **키워드 검색**: 공개 Threads 게시물을 프로그래밍 방식으로 쿼리 가능
  - 한도: 7일 롤링 기간 내 500회 쿼리
- **토픽 태그(해시태그)**: API를 통한 태그 기반 콘텐츠 탐색 지원
- **멘션 조회**: 특정 프로필이 태그된 공개 게시물 조회 가능
- **멘션 웹훅**: 계정 멘션 시 실시간 알림 수신
- **oEmbed 임베드**: 공개 게시물 임베드 코드 생성 엔드포인트 제공
  - 2025년 4월 구형 oEmbed 시스템 종료 → Meta oEmbed Read(Graph API)로 전환
  - 엔드포인트: `/threads_oembed` (Graph API)
  - 요건: Facebook 개발자 앱 등록 + `threads_basic` 권한 + Meta 앱 검토 승인

#### 인스타그램 계정 연동 구조 (Meta 플랫폼 통합)

- Threads는 인스타그램 계정과 1:1 연동 (별도 가입 불가, IG 계정 필수)
- Meta Graph API 및 Facebook 개발자 플랫폼 앱 등록 체계를 공유
- 인스타그램 비즈니스 계정으로 이미 Meta 앱 심사를 받은 경우, Threads API 추가 권한 요청으로 확장 가능
- 인스타그램 연동과 Threads 연동을 동일 Meta 앱에서 통합 관리 가능 → **기존 Instagram Graph API 구현과 연동 비용 최소화**

#### 검색·임베드 실무 적용 시 유의사항

- 비공개 계정 게시물 탐지 불가 (인스타그램과 동일)
- 검색 한도(500회/7일)는 소규모 운영에는 충분하나 대량 멘션 탐지 시 부족할 수 있음
- 임베드는 앱 검토 승인 후 무료 사용 가능
- 결론: **스레드는 Meta 플랫폼 내에서 인스타그램 연동과 통합 구현이 가능하여 추가 개발 비용이 가장 낮은 채널 — Phase 2에서 인스타그램 구현 확장 형태로 우선 고려 권장**
