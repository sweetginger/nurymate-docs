## 1. 문제 정의 — Shopify의 일본 결제 페인포인트

### 1.1 배경

Shopify는 일본 이커머스 시장에서 글로벌 표준 결제수단(신용카드, Apple Pay, Google Pay)만 네이티브로 지원하며, 일본 소비자가 실제로 사용하는 핵심 결제수단 대부분을 커버하지 못한다. 이를 보완하기 위해 한국 판매자들은 KOMOJU와 같은 서드파티 결제 게이트웨이를 추가로 도입해야 하며, 이 과정에서 구조적 페인포인트가 발생한다.

### 1.2 KOMOJU 연동의 실제 단계

KOMOJU를 Shopify에 완전 연동하기까지의 단계:

| 단계 | 내용 | 소요 시간 |
|------|------|-----------|
| 1 | Shopify 앱스토어에서 KOMOJU 앱 설치 | 즉시 |
| 2 | KOMOJU 계정 생성 및 사업자 정보 제출 | 1~2일 |
| 3 | 사업자 심사 통과 대기 | 3~5 영업일 |
| 4 | Shopify 관리자 → 설정 → 결제에서 서드파티 게이트웨이로 추가 | 즉시 |
| 5 | KOMOJU 대시보드에서 API 키 발급 후 Shopify에 입력 | 즉시 |
| 6 | 활성화할 결제수단 개별 선택 (콘비니, PAY-EASY, 후불 등) | 즉시 |
| 7 | PayPay·LINE Pay 등 스마트폰 결제 별도 심사 신청 | — |
| 8 | 스마트폰 결제 심사 통과 대기 | 결제수단별 3~4주 |

**요약**: 기본 설치만 해도 최소 3~5 영업일, 주요 QR 결제(PayPay 등)까지 완전히 활성화하려면 **최소 4~5주** 소요.

### 1.3 이중 수수료 구조

KOMOJU를 Shopify에 연동하면 두 개의 수수료가 중복 부과된다.

| 수수료 항목 | 비율 | 비고 |
|------------|------|------|
| KOMOJU 트랜잭션 수수료 (신용카드) | 3.6~3.85% | 결제수단별 상이 |
| KOMOJU 트랜잭션 수수료 (콘비니) | 2.75% | — |
| Shopify 서드파티 수수료 (Basic 플랜) | 2.0% | Shopify Payments 미사용 시 부과 |
| Shopify 서드파티 수수료 (Shopify 플랜) | 1.0% | — |
| **실질 최대 부담 (Basic + KOMOJU 신용카드)** | **최대 ~5.85%** | — |

추가 비용:
- 환불 처리 시: KOMOJU 트랜잭션 수수료 환불 불가
- 콘비니 결제 환불 시: 건당 **JPY 300** 추가 수수료 발생
- 정산 데이터 수동 대조 필요 (KOMOJU 정산 주기 ≠ Shopify Payments 정산 주기)

### 1.4 한국 법인의 구조적 진입 장벽

일본 스마트폰 결제(PayPay, LINE Pay, Rakuten Pay 등)는 각 결제수단 제공사의 직접 심사를 통과해야 연동 가능하다. **KOMOJU 공식 정책상 일본 법인이 없는 해외 사업자는 스마트폰 결제 이용 불가**이다. 이는 한국 법인 판매자가 일본 QR 결제 시장(전체 캐시리스의 9.6%, 13.5조 엔)에 완전히 접근하지 못함을 의미한다.

### 2.2 핵심 인사이트

- **LINE Pay 일본 서비스 2025년 4월 30일 종료** → 기존 사용자 전량 PayPay로 이전 완료. PayPay 사용자 7,000만 명 이상 (2025년 7월 기준), 일본 인구의 2명 중 1명 이상.
- **콘비니 결제 미도입 시 매출 손실**: 콘비니 결제 도입 시 매출 15~25% 상승 효과 추정 (KOMOJU/CartDNA 리포트).
- **BNPL(Paidy)**: 2024년 일본 BNPL 시장 규모 약 1.8조 엔 예측. 10~30대 구매층 공략에 유효.
- **신용카드**: 이커머스 결제의 65% 차지. 반드시 첫 번째로 완성해야 하는 기반.

## 4. Payment Aggregator 모델 검토

### 4.1 단일 계약으로 다수 결제수단 지원 구조

일본에서 각 결제수단을 직접 계약으로 연동하면 결제수단별 개별 심사(3~4주), 별도 계약, 별도 API 연동, 별도 정산 채널이 필요하다. Payment Aggregator(결제 집계 서비스)는 하나의 계약으로 다수 결제수단을 일괄 지원하는 구조다.

```
[판매자 단일 계약]
       ↓
[Payment Aggregator (GMO-PG, SBPS, KOMOJU 등)]
       ↓
┌─────────────────────────────────────┐
│ 신용카드 │ 콘비니 │ PayPay │ Paidy  │
│ Pay-easy│ 口座振替│ d払い  │ au PAY │
└─────────────────────────────────────┘
       ↓
[정산 단일화 → 판매자 계좌]
```

**장점**: 계약 1건, API 1개, 정산 채널 1개로 일본 전 결제수단 커버 가능.

### 4.2 후보 파트너사 비교

| 파트너사 | 포지션 | 지원 결제수단 수 | 해외 법인 계약 | 수수료 구조 | 셀메이트CM 적합성 |
|---------|-------|--------------|-------------|-----------|--------------|
| **GMO Payment Gateway (GMO-PG)** | 일본 최대 PG, 연간 처리액 10.6조 엔 | 신용카드·콘비니·PayPay·후리코미·au PAY·d払い 등 30종 이상 | 해외 법인 계약 가능 (하나은행 제휴 경로 활용) | 초기비 + 월정액 + 트랜잭션 수수료 | 최우선 검토 |
| **SoftBank Payment Service (SBPS)** | SoftBank 그룹, B2B 결제 강점 | 신용카드·콘비니·PayPay·LINE Pay·口座振替 등 20종 이상 | 한국 법인 계약 경로 확인 필요 | GMO-PG와 유사 구조 | 2순위 |
| **KOMOJU** | 해외 판매자 전문, Shopify 연동 최적화 | 신용카드·콘비니·PayPay·Paidy·Rakuten Pay 등 | 해외 법인 계약 가능. 단, 스마트폰 결제는 일본 법인 필요 | 트랜잭션 수수료 위주 (초기비 낮음) | 단기 브릿지용 |
| **Stripe Japan** | 글로벌 인프라, 2025년 일본 확장 | 신용카드·콘비니·PayPay·후리코미 (2025년 신규) | 한국 법인 계약 가능 | 단순 트랜잭션 수수료 (2.9%~3.6%) | 기술 파트너 후보 |
| **Paidy** | BNPL 전문 | 후불결제 전문 | 해외 법인 계약 가능 | 가맹점 수수료 2~6% | BNPL 단독 파트너 |

### 4.3 셀메이트 일본 법인 × GMO PG 직접 제휴 구조

셀메이트(운영사)의 일본 법인이 GMO Payment Gateway(GMO PG)와 직접 제휴 계약을 체결하는 구조를 채택한다. 이 구조에서 셀메이트CM 플랫폼은 GMO PG 계약의 서브머천트(Sub-Merchant) 역할을 하며, 거래 마진 구조로 수익을 확보한다.

**셀메이트 PG 파트너십 현황 (2026-03-11 기준)**: 이니시스(한국·일본), 토스페이먼츠, 엑심베이, PayPal multiparty 계약 완료. 이들 파트너십을 통해 수수료 수익 확보 구조가 구축되어 있음.

**계약 구조**

```
셀메이트 일본 법인
    │ (GMO PG 직접 제휴 계약)
    ▼
GMO Payment Gateway (일본 최대 PG)
    │
    ├─ 신용카드 (Visa/MC/JCB/AMEX)
    ├─ 콘비니 결제
    ├─ PayPay / Rakuten Pay / au PAY / d払い
    ├─ 은행 자동이체 (口座振替)
    └─ Paidy (BNPL) ← 별도 직접 계약

[셀메이트CM 플랫폼]
    │ (Sub-Merchant 역할)
    ├─ 판매자 결제 요청 → GMO PG 라우팅
    └─ 거래 마진 수익: GMO PG 정산 수수료 - 판매자 청구 수수료 = 마진
```

**거래 마진 구조**

| 결제수단 | GMO PG 원가 수수료 | 판매자 청구 수수료 | 셀메이트CM 마진 |
|---------|-----------------|----------------|--------------|
| 신용카드 | 2.5~3.0% | 2.9% | ~0.0~0.4% |
| 콘비니 결제 | 2.0~2.3% | 2.5% | ~0.2~0.5% |
| PayPay | 2.0~2.5% | 2.8% | ~0.3~0.8% |

> 주의: 상기 수치는 협상 목표치이며 실제 GMO PG 계약 조건에 따라 변동될 수 있다.

**장점**

- 셀메이트 일본 법인 기반으로 일본 내 모든 결제수단 직접 계약 가능 (스마트폰 결제 포함)
- 하나의 GMO PG 계약으로 30종 이상 결제수단 일괄 지원
- 거래 마진 구조로 수수료 수익 확보
- 한국 법인 경유 시 발생하는 우회 구조 불필요

**참고: 기존 경로와의 비교**

| 구분 | 기존 검토 경로 (한국 법인) | 셀메이트 일본 법인 직접 계약 |
|------|--------------------------|--------------------------|
| PayPay 연동 | 불가 (일본 법인 필요) | **가능** |
| 계약 주체 | 한국 법인 (제약 많음) | 일본 법인 (직접 계약) |
| 수익 구조 | 수수료 pass-through | **거래 마진 수익** |

## 5. 정산 단일화 설계

### 5.1 통합 정산 대시보드 구조

Shopify + KOMOJU 구조에서는 두 개의 정산 채널이 분리되어 수동 대조가 필요하다. 셀메이트CM은 모든 결제수단의 정산을 **단일 어드민 화면**에서 통합 관리한다.

```
[소비자]
  신용카드 결제 ─┐
  콘비니 결제   ─┤
  PayPay 결제   ─┤→ [셀메이트CM 결제 레이어] → [GMO-PG 단일 정산]
  Paidy 결제    ─┤                                     ↓
  기타 결제     ─┘                          [판매자 어드민 - 통합 정산 뷰]
                                                        ↓
                                           [JPY → KRW 자동 환전 정산]
```

**어드민 통합 뷰 제공 항목**:
- 결제수단별 매출 합계 (실시간)
- 미정산 잔액 (콘비니 미납 건 포함 현황)
- 정산 예정일 및 예상 정산액
- 환불 처리 현황 (건당 수수료 포함 자동 계산)
- CSV 다운로드 (일본 세무 대응용 세금 포함가 기재)

### 5.2 정산 주기 및 통화 처리

| 항목 | 설계 방향 |
|------|---------|
| 정산 구조 | **PG사 → 판매자 직접 정산** (셀메이트CM를 경유한 재정산 불가) |
| 정산 주체 | GMO PG가 판매자 계좌로 직접 정산 |
| 셀메이트CM 역할 | 거래 마진은 GMO PG 계약 시 수수료 구조에서 직접 수취 (재정산 아님) |
| 정산 주기 | GMO PG 기준: 월 2회 (매월 15일, 말일) 또는 주 1회 선택 가능 |
| 정산 통화 | JPY 정산 또는 KRW 자동 환전 중 판매자 선택 |
| 환전 타이밍 | 정산일 기준 전일 환율 고정 또는 월평균 환율 적용 선택 |
| 콘비니 미납 처리 | 유효기간(3~7일) 내 미납 시 자동 주문 취소 및 재고 반환 |
| 환불 처리 | 어드민 1클릭으로 처리. 결제수단별 환불 가능 여부 자동 판단 |

**정산 구조 선택 근거**

셀메이트CM이 PG사로부터 결제 대금을 수취한 뒤 판매자에게 재정산하는 구조는 **한국·일본 전자금융거래법·자금결제법상 결제 서비스 제공자(PG) 면허 없이는 불법**이다. 따라서 셀메이트CM은 다음 구조를 채택한다:

- **PG사 → 판매자 직접 정산**: GMO PG가 판매자 계좌로 직접 정산
- **셀메이트CM 수익**: 거래 마진(GMO PG 계약상 수수료 스플릿)으로 수취 → 재정산 없음

**미결 조사 항목**

> ⚠️ **[현재 보류 — 2026-03-25 전략 확정]**: Stripe Connect(미국 법인 기반) 크로스보더 정산 가능 국가 범위 조사 항목은 현재 보류 상태입니다. 2026-03-25 전략 회의에서 크로스보더 결제보다 한국·일본 국내 결제에 우선 집중하기로 결정되었으며, Stripe Connect + 미국 법인 설립 구조는 현재 추진 보류입니다. 관련 조사 문서: `research/stripe-connect-countries.md` (보류 중).

## 7. 수수료 구조 설계

### 7.1 셀메이트CM 수수료 원칙

1. **이중 수수료 없음**: 플랫폼 이용료와 결제 수수료가 분리되어 중복 부과되지 않는다.
2. **서드파티 앱 추가 수수료 없음**: 결제수단별 추가 앱 설치 불필요. 모든 결제수단이 플랫폼에 내장.
3. **결제수단별 단일 수수료 고지**: 판매자가 예측 가능한 비용 구조 제공.

### 7.2 경쟁사 대비 수수료 절감 시뮬레이션

**전제**: 월 거래액 1,000만 엔, 신용카드 결제 비율 65%, 콘비니 결제 15%, PayPay 20%, Shopify Basic 플랜 사용.

| 항목 | Shopify Basic + KOMOJU | 셀메이트CM (목표) | 절감액 |
|------|----------------------|----------------|--------|
| 신용카드 수수료 (650만 엔) | 3.25% (KOMOJU) + 2.0% (Shopify) = **5.25%** → 341,250엔 | **2.9%** → 188,500엔 | △152,750엔 |
| 콘비니 결제 수수료 (150만 엔) | 2.75% (KOMOJU) + 2.0% (Shopify) = **4.75%** → 71,250엔 | **2.5%** → 37,500엔 | △33,750엔 |
| PayPay (200만 엔) | 불가 (한국 법인) | **2.8%** → 56,000엔 | N/A (신규 수익) |
| 환불 수수료 (가정: 월 10건) | 건당 JPY 300 = 3,000엔 | **없음** | △3,000엔 |
| **월 수수료 합계** | **약 415,500엔+** | **약 282,000엔** | **△133,500엔 이상** |
| **연간 절감액** | — | — | **약 160만 엔** |

> **수수료 근거 (2026-03-18 조사 기준)**:
> - KOMOJU 신용카드: **3.25%** (공식 공개 수수료. 구 3.85%에서 인하됨. 출처: komoju.com/pricing)
> - KOMOJU 콘비니: **2.75%** (공식 공개 수수료)
> - Stripe Japan 신용카드: **3.6%** (공식 공개 수수료. Shopify Payments 미제공 시 대안)
> - Shopify Basic 서드파티 수수료: **2.0%** (Shopify Payments 미사용 시 부과)
> - 셀메이트CM 2.9% 달성 조건: **GMO-PG 직접 계약(셀메이트 일본 법인 기반)으로 신용카드 원가 2.5% 이하 확보 시 마진 +0.4% 가능**. KOMOJU 또는 Stripe 경유 시 적자 구조. 상세 분석: `research/payment-fee-structure.md` 참조.

> **PayPay 항목**: Shopify+KOMOJU 구조에서는 한국 법인 판매자의 PayPay 연동 자체가 불가 (KOMOJU 정책상 일본 법인 필요). 셀메이트CM은 셀메이트 일본 법인 × GMO-PG 계약으로 PayPay 제공 가능 → 이 항목은 수수료 절감이 아닌 **신규 매출 창출 효과**다.

### 7.3 Shopify Plus 대비 비교

Shopify Plus(서드파티 수수료 0%)를 사용해도 KOMOJU 수수료(신용카드 3.25%)는 그대로 발생한다. 셀메이트CM 목표 수수료(2.9%)는 Shopify Plus 구조 대비도 유리하다.

|------|-------------------|---------|

| 셀메이트CM | 2.9% (GMO-PG 직접 계약 달성 시) | 별도 플랜 설계 |

## 8. 구현 우선순위 (Phase별)

### Phase 1 — 기본 결제 패키지 (MVP, 목표: 2026년 Q3)

**목표**: 일본 이커머스 매출의 80% 이상을 커버하는 최소 필수 결제수단 지원.

| 결제수단 | 기술 파트너 | 난이도 | 비고 |
|---------|----------|-------|------|
| 신용카드 (Visa/MC/JCB/AMEX) | GMO-PG 또는 Stripe | 낮음 | 이커머스 65% |
| 콘비니 결제 (7-Eleven, Lawson, FamilyMart) | GMO-PG | 중간 | 미도입 시 매출 15~25% 손실 |
| PayPay | GMO-PG | 높음 | QR결제 1위, 7,000만 사용자 |

**완료 기준**: 판매자가 어드민에서 토글 1회로 위 3종 결제수단 즉시 활성화 가능.

### Phase 2 — 확장 결제 패키지 (목표: 2026년 Q4)

**목표**: QR 결제 전체 커버 + BNPL 지원으로 10~30대 타겟 강화.

| 결제수단 | 기술 파트너 | 난이도 | 비고 |
|---------|----------|-------|------|
| Paidy (후불결제 BNPL) | Paidy 직접 계약 | 중간 | BNPL 시장 1.8조 엔 |
| Rakuten Pay | GMO-PG | 중간 | QR결제 2위 |
| au PAY / d 払い | GMO-PG | 중간 | 각각 3위권 |
| Pay-easy (은행·ATM 결제) | GMO-PG | 중간 | 법인 B2B 거래에 유효 |

### Phase 3 — 심화 결제 패키지 (목표: 2027년 H1)

**목표**: 구독 서비스·오프라인 병행·고령층 커버로 시장 침투율 극대화.

| 결제수단 | 기술 파트너 | 난이도 | 비고 |
|---------|----------|-------|------|
| 은행 자동이체 (口座振替) | GMO-PG | 매우 높음 | 구독형 서비스 필수 |
| 착불 결제 (代金引換) | 야마토·사가와 연동 | 매우 높음 | 물류 시스템 연동 필요 |
| 楽天ポイント 포인트 결제 | Rakuten 직접 계약 | 높음 | 라쿠텐 생태계 사용자 |
| Amazon Pay | Amazon 직접 계약 | 높음 | Shopify 2025년 1월 종료 → 차별화 포인트 |

> Amazon Pay: Shopify가 2025년 1월 6일 일본에서 지원 종료. 셀메이트CM이 지원 시 이탈한 Shopify 판매자 유입 기대.

## 9. 기술 요건 개요

### 9.1 결제 레이어 아키텍처

```
[셀메이트CM 프론트엔드 체크아웃]
           ↓
[결제 오케스트레이션 레이어]
  - 결제수단 라우팅 (신용카드 → GMO-PG, Paidy → Paidy API 등)
  - 웹훅 수신 및 상태 동기화
  - 정산 데이터 집계
           ↓
┌─────────────────────────────────┐
│ GMO-PG API (메인 집계 파트너)    │
│  - 신용카드, 콘비니, PayPay 등   │
│ Paidy API (BNPL 전용)           │
└─────────────────────────────────┘
           ↓
[정산 엔진]
  - 결제수단별 정산 데이터 통합
  - JPY → KRW 환전 처리
  - 판매자 어드민 리포팅
```

### 9.2 주요 기술 요건

| 요건 | 설명 |
|------|------|
| 결제 상태 동기화 | 콘비니 결제 미납/납부 완료 웹훅 처리 및 주문 상태 자동 업데이트 |
| 환불 자동화 | 결제수단별 환불 가능 여부 판단 + 처리 API 호출 자동화 |
| 실패 결제 재시도 | 신용카드 승인 실패 시 대체 결제수단 안내 (콘비니 등) |
| 세금 처리 | 일본 소비세 10% (경감세율 8%) 자동 분류, 세금 포함가 표시 (内税表示) |
| 보안 | PCI DSS Level 1 준수, 카드 정보 직접 보유 불필요 (GMO-PG 토큰화) |
| 다국어 체크아웃 | 일본어 체크아웃 UI 기본 제공, 콘비니 안내 메시지 일본어 자동 생성 |
| 정산 리포트 API | 판매자 어드민 대시보드용 정산 데이터 API (일별/주별/월별) |

### 9.3 성공 지표 (KPI)

| 지표 | 목표값 | 측정 방법 |

| 판매자 수수료 만족도 | NPS +30 이상 | 분기 설문 |
| 이중 수수료 제거율 | 100% (서드파티 앱 추가 수수료 0원) | 리뷰 감사 |
