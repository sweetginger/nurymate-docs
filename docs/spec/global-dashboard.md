## 1. 개요

### 1.1 배경 및 목적

누리메이트의 핵심 차별화 전략은 한국 판매자의 글로벌 진출 지원이다. 판매자가 한국·일본·동남아시아 등 다국가에서 동시에 판매하게 되면, **분산된 국가별 데이터를 한눈에 파악하고 의사결정할 수 있는 통합 뷰**가 필수적으로 요구된다.

기존 analytics-spec.md의 어드민 홈 대시보드는 단일 스토어 기준 KPI에 집중한다. 글로벌 통합 대시보드는 이를 확장하여 **다국가 운영 판매자가 전체 사업을 하나의 화면에서 조망**할 수 있게 한다.

**해결할 핵심 문제:**

| 문제 | 현재 상황 | 누리메이트 글로벌 대시보드 목표 |
|------|----------|-------------------------------|
| 국가별 매출 파편화 | 일본 어드민, 한국 어드민 별도 접근 | 단일 화면에서 전체 국가 매출 통합 조회 |
| 환율 혼재 | JPY, KRW, USD 수치가 혼재되어 비교 불가 | KRW 기준 환산 통합 GMV 즉시 확인 |
| 국가별 성과 비교 불가 | 각국 전환율·반품율을 별도로 확인 | 나란히 비교하는 국가별 KPI 패널 |
| 시간대 혼란 | KST 기준 집계가 JST 영업 현황과 불일치 | 국가별 현지 시간 기준 데이터 + 글로벌 통합 기준 명시 |

### 1.2 타겟 사용자

| 사용자 유형 | 대시보드 활용 목적 |
|------------|----------------|
| **다국가 운영 판매자** | 한국+일본 동시 운영, 전체 GMV 파악, 국가별 성과 비교 |
| **글로벌 진출 준비 판매자** | 진출 전 시뮬레이션, 진출 후 초기 성과 모니터링 |
| **브랜드 오너 (운영 위임)** | 요약된 국가별 현황 한눈에 파악 |

### 1.3 설계 원칙

1. **단일 진입점**: 국가별 어드민을 별도로 열지 않아도 글로벌 대시보드 하나로 전체 조망 가능
2. **드릴다운 구조**: 글로벌 → 국가 → 카테고리 → 상품 순서로 점진적 상세 접근
3. **통화 일관성**: 기본 기준 통화(KRW)로 통합 표시, 판매자 선택으로 변경 가능
4. **시간대 명시**: 모든 지표에 해당 데이터의 기준 시간대를 명시하여 혼란 방지
5. **실행 가능한 인사이트**: 숫자 나열이 아닌 "이 국가에서 이 액션이 필요하다"는 수준의 인사이트 제공

## 2. 글로벌 통합 대시보드 지표 정의

### 2.1 최상위 통합 지표 (Global Summary Cards)

판매자가 대시보드 진입 즉시 보는 전체 사업 현황 요약 카드.

```
┌─────────────────────────────────────────────────────────────────────┐
│  글로벌 통합 현황  기준 통화: KRW  |  기준 시간: KST  |  오늘 기준    │
├──────────────┬──────────────┬──────────────┬──────────────┬─────────┤
│   총 GMV     │  총 주문 수  │  통합 전환율  │   총 방문자  │ 평균 AOV │
│  ₩24,820,000 │    312건     │    3.2%      │   9,750명    │ ₩79,551 │
│  +8.3% DoD   │  +12% DoD   │  -0.1%pp    │  +5.2% DoD  │ +3.1%   │
├──────────────┴──────────────┴──────────────┴──────────────┴─────────┤
│  국가별 GMV 비중                                                       │
│  [한국 ██████████████████ 68%] [일본 ████████ 28%] [기타 █ 4%]        │
└─────────────────────────────────────────────────────────────────────┘
```

| 지표 | 집계 방식 | 통화 기준 | 갱신 주기 |
|------|----------|---------|---------|
| **총 GMV** | 전체 국가 결제 완료 거래액 합산 (환불 전) | KRW 환산 합산 | 5분 배치 |
| **총 순 매출 (Net Sales)** | GMV - 할인 - 환불 | KRW 환산 합산 | 5분 배치 |
| **총 주문 수** | 전체 국가 주문 건수 합산 | 건수 (단위 없음) | 실시간 |
| **통합 전환율 (CVR)** | 전체 세션 대비 구매 완료 세션 비율 | % | 5분 배치 |
| **총 방문자 수** | 전체 국가 순 방문자 합산 | 명 | 5분 배치 |
| **통합 AOV** | 총 GMV ÷ 총 주문 수 | KRW | 5분 배치 |
| **국가별 GMV 비중** | 국가별 GMV ÷ 총 GMV × 100 | % | 5분 배치 |

### 2.2 국가별 KPI 비교 패널

각 국가를 나란히 비교할 수 있는 패널. 국가별 성과 차이를 즉각 파악 가능.

```
┌───────────────────┬──────────────────┬──────────────────┬─────────────┐
│ 지표               │ 한국 (KST)       │ 일본 (JST)        │ 글로벌 합계  │
├───────────────────┼──────────────────┼──────────────────┼─────────────┤
│ 오늘 GMV          │ ₩16,877,600      │ ₩6,973,400 (¥)   │ ₩24,820,000 │
│ 오늘 주문 수       │ 214건            │ 87건             │ 312건        │
│ 전환율            │ 3.5%             │ 2.6%             │ 3.2%        │
│ AOV              │ ₩78,867          │ ₩80,153          │ ₩79,551     │
│ 반품율            │ 2.1%             │ 3.8%             │ 2.7%        │
│ 신규 고객 비율     │ 41%              │ 58%              │ 46%         │
│ 재구매 고객 비율   │ 59%              │ 42%              │ 54%         │
├───────────────────┼──────────────────┼──────────────────┼─────────────┤
│ 환율 기준 (오늘)   │ KRW 기준         │ 1 JPY = 9.12 KRW │ -           │
│ 환율 적용 GMV     │ ₩16,877,600      │ ₩63,597,408 (JPY 기준 원화 환산) │ - │
└───────────────────┴──────────────────┴──────────────────┴─────────────┘
```

**국가별 KPI 정의:**

| 지표 | 정의 | 집계 단위 |
|------|------|---------|
| **국가별 GMV** | 해당 국가 구매자의 결제 완료 총액 | 현지 통화 + KRW 환산 병기 |
| **국가별 주문 수** | 해당 국가 발생 주문 건수 | 건 |
| **국가별 전환율** | 해당 국가 세션 대비 구매 완료 비율 | % |
| **국가별 AOV** | 해당 국가 평균 주문 금액 | 현지 통화 + KRW 환산 |
| **국가별 반품율** | 해당 국가 주문 중 반품 처리된 비율 | % |
| **국가별 신규 고객 비율** | 해당 기간 첫 구매 고객 비율 | % |
| **국가별 재구매율** | 해당 기간 2회 이상 구매 고객 비율 | % |

### 2.4 환율 기준 통합 정산액

판매자가 실제 수령하게 될 정산액을 통화별로 분리하여 표시.

```
┌─────────────────────────────────────────────────┐
│  이번 달 정산 예정액  (정산 주기: D+1 기준)           │
│                                                 │
│  한국 (KRW)   ₩68,450,000    정산일: 03-07      │
│  일본 (JPY)   ¥2,180,000     정산일: 03-08      │
│             = ₩19,881,600    (환율: 9.12 KRW/JPY)│
│                                                 │
│  통합 정산 예정액: ₩88,331,600                    │
│                                                 │
│  [정산 내역 상세 보기]                             │
└─────────────────────────────────────────────────┘
```

| 지표 | 정의 | 표시 방식 |
|------|------|---------|
| **국가별 정산 예정액** | 해당 국가 PG사에서 정산 예정인 금액 | 현지 통화 |
| **KRW 환산 정산액** | 정산 시점 환율 기준 원화 환산 금액 | ₩ |
| **통합 정산 예정액** | 전체 국가 정산액 KRW 합산 | ₩ |
| **환율 변동 리스크** | 기준일 환율 대비 현재 환율 차이에 따른 예상 손익 | ₩ (±) |

## 3. 국가별 ↔ 글로벌 통합 전환 UX

### 3.1 전환 패턴 설계

판매자는 두 가지 뷰 사이를 자유롭게 전환한다.

```
[어드민 상단 내비게이션]

  [글로벌 통합 뷰]  |  [한국]  |  [일본]  |  [+ 국가 추가]

   ▲ 현재 선택된 뷰에 따라 대시보드 내용 전환
```

**뷰 전환 방식:**

| 전환 | 방법 | UX 패턴 |
|------|------|---------|
| 글로벌 → 특정 국가 | 상단 탭 클릭 또는 국가별 패널 클릭 | 슬라이드 전환 애니메이션 |
| 특정 국가 → 글로벌 | 상단 "글로벌 통합 뷰" 탭 클릭 | 슬라이드 전환 |
| 글로벌 차트 드릴다운 | 파이 차트 국가 섹터 클릭 | 인라인 확장 |
| 지표 드릴다운 | 카드의 숫자 클릭 | 모달 or 슬라이드 패널 |

**URL 구조:**
```
/admin/analytics                → 글로벌 통합 뷰
/admin/analytics?country=kr     → 한국 단독 뷰
/admin/analytics?country=jp     → 일본 단독 뷰
```

북마크·공유 가능하도록 URL 파라미터로 뷰 상태를 유지한다.

### 3.2 드릴다운 구조 (글로벌 → 국가 → 카테고리 → 상품)

**레벨 1 — 글로벌 통합:**
- 전체 GMV, 국가별 비중 파이 차트, 국가별 KPI 비교 패널
- 트리거: 대시보드 기본 진입 화면

**레벨 2 — 국가:**
- 선택 국가의 단독 매출·주문·방문자·전환율
- 해당 국가 현지 통화 + KRW 환산 병기
- 트리거: 파이 차트 국가 클릭 또는 상단 탭

**레벨 3 — 카테고리:**
- 선택 국가 내 카테고리별 매출 분해
- 카테고리별 전환율·반품율 비교
- 트리거: 국가 뷰에서 카테고리 차트 클릭

**레벨 4 — 상품:**
- 선택 카테고리 내 상품별 판매 순위, 재고, 수익성
- 트리거: 카테고리 뷰에서 상품 행 클릭

```
[드릴다운 브레드크럼 예시]

글로벌 > 일본 > 의류 > 카고팬츠

[◀ 글로벌 통합으로 돌아가기]
```

### 3.3 크로스 필터링

글로벌 대시보드의 필터는 드릴다운된 뷰에도 유지된다.

| 필터 | 적용 범위 | 기본값 |
|------|---------|--------|
| 기간 | 전체 글로벌 + 국가 뷰 동시 적용 | 오늘 |
| 기준 통화 | 통합 지표 환산 기준 | KRW |
| 국가 비교 선택 | 2개 국가 선택 시 나란히 비교 | 전체 |

## 4. 실시간 vs 배치 처리 기준

### 4.1 국가별 시간대 고려 원칙

누리메이트의 주요 판매 국가는 KST(UTC+9)와 JST(UTC+9)로 동일한 시간대이나, 이후 동남아(UTC+7~+8), 미국(UTC-5~-8) 등으로 확장 시 시간대 처리가 복잡해진다.

**시간대 처리 원칙:**

| 원칙 | 내용 |
|------|------|
| 데이터 저장 기준 | 모든 이벤트 데이터는 UTC로 저장 |
| 표시 기준 | 판매자 설정 기준 시간대 (기본: KST)로 변환하여 표시 |
| 국가별 "오늘" 기준 | 해당 국가 현지 시간 기준 "오늘 00:00~23:59"로 집계 |
| 통합 집계 기준 | 글로벌 통합 뷰는 판매자 기준 시간대(KST) 기준 집계 |

**일본 판매자 예시:**
```
한국 판매자의 글로벌 대시보드 기준: KST
- "오늘 일본 매출" = 한국 시간 기준 오늘 (KST 00:00~23:59) 발생한 일본 주문
- 일본 어드민 단독 뷰 = JST 기준 오늘로 표시
→ 두 뷰가 동일 데이터이나 집계 시간대가 다를 수 있음 — UI에서 명시
```

### 4.2 처리 방식별 지표 분류

| 지표 | 처리 방식 | 지연 허용 | 근거 |
|------|---------|---------|------|
| **국가별 주문 수 (오늘)** | 실시간 스트리밍 | < 1초 | Live View 실시간 카운터 |
| **국가별 GMV (오늘)** | 5분 배치 | < 5분 | 대시보드 홈 KPI |
| **통합 GMV / 정산 예정액** | 5분 배치 | < 5분 | 판매자 의사결정 기준 지표 |
| **환율 갱신** | 1시간 배치 | < 1시간 | 외부 환율 API 호출 비용 절감 |
| **국가별 전환율** | 5분 배치 | < 5분 | 대시보드 KPI |
| **국가별 반품율** | 일별 배치 | < 24시간 | 반품 처리 완료 후 집계 |
| **코호트 분석 (국가별)** | 일별 배치 | < 24시간 | 계산 비용 큼 |
| **RFM 세그먼테이션 (국가별)** | 일별 배치 | < 24시간 | 계산 비용 큼 |
| **GA4 데이터 Pull** | 3시간 배치 | < 3시간 | GA4 API 할당량 절약 |
| **마케팅 채널 어트리뷰션** | 3시간 배치 | < 3시간 | GA4 데이터 의존 |

### 4.3 국가별 마감 처리

일별 리포트 생성 시 각 국가의 마감 시간을 현지 시간대 기준 자정(00:00)으로 처리한다.

```
[야간 배치 스케줄 예시]

UTC 15:00 (KST/JST 00:00) → 한국·일본 일별 리포트 생성
UTC 06:00 (ICT 13:00)     → 동남아 일별 리포트 생성 (Phase 3~)
UTC 06:00 (EST 01:00)     → 미국 동부 일별 리포트 생성 (Phase 4~)
```

## 5. 환율 처리 정책

### 5.1 환율 갱신 주기 및 소스

| 항목 | 정책 |
|------|------|
| **환율 갱신 주기** | 1시간마다 외부 API 호출하여 최신 환율 갱신 |
| **환율 데이터 소스** | Open Exchange Rates API (기본) / ECB (유럽 통화) / 한국은행 기준 환율 (KRW 관련 보조) |
| **거래 시점 환율** | 주문 결제 시점의 환율을 주문 레코드에 스냅샷으로 저장 (`exchange_rate` 필드) |
| **집계 표시 환율** | 대시보드 조회 시점 기준 최신 환율로 환산 (집계 목적) |
| **정산 환율** | PG사 실제 정산 시 적용된 환율 (PG사 정산 데이터 기준, 사후 업데이트) |

### 5.2 기준 통화 설정

**기본값: KRW (한국 원)**

한국 판매자가 기준이므로 KRW를 기본 기준 통화로 설정한다. 판매자는 어드민 설정에서 기준 통화를 변경할 수 있다.

| 항목 | 내용 |
|------|------|
| 기본 기준 통화 | KRW |
| 변경 가능 통화 | KRW, JPY, USD, EUR |
| 적용 범위 | 글로벌 통합 대시보드의 통합 집계 지표 |
| 국가별 단독 뷰 | 해당 국가 현지 통화로 표시 (기준 통화 설정과 무관) |

**기준 통화 표시 원칙:**

```
[글로벌 통합 뷰 — 기준 통화 KRW 설정 시]

총 GMV: ₩24,820,000  (KRW 기준)
  ├── 한국:  ₩16,877,600  (KRW 원본)
  └── 일본:  ₩7,942,400   (¥870,879 → KRW 환산, 환율 9.12 적용)

[일본 단독 뷰]

오늘 매출: ¥870,879     (JPY 원본 — 현지 통화 우선 표시)
           ₩7,942,400   (KRW 환산 참고값 병기)
```

### 5.3 환율 변동 리스크 표시

다국가 정산 시 환율 변동으로 인해 실수령액이 달라질 수 있다. 이를 판매자에게 투명하게 표시한다.

```
┌─────────────────────────────────────────────────────┐
│  환율 변동 리스크 알림  (일본 JPY)                     │
│                                                     │
│  이번 달 일본 정산 예정: ¥2,180,000                   │
│  현재 환율:   1 JPY = 9.12 KRW  (03-06 09:00 기준)  │
│  30일 전 환율: 1 JPY = 9.45 KRW                     │
│                                                     │
│  환율 변동 영향: -₩720,000 (예상 대비 감소)            │
│  현재 환산 기준 정산 예정: ₩19,881,600                │
│                                                     │
│  [환율 추이 보기]  [환리스크 관리 가이드]               │
└─────────────────────────────────────────────────────┘
```

**환율 리스크 알림 트리거 기준:**

| 조건 | 알림 방식 |
|------|---------|
| 30일 전 대비 환율 변동 ±5% 이상 | 대시보드 정산 패널에 경고 배너 |
| 30일 전 대비 환율 변동 ±10% 이상 | 어드민 상단 전역 경고 + 이메일 알림 |
| 환율 데이터 갱신 1시간 초과 실패 | 데이터 신선도 경고 ("환율 정보가 오래됐습니다") |

### 5.4 환율 데이터 보존 정책

| 데이터 유형 | 보존 기간 | 저장소 |
|-----------|---------|------|
| 시간별 환율 스냅샷 | 2년 | PostgreSQL |
| 주문 시점 환율 (스냅샷) | 무기한 (주문 데이터와 함께) | PostgreSQL |
| 현재 환율 캐시 | 1시간 | Redis |

## 6. 기술 요건

### 6.1 멀티테넌시 글로벌 집계 아키텍처

```
[데이터 수집 — 국가별 분산]

한국 주문/결제 이벤트 ──┐
일본 주문/결제 이벤트 ──┼──▶ [이벤트 큐 (Kinesis/Kafka)]
기타 국가 이벤트       ──┘              │
                                        ▼
                         [글로벌 집계 엔진]
                           ├── 국가별 테넌트 스키마 분리
                           ├── 환율 변환 레이어
                           └── 통합 집계 뷰 (Materialized View)
                                        │
                         ┌──────────────┼──────────────┐
                         ▼              ▼              ▼
                    국가별 스키마  통합 집계 DB    환율 캐시
                    (KR, JP...)   (글로벌 통합)   (Redis)
                                        │
                                        ▼
                              [Analytics API Layer]
                                        │
                                        ▼
                               글로벌 대시보드 (Next.js)
```

### 6.2 글로벌 통합 집계 API 설계

**엔드포인트 구조:**

```
GET /api/v1/analytics/global/summary
  ?period=today|7d|30d|90d|custom
  &start_date=2026-03-01
  &end_date=2026-03-06
  &base_currency=KRW
  → 글로벌 통합 KPI (GMV, 주문수, 전환율, AOV, 국가별 비중)

GET /api/v1/analytics/global/countries
  ?period=today
  &base_currency=KRW
  → 국가별 KPI 비교 테이블

GET /api/v1/analytics/global/settlement
  ?month=2026-03
  → 국가별·통화별 정산 예정액 + KRW 환산

GET /api/v1/analytics/global/exchange-rates
  → 현재 환율 캐시 조회 (Redis)

GET /api/v1/analytics/countries/{country_code}/summary
  ?period=today
  → 특정 국가 단독 KPI (현지 통화 기준)
```

**공통 응답 필드:**

```json
{
  "data": { ... },
  "meta": {
    "base_currency": "KRW",
    "timezone": "Asia/Seoul",
    "data_updated_at": "2026-03-06T09:05:00Z",
    "exchange_rates": {
      "JPY": { "rate": 9.12, "rate_at": "2026-03-06T09:00:00Z" },
      "USD": { "rate": 1340.5, "rate_at": "2026-03-06T09:00:00Z" }
    }
  }
}
```

모든 글로벌 집계 API 응답에 `exchange_rates` 객체를 포함하여 프론트엔드가 적용된 환율을 투명하게 표시할 수 있도록 한다.

### 6.3 성능 요건

| 항목 | 목표 |
|------|------|
| 글로벌 대시보드 초기 로드 | 2초 이내 (P95) |
| 국가 간 뷰 전환 응답 | 500ms 이내 |
| 글로벌 집계 API 응답 | 500ms 이내 (캐시 히트 시 100ms) |
| 5분 배치 집계 처리 완료 | 5분 이내 (VIP 고객사 3분 이내) |
| 실시간 주문 카운터 갱신 | 1초 이내 (SSE 기반) |
| 환율 캐시 만료 후 갱신 | 3초 이내 |

### 6.4 데이터 정합성 보장

글로벌 통합 집계는 여러 국가·통화·시간대가 혼재하므로 정합성 오류가 발생하기 쉽다.

**정합성 검증 포인트:**

| 검증 항목 | 방법 | 주기 |
|---------|------|------|
| 국가별 합계 = 통합 합계 | Materialized View 재계산 후 합산 검증 | 5분 배치마다 |
| 환율 적용 전후 값 일치 | 환율 변환 로직 단위 테스트 | CI/CD |
| 타임존 전환 오류 감지 | UTC 저장값 vs 현지 표시값 역산 검증 | 일별 배치 |
| 환율 데이터 신선도 | Redis TTL 만료 감지 + 갱신 실패 알림 | 실시간 |

### 6.5 캐싱 전략

| 대상 | 캐시 저장소 | TTL | 무효화 조건 |
|------|-----------|-----|-----------|
| 글로벌 통합 KPI (오늘) | Redis | 5분 | 새 주문 발생 시 즉시 무효화 |
| 국가별 KPI | Redis | 5분 | 해당 국가 새 주문 발생 시 |
| 환율 캐시 | Redis | 1시간 | 1시간 후 자동 만료 → 재조회 |
| 월별 정산 예정액 | Redis | 30분 | PG 정산 데이터 업데이트 시 |
| 국가별 매출 비중 차트 | Redis | 5분 | 새 주문 발생 시 |
