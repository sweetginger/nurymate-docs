## 1. 개요 및 배경

### 1.1 배경

디자이너 브랜드 셀러의 핵심 비즈니스 워크플로우는 프리오더(예약 주문) 기반이다.

```
[프리오더 오픈] → [수요 파악] → [생산 수량 확정] → [생산 진행] → [차수별 배송]
```

이 방식은 재고 리스크를 최소화하면서 수요에 맞춘 정확한 생산을 가능하게 한다.

**카페24의 한계**: 카페24는 프리오더 전용 기능을 제공하지 않는다. 셀러들이 직접 상품 설명에 "예약 주문" 안내를 기재하고, 엑셀로 수작업 관리하는 방식으로 운영 중이다.

**누리메이트의 차별점**: 프리오더 전 과정(예약 접수 → 생산 확정 → 차수 배분 → 배송 추적 → 구매자 알림)을 플랫폼 차원에서 자동화한다.

### 1.2 목표

- 셀러가 별도 엑셀·수작업 없이 프리오더 상품을 운영할 수 있도록 한다
- 구매자에게 예약 현황과 배송 예정일을 투명하게 전달한다
- 차수별 배송 분할을 시스템이 자동 처리하여 셀러 운영 부담을 낮춘다

### 1.3 설계 원칙

- **셀러 중심 자동화**: 반복적인 수작업(주문 분류, 알림 발송)은 시스템이 처리
- **구매자 투명성**: 예약 확정 여부, 생산 진행 상황, 배송 예정일을 실시간으로 공유
- **유연한 생산 확정 조건**: MOQ 자동 확정부터 셀러 수동 확정까지 다양한 옵션 지원
- **기존 배송 인프라 연계**: 배송 처리 단계는 `spec/shipping-integration.md`의 셀메이트 OMS 모듈 활용

## 2. 프리오더 상품 설정

### 2.1 상품 유형 선택

셀러가 상품 등록 시 상품 유형을 선택한다.

| 유형 | 설명 |
|------|------|
| **일반 상품** | 즉시 배송 가능한 실재고 상품 |
| **프리오더 상품** | 예약 접수 후 생산 확정 → 차수별 배송 |

프리오더 상품 선택 시 아래 추가 설정 항목이 노출된다.

### 2.2 예약 기간 설정

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 예약 시작일시 | 프리오더 접수 시작 날짜·시각 | 필수 |
| 예약 마감일시 | 프리오더 접수 종료 날짜·시각 | 필수 |
| 마감 후 처리 | 자동 마감 or 셀러 수동 마감 선택 | 필수 |

- 예약 마감 이후에는 신규 주문 접수가 자동 차단된다.
- 셀러는 마감 전 언제든 예약을 조기 종료할 수 있다.

### 2.3 생산 및 배송 일정 설정

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 생산 예정 착수일 | 생산 확정 후 공장 착수 예정일 | 선택 |
| 생산 완료 예정일 | 제품 완성 예정일 | 권장 |
| 1차 배송 예정일 | 1차 배송분 발송 예정일 | 필수 |
| 2차 배송 예정일 | 2차 배송분 발송 예정일 (있을 경우) | 선택 |
| N차 배송 예정일 | 추가 차수 배송 예정일 (최대 5차) | 선택 |

- 배송 차수는 최대 5차까지 설정 가능하다.
- 셀러가 배송 예정일을 변경하면 해당 차수 구매자에게 자동 알림이 발송된다.

### 2.4 최소 주문 수량(MOQ) 설정

| 항목 | 설명 |
|------|------|
| MOQ 설정 여부 | MOQ 적용 / 미적용 선택 |
| MOQ 수량 | 생산 진행 최소 주문 수량 (예: 50개) |
| MOQ 미달 시 처리 | 자동 취소 + 전액 환불 / 셀러 수동 처리 선택 |
| MOQ 달성 현황 공개 여부 | 구매자 화면에 달성률 표시 여부 |

**MOQ 달성률 표시 예시 (구매자 화면)**:
```
현재 예약 수량: 37 / 50개 (74% 달성)
마감까지 D-3 | 50개 달성 시 생산이 확정됩니다.
```

### 2.5 생산 자동 확정 설정

| 옵션 | 동작 방식 |
|------|----------|
| **수동 확정** | 예약 마감 후 셀러가 직접 생산 확정 버튼 클릭 |
| **MOQ 달성 즉시 자동 확정** | 예약 수량이 MOQ에 도달하는 순간 자동 확정 |
| **예약 마감 후 자동 확정** | 마감일 이후 MOQ 달성 여부와 무관하게 자동 확정 |
| **목표 수량 달성 시 자동 확정** | MOQ와 별개로 셀러가 설정한 목표 수량 달성 시 확정 |

- 자동 확정 발생 시 셀러에게 즉시 알림 + 구매자 전원에게 생산 확정 알림 발송.

## 3. 소비자 주문 플로우

### 3.1 주문 전 정보 제공

구매자가 프리오더 상품 페이지에서 확인할 수 있는 정보:

```
[프리오더 상품 페이지]
├── 예약 마감일: 2026-04-15 23:59
├── 생산 예정일: 2026-05-01
├── 배송 예정일: 1차 2026-05-20 / 2차 2026-06-05
├── 현재 예약 수량: 37 / 50개 (MOQ 74% 달성)  ← 공개 여부는 셀러 설정
└── 결제 방식: [사전 결제] or [생산 확정 후 결제]
```

### 3.2 결제 방식 선택 (셀러가 상품별로 설정)

| 결제 방식 | 설명 | 특징 |
|----------|------|------|
| **사전 결제** | 예약 주문 시 즉시 결제 완료 | 셀러 자금 유동성 확보 / 구매자 MOQ 미달 시 전액 환불 |
| **생산 확정 후 결제** | 생산 확정 알림 수신 후 N일 이내 결제 | 구매자 부담 낮음 / 미결제 시 자동 예약 취소 |

**생산 확정 후 결제 시 미결제 처리**:
- 확정 알림 발송 후 결제 마감일(셀러 설정, 기본 3일) 이내 미결제 시 예약 자동 취소
- 미결제 취소된 수량은 다음 차수 배정 또는 실재고 전환 가능

### 3.3 주문 완료 후 구매자 화면

```
[예약 주문 완료]
├── 주문 번호: ORD-2026-XXXXX
├── 상태: 예약 접수 완료 (생산 확정 대기 중)
├── 예상 배송 차수: 1차 배송 (2026-05-20 예정)
├── 결제 금액: 59,000원 (사전 결제 완료)
└── 예약 현황: 셀러가 생산 확정 후 알림 드립니다.
```

## 4. 셀러 생산·배송 관리 플로우

### 4.1 프리오더 운영 단계

```
1단계: 예약 접수 중
   │  - 실시간 예약 수량 모니터링
   │  - MOQ 달성 현황 확인
   ▼
2단계: 예약 마감
   │  - 자동 마감 or 수동 마감
   │  - 최종 예약 수량 확정
   ▼
3단계: 생산 확정
   │  - 수동 확정 or 자동 확정 (설정에 따라)
   │  - 구매자 전원 생산 확정 알림 자동 발송
   │  - 생산 수량 입력 (공장 발주 기준)
   ▼
4단계: 차수별 배송 분할 (시스템 자동 처리)
   │  - 생산 수량 기준으로 1차/2차/N차 자동 배분
   │  - 셀러 수동 조정 가능
   ▼
5단계: 배송 처리
   │  - 차수별 운송장 발급 (셀메이트 OMS 연동)
   │  - 배송 출발 알림 자동 발송
   ▼
6단계: 배송 완료 → 재고 전환 or 종료
```

### 4.2 생산 수량 입력 및 관리

생산 확정 후 셀러 어드민에서:

| 항목 | 설명 |
|------|------|
| 총 생산 수량 입력 | 실제 공장 발주 수량 입력 |
| 차수별 배정 수량 확인 | 시스템 자동 배분 결과 검토 |
| 잉여 수량 처리 방식 | 다음 차수 배정 / 실재고 전환 / 사용 안 함 |
| 부족 수량 처리 방식 | 구매자 취소 처리 / 다음 차수로 이월 |

## 5. 차수별 배송 분할 관리

### 5.1 자동 배분 방식

생산 수량 입력 후 시스템이 구매자를 차수별로 자동 배분한다.

**배분 기준 (셀러가 선택)**:

| 기준 | 설명 |
|------|------|
| **선착순** | 예약 주문 시간 순서대로 1차부터 배정 (기본값) |
| **랜덤** | 전체 예약자 중 무작위 추첨으로 차수 배정 |
| **프리미엄 멤버 우선** | 셀러 지정 등급(VIP, 정기구독자 등) 우선 배정 후 나머지 선착순 |

**배분 예시 (선착순 기준)**:

```
총 예약 수량: 80명
1차 생산 수량: 50개 → 주문 1번~50번 → 1차 배송 (2026-05-20)
2차 생산 수량: 30개 → 주문 51번~80번 → 2차 배송 (2026-06-05)
```

### 5.2 차수 배정 결과 처리

- 배분 완료 후 각 구매자에게 배정된 차수와 예상 배송일 알림 자동 발송
- 셀러는 배분 결과를 어드민에서 확인하고 개별 수동 조정 가능
- 수동 조정 시 변경된 구매자에게 재알림 발송

### 5.3 배송 예정일 변경 처리

셀러가 특정 차수의 배송 예정일을 변경할 경우:

1. 변경 사유 입력 (선택, 구매자에게 전달 가능)
2. 해당 차수 구매자 전원에게 자동 알림 발송
3. 구매자 주문 상세 화면의 예상 배송일 즉시 업데이트

```
[알림 예시]
주문하신 [상품명] 2차 배송 예정일이 변경되었습니다.
변경 전: 2026-06-05 → 변경 후: 2026-06-15
(사유: 원단 수급 지연)
```

## 6. 자동 알림 시스템

### 6.1 구매자 알림 트리거 및 내용

| 단계 | 트리거 | 알림 채널 | 주요 내용 |
|------|--------|----------|----------|
| **예약 확정** | 주문 완료 즉시 | 카카오 알림톡 + 이메일 | 주문번호, 예약 현황, 결제 방식 안내 |
| **MOQ 달성** | MOQ 달성 즉시 | 카카오 알림톡 | 생산 확정 예정 안내 |
| **생산 확정** | 셀러 확정 처리 시 | 카카오 알림톡 + 이메일 | 생산 확정 완료, 배정 차수, 배송 예정일 |
| **결제 요청** | 생산 확정 후 (확정 후 결제 방식) | 카카오 알림톡 + 이메일 | 결제 링크, 마감일 |
| **생산 시작** | 셀러 수동 발송 or 자동 | 카카오 알림톡 | 생산 진행 현황, 배송 예정일 재확인 |
| **배송 예정일 확정** | 셀러 운송장 입력 전 | 카카오 알림톡 | 최종 배송 예정일, 차수 확인 |
| **배송 예정일 변경** | 셀러 날짜 수정 시 | 카카오 알림톡 + 이메일 | 변경 전·후 날짜, 사유 |
| **배송 출발** | 운송장 등록 즉시 | 카카오 알림톡 | 택배사, 운송장 번호, 배송 추적 링크 |
| **MOQ 미달 취소** | 마감 후 MOQ 미달 확정 시 | 카카오 알림톡 + 이메일 | 취소 안내, 환불 일정 |

### 6.2 알림 발송 방식

- 카카오 알림톡 1순위, 수신 불가 시 SMS 대체 발송 (셀메이트 알림 인프라 활용)
- 셀러는 알림 발송 이력을 어드민에서 확인 가능
- 알림 수신 거부한 구매자는 이메일로만 발송

## 7. 셀러 어드민 UI 정의

### 7.1 프리오더 상품 목록

어드민 상품 목록에서 프리오더 상품은 별도 뱃지로 구분:

```
[상품 목록]
├── [프리오더] 핸드메이드 가죽 파우치 — 예약 중 (D-3) | 37/50개
├── [프리오더] 린넨 원피스 — 생산 확정 | 80명 배분 완료
└── [일반] 실크 스카프 — 재고 15개
```

### 7.2 차수별 주문 현황 대시보드

**위치**: 어드민 > 주문 관리 > 프리오더 관리

```
[프리오더 현황 대시보드]

상품: 핸드메이드 가죽 파우치
예약 기간: 2026-04-01 ~ 2026-04-15
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
총 예약 수량:       80명
생산 확정 수량:     80개
└── 1차 배송 배정:  50명 (배송 예정: 2026-05-20)
└── 2차 배송 배정:  30명 (배송 예정: 2026-06-05)

[생산 수량 수정] [차수 배정 조정] [알림 발송]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1차 배송 진행 현황
├── 운송장 미등록:  12명
├── 배송 중:       30명
└── 배송 완료:      8명
```

### 7.3 생산 수량 입력 화면

**위치**: 프리오더 관리 > [상품명] > 생산 수량 입력

| 입력 항목 | 설명 |
|----------|------|
| 총 생산 수량 | 실제 공장 발주 수량 |
| 1차 배송 배정 수량 | 1차 발송분 수량 |
| 2차 배송 배정 수량 | 2차 발송분 수량 (설정된 경우) |
| 잉여분 처리 | 실재고 전환 / 추가 차수 / 보관 |
| 부족분 처리 | 마지막 차수 구매자 취소 / 이월 |

시스템이 제안하는 자동 배분안을 셀러가 확인 후 수정·확정한다.

### 7.4 차수별 구매자 목록

**위치**: 프리오더 관리 > [상품명] > 차수별 구매자

| 기능 | 설명 |
|------|------|
| 차수 탭 전환 | 1차 / 2차 / N차 탭으로 구분 |
| 구매자 검색 | 이름, 주문번호, 연락처로 검색 |
| 차수 변경 | 특정 구매자를 다른 차수로 이동 |
| CSV 다운로드 | 차수별 주문 데이터 내보내기 |
| 일괄 운송장 등록 | 운송장 번호 일괄 업로드 (엑셀 업로드) |

## 8. 재고 전환 메커니즘

### 8.1 프리오더 → 실재고 전환 시나리오

| 시나리오 | 설명 | 처리 방식 |
|---------|------|----------|
| 생산 수량 > 총 예약 수량 | 잉여 재고 발생 | 셀러 선택: 실재고 상품으로 전환 or 보관 |
| 차수 배송 완료 후 반품 재고 | 교환·반품으로 재고 복귀 | 실재고 자동 편입 후 별도 관리 |
| 차수 배정 후 구매자 취소 | 이미 배정된 수량 반환 | 다음 차수로 이월 or 실재고 전환 |

### 8.2 실재고 전환 처리 절차

```
[셀러 어드민]
1. 프리오더 관리 > [상품명] > 재고 전환
2. 전환할 수량 입력
3. 전환 후 상품 페이지 상태 선택:
   ├── [즉시 판매 시작] — 일반 상품으로 전환, 재고 노출
   └── [비공개 보관]   — 상품 비공개 상태로 재고만 적립
4. 확인 → 재고 전환 완료
```

### 8.3 전환 후 상품 상태 관리

- 프리오더 상품에서 일반 상품으로 전환 시 상품 유형이 자동 변경된다.
- 기존 프리오더 주문 이력은 그대로 보존된다.
- 전환된 실재고 수량은 일반 재고 관리 화면에서 통합 관리된다.
