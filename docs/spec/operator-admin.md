## 1. 운영자 어드민 필요성 판단

### 1.1 결론: 필요함 (런칭 시점부터)

셀메이트CM는 B2B SaaS 플랫폼이다. 판매자(셀러)를 위한 셀러 어드민 외에, 플랫폼을 운영·관리하는 내부 운영팀을 위한 **별도 운영자 어드민**이 필수적이다.

**접속 URL**: `admin.sellmatecm.com`

**판단 근거:**

| 운영 업무 | 셀러 어드민으로 불가능한 이유 | 운영자 어드민 필요성 |
|---------|--------------------------|-------------------|
| PG 가입 심사 상태 모니터링 | 판매자가 PG 심사 신청 후 진행 상황 확인 필요 | ✅ PG 심사 상태 일괄 조회·알림 관리 필요 |
| 플랜 강제 변경 | 판매자 자체 변경만 가능 | ✅ CS 대응 시 수동 처리 필요 |
| 이상 거래 모니터링 | 판매자별 격리 데이터 | ✅ 전체 GMV 이상 탐지 필요 |
| 정산 분쟁 처리 | 판매자 정산 내역만 열람 | ✅ 내부 조사 및 환불 처리 필요 |
| 공지 발송 | 없음 | ✅ 전체 or 세그먼트 공지 필요 |
| 시스템 설정 | 판매자 스토어 범위로 제한 | ✅ 플랫폼 전역 설정 필요 |

### 1.2 셀러 어드민 vs 운영자 어드민 역할 분리

```
셀러 어드민 (Seller Admin)              운영자 어드민 (Operator Admin)
────────────────────────────           ────────────────────────────
- 내 스토어 상품·주문 관리             - 전체 셀러 현황 모니터링
- 내 스토어 디자인 설정               - PG 가입 심사 상태 모니터링·관리
- 내 정산·수수료 확인                 - 셀러 상태 관리 (활성/정지/해지)
- 내 고객 데이터                      - 플랫폼 정산 모니터링 (조회 전용)
                                       - 이상 거래·정책 위반 처리
                                       - 공지·이메일 발송
                                       - 시스템 설정 (수수료율, 플랜 등)
```

### 1.3 설계 원칙

- **최소 권한 원칙**: 역할별로 필요한 기능만 접근 허용
- **감사 추적**: 모든 변경 액션은 운영자 ID + 타임스탬프 기록
- **실수 방지**: 중요 액션 (정지, 해지) 시 2단계 확인 필수
- **속도 최우선**: 대량 셀러 조회 시 페이지네이션 + 캐싱 최적화

## 2. 권한 레벨 정의 (RBAC)

### 2.1 권한 레벨 구조

```
슈퍼 어드민 (Level 4)
  └─ 운영 매니저 (Level 3)
       ├─ 운영팀 (Level 2)
       └─ CS팀 (Level 1)
```

### 2.2 기능별 접근 권한 매트릭스

| 기능 | CS팀 (L1) | 운영팀 (L2) | 운영 매니저 (L3) | 슈퍼 어드민 (L4) |
|------|-----------|-------------|-----------------|-----------------|
| 셀러 목록 조회 | ✅ (읽기) | ✅ | ✅ | ✅ |
| 셀러 상세 조회 | ✅ (제한) | ✅ | ✅ | ✅ |
| CS 이력 조회/작성 | ✅ | ✅ | ✅ | ✅ |
| 셀러 상태 변경 (활성화/정지) | ❌ | ✅ | ✅ | ✅ |
| 셀러 해지 처리 | ❌ | ❌ | ✅ | ✅ |
| 플랜 수동 변경 | ❌ | ✅ | ✅ | ✅ |
| 정산 조회 | ❌ | ✅ (읽기) | ✅ | ✅ |
| 정산 이슈 메모 | ❌ | ✅ | ✅ | ✅ |
| 공지/이메일 발송 | ❌ | ✅ (초안) | ✅ (발송) | ✅ |
| 운영자 계정 관리 | ❌ | ❌ | ❌ | ✅ |
| 시스템 설정 | ❌ | ❌ | ❌ | ✅ |
| 감사 로그 조회 | ❌ | ❌ | ✅ | ✅ |

### 2.3 CS팀 조회 제한 사항

CS팀은 개인정보 보호를 위해 다음 정보를 마스킹 처리:
- 정산 계좌번호: `****-****-1234` (뒷 4자리만 표시)
- 사업자 주민번호: 전체 마스킹
- 개인 연락처: `010-****-5678`


- 이메일 발송: 선택된 셀러들에게 커스텀 이메일 일괄 발송
- 플랜 변경: 선택된 셀러들의 플랜을 일괄 변경 (슈퍼 어드민만)

## 5. 셀러 상태 관리 플로우

### 5.1 셀러 생애주기 상태도

```
신규 가입
   │
   ▼
즉시 활성 (Active)  ← 별도 심사 없이 자동 활성화
   │
   ├─ PG 심사 신청 → PG 심사 진행 중 (운영팀 모니터링)
   │       │
   │       ├─ PG 심사 완료 → 결제 활성화
   │       └─ PG 심사 반려 → 판매자에게 보완 안내
   │
   ├─ 휴면 (Dormant) ←── 60일 미접속
   │       ↓↑ 재접속 시 자동 복귀
   │
   ├─ 정지 (Suspended) ←── 이상 거래 감지, 정책 위반
   │       │
   │       └─ 재활성화 (이의 처리 후) → Active
   │
   └─ 해지 (Terminated) ←── 셀러 요청 or 운영자 (데이터 30일 보관)
```

#### PG 가입 심사 상태 관리

운영자 어드민에서 셀러별 PG 가입 심사 현황을 일괄 조회하고 알림을 관리한다.

| 심사 상태 | 표시 | 운영자 액션 |
|---------|------|-----------|
| 미신청 | ⬜ 미신청 | 셀러에게 PG 심사 안내 발송 |
| 심사 진행 중 | 🔵 진행 중 | 예상 완료일 모니터링 |
| 심사 완료 | ✅ 완료 | 결제 활성화 확인 |
| 심사 반려 | ❌ 반려 | 보완 필요 사항 셀러 안내 |
| 보완 대기 | 🟡 보완 대기 | 셀러 응답 모니터링 |

### 5.3 해지 처리 플로우

- 해지는 **운영 매니저 (Level 3) 이상**만 처리 가능
- 해지 전 체크리스트:
  - [ ] 미정산 금액 정산 완료 확인
  - [ ] 진행 중 주문 처리 완료 확인 (or 구매자 환불 처리)
  - [ ] 셀러에게 해지 안내 이메일 발송 확인
- 해지 후: 데이터 30일 보관 → 자동 삭제 스케줄 등록
- 해지 30일 내 재가입 시: 운영 매니저 수동 심사 필수

#### 셀러 상태 변경 시 자동 액션

| 상태 변경 | 자동 액션 |
|---------|---------|

| 정지 | 셀러에게 정지 사유 및 해제 방법 이메일 발송, 스토어 접근 차단 |
| 해지 | 데이터 보존 기간 안내 이메일, 30일 후 데이터 삭제 스케줄 |

## 9. 정산 모니터링

> **정산 구조 원칙**: 셀메이트CM는 PG사로부터 결제 대금을 수취한 뒤 판매자에게 재정산하지 않는다. 정산은 **PG사 → 판매자 직접 정산** 구조다. 셀메이트CM 운영자 어드민의 정산 모니터링은 각 PG사의 정산 API를 통해 집계된 데이터를 표시하는 **조회 전용** 기능이다.

### 9.1 전체 정산 현황 대시보드

- **플랫폼 총 GMV**: 일/주/월 기준
- **채널별 GMV 분포**: 한국 PG / 일본 PG / 글로벌 비율 파이차트
- **셀메이트CM 수수료 수익**: 실시간 집계 (거래 마진 기반)
- **정산 이슈 셀러 현황**: PG 심사 지연 또는 이슈 셀러 목록

```
+=========================================================================+
|  정산 모니터링                               기준: 2026-03 (이번 달)     |
+=========================================================================+
|  플랫폼 전체 GMV    셀메이트CM 수익       PG별 GMV        이번 달 이슈    |
|  ₩2.4B            ₩18.4M              이니시스 65%      2건 이슈        |
+=========================================================================+
```

### 9.2 셀러별 정산 현황 (조회 전용)

```
+=========================================================================+
|  셀러별 정산 현황           필터: [이번 달 ▾]  [PG 이슈 ▾]              |
+=========================================================================+
|  셀러명      이번 달 GMV    PG 정산 상태    이슈 메모     액션             |
|  BeautyLab  ₩12,400,000   이니시스 정상     없음         [메모]           |
|  MyFashion  ₩3,240,000    이니시스 정상     없음         [메모]           |
|  StyleBox   정지됨          N/A           정지 중        [보기]           |
+=========================================================================+
```

### 9.3 정산 이슈 기록 (운영팀 이상)

PG사 직접 처리 요청용 메모 기능. 셀메이트CM가 직접 정산을 처리하는 것이 아니라, PG사에 문의할 때 필요한 내용을 기록한다.

```
+------------------------------------------------------+
|  정산 이슈 메모 - BeautyLab (이수진)                  |
|                                                      |
|  관련 PG사: 이니시스                                  |
|  이슈 내용: [PG사에 문의할 내용 입력...]               |
|  담당자: [운영팀_이지수]                               |
|                                                      |
|  [취소]                      [메모 저장]              |
+------------------------------------------------------+
```
