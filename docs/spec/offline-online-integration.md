# 온오프라인 연계 기능 기획서

> 작성일: 2026-03-31

---

## 1. 개요

### 1.1 한 줄 요약

셀메이트 온오프라인 몰을 운영하는 디자이너 브랜드가 셀메이트CM 하나로 온라인 쇼핑몰과 오프라인 매장을 통합 운영할 수 있도록, 재고·주문·고객 데이터를 실시간 연계하는 O2O(Online-to-Offline) 기능 묶음이다.

### 1.2 전략적 위치

카페24는 온라인 전용 플랫폼으로 오프라인 매장 재고·POS 연동을 기본 제공하지 않는다. 셀메이트CM은 셀메이트 스캔페이·POS와의 깊은 연동을 통해 온오프라인 이중 운영 브랜드를 강력하게 락인(lock-in)한다. 특히 **1차 타겟인 셀메이트 이용 디자이너 브랜드**는 이미 오프라인 팝업·상설매장을 병행 운영하는 경우가 많아, 이 기능이 셀메이트CM 전환 동기의 핵심이 된다.

### 1.3 엣지포인트(Edge Point) 정의

온오프라인 연계가 실제로 마찰을 만드는 접점을 "엣지포인트"로 정의한다.

| 엣지포인트 | 기존 고통 | 셀메이트CM 해결 |
|-----------|---------|--------------|
| 재고 관리 | 온라인·오프라인 재고 시트 별도 관리, 이중 판매(oversell) 발생 | 단일 재고 원장(inventory ledger)에서 온·오프라인 차감 통합 |
| 주문 수령 | 온라인 주문은 택배만 가능. 매장 방문 고객도 앱 주문 불가 | 클릭앤콜렉트: 온라인 주문 → 매장 수령 선택 |
| 고객 포인트 | 온라인 포인트·오프라인 스탬프가 분리돼 있어 CRM 불가 | 통합 고객 ID 기반 포인트·쿠폰 온오프라인 공용 사용 |
| 매장 재고 조회 | 소비자가 "이 매장에 재고 있나요?" 카카오 문의를 보냄 | 상품 상세 페이지에서 매장별 실시간 재고 조회 |

## 2. 기능 목록 및 상세 명세

### 2.1 클릭앤콜렉트 (매장 수령 주문)

**개념**: 서점 바로드림 서비스와 같은 개념. 소비자가 온라인에서 결제 후 원하는 매장을 선택해 직접 방문해 수령하는 방식.

**판매자 설정 조건**

- 셀러 어드민에서 클릭앤콜렉트 활성화 토글
- 활성화 가능 조건: ① 매장이 1개 이상 등록됨 ② 해당 매장에 POS/스캔페이 연동됨
- 매장별로 클릭앤콜렉트 허용 여부 개별 설정 가능

**소비자 화면 — 수령 방법 선택 UI**

```
[배송 방법 선택]
─────────────────────────────────────
  ● 배송 받기 (택배)     2~3일 소요, 배송비 3,000원
  ○ 매장 수령 (무료)     오늘~1일 내 수령 가능
    ↳ 재고 있는 매장: 강남점 · 홍대점 · 성수점
      [수령 매장 선택 ▼]
─────────────────────────────────────
```

**주문 완료 후 픽업 안내 메시지**

```
주문 완료 ✅
수령 예정 매장: 성수점 (서울 성동구 성수일로 12)
픽업 준비 완료 시 카카오 알림·문자로 안내드립니다.
픽업 코드: A4-8291  (매장에서 코드 제시)
유효 기간: 2026-04-07 (7일)
```

**픽업 코드 처리 흐름 (매장 직원)**

1. POS 또는 셀러 어드민 매장 탭에서 픽업 대기 주문 목록 확인
2. 고객이 픽업 코드 또는 주문 확인 화면 제시
3. 직원이 코드 스캔 → 픽업 처리 → 주문 상태 "수령 완료" 자동 전환
4. 미수령 기간(기본 7일) 초과 시 자동 취소 및 환불 처리

### 2.2 온라인 주문 → 오프라인 재고 차감

**재고 동기화 메커니즘**

> **재고 원장 관리 주체**: 재고 원장(Inventory Ledger)은 **셀메이트 OMS 재고관리 모듈**의 역할이다. 셀메이트CM은 셀메이트 OMS 재고관리 모듈의 API를 통해 재고 조회 및 차감을 요청하는 방식으로 동작한다.

- 오프라인 POS(셀메이트 포스 모듈) 판매 발생 시 → OMS 재고관리 모듈에 차감 요청
- 온라인 주문 발생 시 → 셀메이트CM이 OMS 재고관리 모듈에 차감 요청
- 두 채널 모두 OMS 단일 재고 원장에서 차감 → oversell 방지

**재고 차감 우선순위 처리**

| 시나리오 | 처리 방식 |
|---------|---------|
| 온라인 주문 결제 완료 → 동시에 오프라인 판매 발생 | 먼저 API 요청 도달한 트랜잭션 처리, 후발 요청은 재고 부족 에러 반환 |
| 오프라인 POS 오프라인 상태 (네트워크 단절) | POS 로컬 캐시에 임시 기록 → 재연결 시 셀메이트CM와 동기화. 충돌 시 판매자 어드민 알림 |
| 클릭앤콜렉트 주문의 재고 할당 | 주문 결제 완료 시 해당 매장 재고에 "예약 차감" 처리 (픽업 완료 시 확정 차감) |

### 2.3 매장별 재고 실시간 조회 (소비자 화면)

**상품 상세 페이지 — 매장 재고 섹션**

```
[매장 재고 확인]
──────────────────────────────────────
  📍 내 위치 기준 가까운 매장 순

  강남점    강남구 신사동 537     재고 있음 ✅  (3개)
  성수점    성동구 성수일로 12    재고 있음 ✅  (1개)
  홍대점    마포구 와우산로 21    품절 ✗
  인사동점  종로구 인사동길 44    재고 있음 ✅  (5개)

  [더 많은 매장 보기]
──────────────────────────────────────
```

- 기본 노출: 위치 기반 가까운 매장 4곳
- 재고 수량은 판매자 설정에 따라 숫자 표시 / "재고 있음/품절"만 표시 선택 가능
- 위치 권한 미허용 시: 판매자가 등록한 매장 순서대로 표시

### 2.4 가까운 재고 있는 매장 찾기 (위치 기반)

**진입 경로**: 상품 상세 페이지 → "내 주변 재고 매장 찾기" 버튼

**지도 화면 구성**

```
[지도 뷰]
──────────────────────────────────────────
  현재 위치 📍 기준 반경 5km 내 매장 (재고 있는 곳만 표시)

  ┌─────────────────────────────────┐
  │   [지도 영역 — 매장 핀 표시]      │
  │   📌 강남점 (1.2km)              │
  │   📌 성수점 (2.8km)              │
  └─────────────────────────────────┘

  ─ 목록 뷰 ────────────────────────
  📌 강남점  1.2km   재고 3개   영업 중 (21:00까지)
     서울 강남구 신사동 537
     [길 찾기]   [매장 수령 주문하기]

  📌 성수점  2.8km   재고 1개   영업 중 (20:00까지)
     서울 성동구 성수일로 12
     [길 찾기]   [매장 수령 주문하기]
──────────────────────────────────────────
```

- 반경 필터: 2km / 5km / 10km / 전체
- "매장 수령 주문하기" 버튼 → 해당 매장을 수령지로 선택한 채로 장바구니 이동

### 2.5 온오프라인 통합 고객 포인트·쿠폰 관리

**통합 고객 ID 체계**

> **스캔페이 정의**: 스캔페이(ScanPay)는 셀메이트CM 앱에 탑재된 **오프라인 QR PG 결제 기능**이다. 셀러가 셀메이트CM variants(상품 옵션)에 바코드를 입력해두고, 실물 상품 택에 바코드 스티커를 부착하면 → 매장 방문 고객이 앱에서 바코드를 스캔한 후 그 자리에서 결제하는 방식이다. 스캔페이는 POS 시스템이 아니며, 셀메이트 포스(POS) 모듈은 별도로 존재한다.

- **온라인**: 셀메이트CM 고객사 회원 계정 (이메일·소셜 로그인)
- **오프라인**: 매장 고객 관리 ID 기반
- **온오프라인 계정 연결 플로우**: 고객이 오프라인 매장에서 최초 결제 또는 회원 적립 요청 시 셀메이트CM 계정과 연결 (최초 1회)

**포인트 통합 — 단일 원장 처리**

포인트는 온오프라인 구분 없이 **단일 포인트 원장(Point Ledger)**에서 관리한다. 고객이 온라인과 오프라인 어느 채널에서 적립했든, 어느 채널에서도 사용할 수 있도록 완전 통합된 경험을 제공한다.

| 채널 | 적립 | 사용 |
|------|------|------|
| 온라인 주문 | 결제 금액의 X% 적립 (판매자 설정) | 다음 온라인·오프라인 주문 모두 사용 가능 |
| 오프라인 결제 (스캔페이 / 셀메이트 포스 모듈) | 결제 금액의 X% 적립 (동일 정책 공유) | 온라인·오프라인 모두 사용 가능 |
| 클릭앤콜렉트 주문 | 온라인 주문 기준 적립 | - |

**쿠폰 통합 관리**

셀러가 쿠폰 생성 시 사용 채널을 선택할 수 있다:

| 쿠폰 채널 설정 | 설명 |
|-------------|------|
| **온오프라인 동시 사용** | 온라인 스토어 + 오프라인 매장(스캔페이·포스) 모두에서 사용 가능 |
| **온라인 전용** | 온라인 주문에서만 적용 가능 |
| **오프라인 전용** | 오프라인 매장 결제 시에만 적용 가능 |

- 쿠폰 중복 사용 방지: 셀메이트CM 중앙 검증 서버에서 단일 처리

## 3. 사용자 플로우

### 3.1 소비자 — 클릭앤콜렉트 주문 플로우

```
[소비자]
    │
    ├─ 1. 셀메이트CM 온라인 스토어에서 상품 탐색
    │
    ├─ 2. 상품 상세 페이지
    │      → "매장 재고 확인" 섹션에서 가까운 매장 재고 확인
    │
    ├─ 3. 장바구니 → 결제 화면
    │      → 배송 방법: "매장 수령" 선택
    │      → 수령 매장 선택 (재고 있는 매장만 노출)
    │
    ├─ 4. 결제 완료
    │      → 픽업 코드 발급 (알림 카카오 + 이메일)
    │      → 매장 POS에 픽업 대기 주문 자동 등록
    │
    ├─ 5. 매장 방문
    │      → 픽업 코드 제시 → 직원 스캔 → 상품 수령
    │
    └─ 6. 수령 완료
           → 주문 상태 "수령 완료" 갱신
           → 포인트 적립
```

### 3.2 판매자 — 매장 관리 및 픽업 처리 플로우

매장 등록 경로는 **두 가지**가 있다:

**경로 A — 셀메이트 포스 모듈 연동 시 (자동 동기화)**
```
[판매자 — 셀러 어드민]
    │
    ├─ 설정 > 외부 연동 > 셀메이트 포스 연동
    │      → 포스 인증 코드 입력 → 연동 승인
    │      → 포스에 등록된 매장 목록 자동 불러오기
    │      → 매장 정보 (이름, 주소, 연락처) 자동 동기화
    │      → 이후 포스에서 매장 추가·수정 시 자동 반영
    │
    ├─ 매장 목록 > 각 매장별 설정
    │      → 클릭앤콜렉트 허용 여부 토글
    │      → 스캔페이 QR PG 활성화 여부 설정
    │
    ├─ 매장 관리 > 픽업 대기 주문 탭
    │      → 픽업 대기 / 픽업 완료 / 기한 초과 필터
    │      → 고객 연락처, 주문 상품, 재고 위치 확인
    │
    └─ 재고 관리 > 매장별 재고 현황
           → 매장 선택 → 해당 매장 재고 조회
           → 수동 재고 조정 가능 (실사 후 입력)
```

**경로 B — 포스 연동 없이 직접 등록 시**
```
[판매자 — 셀러 어드민]
    │
    ├─ 설정 > 매장 관리 > 매장 직접 등록
    │      → 매장 정보 직접 입력 (이름, 주소, 영업시간, 연락처)
    │      → 스캔페이 QR PG 활성화 여부 설정
    │      → 클릭앤콜렉트 허용 여부 토글
    │      → 추후 포스 연동 시 기존 직접 등록 매장과 매핑 처리
    │
    └─ (이후 픽업 관리, 재고 조회는 경로 A와 동일)
```

## 4. 기술 구현 방향

### 4.1 재고 원장 (Inventory Ledger) 설계

> **재고 원장 관리 주체**: 재고 원장은 **셀메이트 OMS 재고관리 모듈**이 소유하고 관리한다. 셀메이트CM은 OMS API를 통해 재고를 조회하거나 차감을 요청하며, 직접 재고 원장을 보유하지 않는다.

**셀메이트 OMS 재고관리 모듈의 재고 원장 구조 (참고):**
- 상품 옵션(SKU)별로 단일 재고 원장 테이블 유지
- 채널 구분: `channel_type` 컬럼 (`ONLINE` / `OFFLINE` / `CLICK_AND_COLLECT`)
- 재고 변동 이력(ledger events)을 append-only로 기록 → 특정 시점 재고 재계산 가능
- 동시성 제어: 재고 차감 시 낙관적 잠금(Optimistic Lock) 또는 SELECT FOR UPDATE 사용

```
[셀메이트 OMS] inventory_ledger
├── sku_id
├── store_id          (NULL이면 온라인 전용 재고)
├── event_type        (PURCHASE / CANCEL / ADJUST / RESERVE / RELEASE)
├── delta             (차감: 음수, 추가: 양수)
├── channel_type
├── reference_id      (주문 ID 또는 POS 거래 ID)
└── created_at
```

**셀메이트CM의 역할:**
- 온라인 주문 결제 완료 시 → OMS 재고 차감 API 호출
- 소비자 화면 재고 조회 시 → OMS 재고 조회 API 호출
- 클릭앤콜렉트 주문 시 → OMS에 해당 매장 재고 "예약 차감" 요청

### 4.2 POS 및 스캔페이 연동 API

> **구분 정리**: 스캔페이는 셀메이트CM 앱 기반 QR PG 결제 기능이며, POS 시스템이 아니다. 셀메이트 포스 모듈은 별도 하드웨어/소프트웨어 POS이다.

- **셀메이트 포스 모듈** → OMS 재고 API 호출 (판매 발생 시 실시간 차감)
- **스캔페이 결제** → 앱에서 QR 스캔 → 셀메이트CM 결제 처리 → OMS 재고 차감
- **셀메이트CM** → 스캔페이/포스 재고 조회 (소비자 화면 매장별 재고 표시용)
- 오프라인(네트워크 단절) 대응: 포스 로컬 큐 → 재연결 시 OMS 배치 동기화
- 상세 API 스펙은 `pos-integration.md` 참조

### 4.3 위치 기반 매장 찾기

- 매장 등록 시 위·경도 자동 변환 (카카오 지도 API Geocoding)
- 소비자 위치 권한 허용 시 → 브라우저 Geolocation API 활용
- 가까운 매장 쿼리: PostgreSQL PostGIS 또는 위·경도 거리 공식(Haversine) 적용
- 지도 렌더링: 카카오 지도 SDK (한국), 구글 지도 (해외)

## 5. 우선순위 (P0~P2)

| 기능 | 우선순위 | 근거 |
|------|---------|------|
| 셀메이트 OMS 재고관리 모듈 연동 (온·오프라인 통합 차감) | **P0** | 재고 이중 판매(oversell) 방지가 서비스 신뢰의 기본 조건. OMS가 재고 원장 소유, 셀메이트CM은 API 호출 |
| 매장별 재고 실시간 조회 (소비자 화면) | **P0** | 클릭앤콜렉트 전제 조건. 소비자가 재고 없는 매장 수령 선택 불가 |
| 클릭앤콜렉트 — 결제·픽업 코드 발급 | **P1** | 런칭 직후 디자이너 브랜드 차별점으로 마케팅 활용 가능 |
| 셀러 어드민 매장 등록·픽업 관리 화면 | **P1** | 클릭앤콜렉트 운영에 필수 |
| 위치 기반 가까운 재고 매장 찾기 | **P1** | 소비자 편의 기능. 클릭앤콜렉트 전환율 향상 |
| 온오프라인 통합 포인트·쿠폰 | **P1** | 재방문 유도 CRM 핵심. 셀메이트 생태계 락인 강화 |
| POS 오프라인 동기화 (네트워크 단절 대응) | **P2** | 안정성 고도화. MVP에서는 연결 필수 조건 안내로 대체 가능 |
| 지도 UI (카카오·구글 지도 렌더링) | **P2** | 목록 뷰 우선 제공 후 지도 뷰 추가 가능 |
| 온오프라인 고객 통합 ID 연결 플로우 | **P2** | 포인트 통합의 선결 조건이나, 초기는 온라인 포인트만 제공 가능 |
